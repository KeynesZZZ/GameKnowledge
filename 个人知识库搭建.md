# 私有化个人知识库搭建

> 专题课程 | 个人知识管理 | RAG 实践

---

## 0. 项目概述

### 0.1 需求

- **私有化**：数据完全本地/自托管，不依赖云端服务
- **可交互**：Web 界面，支持自然语言问答
- **支持代码**：能理解和生成代码片段
- **支持文档**：Markdown、PDF 等格式

### 0.2 技术栈

- **LLM 引擎**：Ollama（本地模型）
- **RAG 平台**：Dify（可视化工作流）
- **向量数据库**：Weaviate（Dify 内置）
- **文档格式**：Markdown、PDF、TXT

### 6.3 导入文档

#### 方法一：Web 界面上传

```
1. 进入刚创建的知识库
2. 点击"上传文档"
3. 选择文件:
   - 支持批量上传

   - 支持上传整个文件夹
4. 点击"保存并处理"
5. 等待向量化完成
```

#### 方法二：API 批量上传

```python
import requests
import os

# 配置
DIFY_API_KEY = "your-api-key"
DIFY_API_URL = "http://localhost/v1"
KNOWLEDGE_ID = "your-knowledge-id"

def upload_document(knowledge_id, file_path):
    """上传文档到知识库"""
    url = f"{DIFY_API_URL}/datasets/{knowledge_id}/document/create_by_file"

    headers = {
        'Authorization': f'Bearer {DIFY_API_KEY}'
    }

    with open(file_path, 'rb') as f:
        files = {'file': (os.path.basename(file_path), f)}

        response = requests.post(url, headers=headers, files=files)

        if response.status_code == 200:
            result = response.json()
            print(f"✓ Uploaded: {file_path}")
            print(f"  Document ID: {result['id']}")
            return result
        else:
            print(f"✗ Failed: {file_path}")
            print(f"  Error: {response.text}")
            return None

def sync_folder(folder_path, knowledge_id):
    """同步整个文件夹"""
    total = 0
    success = 0

    for root, dirs, files in os.walk(folder_path):
        for file in files:
            if file.endswith('.md') or file.endswith('.pdf'):
                file_path = os.path.join(root, file)
                total += 1

                result = upload_document(knowledge_id, file_path)
                if result:
                    success += 1

    print(f"\nSummary: {success}/{total} files uploaded")

# 使用示例
doc_folder = "/Users/keynes/git/Doc/学习"
sync_folder(doc_folder, KNOWLEDGE_ID)
```

## 7. 创建对话应用

### 7.1 创建助手

```
1. 进入工作室
   - 顶部导航 → 工作室

2. 创建应用
   - 点击"创建应用"
   - 选择"聊天助手"
   - 应用名称: Unity 学习助手
   - 应用描述: 基于 Unity 文档的智能问答助手
   - 点击"创建"
```

### 7.2 编写系统提示词

```
提示词编辑:

你是一个 Unity 游戏开发专家，擅长回答关于 Unity 引擎、
C# 编程、UGUI、渲染、性能优化、游戏系统开发等方面的问题。

请遵循以下原则：
1. 优先基于提供的知识库文档回答
2. 如果知识库中没有相关信息，请诚实说明
3. 回答代码时使用 Markdown 代码块
4. 标注代码语言类型（csharp、yaml、bash 等）
5. 提供必要的注释和代码解释
6. 引用具体的文档名称方便查阅
```

### 7.3 添加知识库

```
1. 上下文设置
   - 应用编辑页 → 上下文 → 添加知识库

2. 选择知识库
   - 选择: Unity游戏开发

3. 配置检索参数
   - Top K: 3（检索结果数量）
   - Score 阈值: 0.7（相似度阈值）
   - Reranking 模型: bge-reranker-v2-m3（可选）

4. 保存配置
```

### 7.4 调试测试

```
1. 在对话框输入测试问题:
   "UGUI 的合批条件是什么？"

2. 查看右侧检索结果:
   - 显示匹配的文档片段
   - 显示相似度分数
   - 显示引用来源

3. 检查回答质量:
   - 是否准确引用文档
   - 代码是否正确
   - 格式是否清晰
```

### Q3: 知识库检索不准确？

```
调整分段参数:
- 减小分段长度（500 → 300）
- 增加重叠（50 → 100）
- 降低 Score 阈值
```
