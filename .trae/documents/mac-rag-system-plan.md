# Mac 专用 Ollama + Dify + OpenClaw + 向量库 RAG 一键启动系统实施计划

## 项目概述
实现一个完整的 Mac 本地知识库系统，包含一键启动和停止脚本，支持 Obsidian 笔记自动同步到向量库，通过 Dify 提供 RAG 服务。

## 实施步骤

### 第一阶段：环境检查与准备
1. **检查 Ollama 安装状态**
   - 验证 ollama 是否已安装
   - 检查 ollama 服务状态
   - 确认 bge-m3 模型已下载

2. **检查 Docker 环境**
   - 验证 Docker Desktop 是否安装
   - 检查 Docker 服务状态
   - 确认 docker-compose 可用

3. **检查 OpenClaw 安装**
   - 验证 openclaw CLI 是否安装
   - 检查 openclaw 配置

### 第二阶段：脚本创建与配置
1. **创建主启动脚本**
   - 创建 `start-rag.sh` 启动脚本
   - 配置 Obsidian 路径变量
   - 实现服务启动顺序控制
   - 添加错误处理和状态检查

2. **创建停止脚本**
   - 创建 `stop-rag.sh` 停止脚本
   - 实现服务优雅关闭
   - 清理后台进程

3. **创建状态检查脚本**
   - 创建 `status-rag.sh` 状态检查脚本
   - 检查各服务运行状态
   - 提供健康检查功能

### 第三阶段：Dify 配置优化
1. **Dify Docker 配置**
   - 配置本地数据持久化
   - 设置环境变量
   - 优化端口映射

2. **Dify RAG 参数调优**
   - 配置向量检索参数
   - 设置文本分块策略
   - 优化提示词模板

### 第四阶段：OpenClaw 集成配置
1. **OpenClaw 文件监听配置**
   - 配置 Obsidian 目录监听
   - 设置自动索引参数
   - 配置向量库连接

2. **知识库同步优化**
   - 实现增量更新机制
   - 配置冲突解决策略
   - 设置备份机制

### 第五阶段：系统集成与测试
1. **集成测试**
   - 测试一键启动流程
   - 验证服务间通信
   - 检查数据同步功能

2. **性能优化**
   - 优化启动时间
   - 配置资源限制
   - 实现日志管理

3. **用户体验优化**
   - 添加进度显示
   - 实现错误提示
   - 提供状态反馈

### 第六阶段：文档与部署
1. **创建使用文档**
   - 编写安装指南
   - 创建配置说明
   - 提供故障排除指南

2. **打包部署**
   - 创建桌面快捷方式
   - 配置自动启动
   - 实现更新机制

## 具体实现细节

### 启动脚本核心功能
- 按顺序启动 Ollama → Dify → OpenClaw
- 自动配置环境变量
- 实现后台进程管理
- 提供实时状态反馈

### 停止脚本功能
- 优雅关闭所有服务
- 清理临时文件
- 释放系统资源
- 确认关闭状态

### 状态检查功能
- 检测服务运行状态
- 显示端口占用情况
- 提供健康检查报告
- 支持日志查看

### 配置优化要点
- 向量检索 Top-K 设置
- 文本分块大小调整
- 相似度阈值优化
- 提示词模板定制

1. 一键启动完整 RAG 系统
