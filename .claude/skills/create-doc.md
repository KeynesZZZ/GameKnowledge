# 创建 UnityKnowledge 文档

> 确保文档符合 `00_元数据与模板/` 目录下的所有规范

## 使用方法

当需要创建新的 UnityKnowledge 文档时，使用此 Skill：

```bash
/create-doc "文档主题"
```

## 文档创建流程

### 步骤 1：选择文档类型

根据内容选择合适的文档类型：

| 文档类型 | 标题前缀 | 核心问题 | 使用场景 |
|---------|---------|---------|----------|
| 设计原理 | 【设计原理】 | Why | 解释底层机制、设计思想、技术权衡 |
| 教程 | 【教程】 | How to use | 讲解使用方法、API说明、学习路径 |
| 代码片段 | 【代码片段】 | Quick copy | 提供可复用代码、工具类、模板 |
| 性能数据 | 【性能数据】 | How fast | 性能测试、benchmark、对比数据 |
| 最佳实践 | 【最佳实践】 | Best practice | 推荐做法、经验总结、优化技巧 |
| 踩坑记录 | 【踩坑记录】 | Pitfalls | 常见问题、解决方案、错误案例 |
| 架构决策 | 【架构决策】 | Which | 方案对比、技术选择、决策依据 |
| 系统架构 | 【系统架构】 | What | 系统设计、模块关系、架构图 |

**选择文档类型后，继续下一步。**

---

### 步骤 2：选择模板

根据文档类型选择合适的模板：

| 文档类型 | 推荐模板 | 模板位置 |
|---------|---------|----------|
| 设计原理、教程、架构决策、系统架构 | 模板-通用编程笔记.md | `UnityKnowledge/00_元数据与模板/` |
| 代码片段 | 模板-代码片段.md | `UnityKnowledge/00_元数据与模板/` |
| 踩坑记录、Bug记录 | 模板-Bug记录.md | `UnityKnowledge/00_元数据与模板/` |
| 性能数据、性能优化 | 模板-性能分析.md | `UnityKnowledge/00_元数据与模板/` |

**选择模板后，继续下一步。**

---

### 步骤 3：填写 YAML Frontmatter

必须包含以下必填字段：

```yaml
---
title: 【文档类型】文档标题
tags: [技术领域1, 技术领域2, 核心知识点]  # 至少2个
category: 一级分类/二级分类
created: YYYY-MM-DD HH:mm
updated: YYYY-MM-DD HH:mm
description: 一句话描述文档内容（50字以内）
unity_version: 2021.3+  # 如适用
---
```

#### 必填字段说明

1. **title**：
   - 必须包含文档类型前缀（如：【代码片段】）
   - 示例：`【代码片段】对象池通用实现`

2. **tags**：
   - **至少2个标签**
   - 必须包含技术领域标签：
     - `#架构`, `#渲染`, `#物理`, `#动画`, `#UI`, `#音频`, `#网络`, `#AI`
     - `#性能优化`, `#内存优化`, `#加载优化`
   - 可选质量/类型标签：
     - `#最佳实践`, `#反模式`, `#踩坑记录`
     - `#教程`, `#深度解析`, `#代码片段`, `#工具使用`
   - 可选平台标签：
     - `#iOS`, `#Android`, `#WebGL`, `#主机`

3. **category**：
   - 使用标准分类路径
   - 格式：`一级分类/二级分类`
   - 示例：
     - `架构设计/代码片段`
     - `性能优化/内存管理`
     - `核心系统/动画系统`

4. **created / updated**：
   - 格式：`YYYY-MM-DD HH:mm`
   - 示例：`2024-03-05 14:30`

5. **description**：
   - 一句话概括核心内容
   - **不超过50字**
   - 示例：`C#泛型对象池的基础实现，减少GC分配`

---

### 步骤 4：文档结构要求

所有文档必须包含以下章节：

```markdown
## 文档定位

本文档从**[XX角度]**讲解[主题]。

**底层机制详解**：参见 [[相关文档]]
**相关实现**：参见 [[相关文档]]

## [文档具体内容]

...（根据模板填充内容）

## 相关链接

### 同一主题的系列文档
- **理论层**: [[设计原理-XXX]]
- **实现层**: [[代码片段-XXX]]
- **性能层**: [[性能数据-XXX]]
- **实战层**: [[项目实战-XXX]]
```

---

### 步骤 5：内部链接规范

使用 Obsidian 双向链接格式：

```markdown
# 同目录链接
[[文档名]]

# 跨目录链接
[[../20_核心系统/21_动画系统/文档名]]

# 带描述的链接
[[文档名|显示文本]]
```

---

## 合规性检查清单

创建文档后，必须确认以下所有项：

### ✅ YAML Frontmatter 检查

- [ ] `title` 包含正确的类型前缀（如【代码片段】）
- [ ] `tags` 数量 ≥ 2 个
- [ ] `tags` 包含技术领域标签（#架构/#UI/#性能优化等）
- [ ] `category` 使用标准分类路径
- [ ] `created` 和 `updated` 时间戳已填写
- [ ] `description` ≤ 50 字
- [ ] `unity_version` 已填写（如适用）

### ✅ 文档结构检查

- [ ] 文档开头有"文档定位"章节
- [ ] 文档末尾有"相关链接"章节
- [ ] 使用了正确的模板结构
- [ ] 内部链接使用 `[[文档名]]` 格式

### ✅ 内容质量检查

- [ ] 代码示例包含完整注释
- [ ] 文档类型与内容匹配
- [ ] 有实际价值（非空洞内容）

## 常见错误及修正

### ❌ 错误1：缺少类型前缀

**错误**：
```yaml
title: 对象池通用实现
```

**正确**：
```yaml
title: 【代码片段】对象池通用实现
```

---

### ❌ 错误2：标签数量不足

**错误**：
```yaml
tags: [对象池]
```

**正确**：
```yaml
tags: [C#, Unity, 性能优化, 对象池, 代码片段]
```

---

### ❌ 错误3：description 过长

**错误**：
```yaml
description: 这是一个关于对象池的完整实现，包含了泛型支持、状态重置、容量限制等多个功能，适合在Unity项目中使用
```

**正确**：
```yaml
description: C#泛型对象池的基础实现，支持对象复用、状态重置，减少GC分配
```

---

### ❌ 错误4：category 路径不规范

**错误**：
```yaml
category: 代码片段/Unity
```

**正确**：
```yaml
category: 架构设计/代码片段
```

---

### ❌ 错误5：缺少文档定位章节

**错误**：
```markdown
# 【代码片段】对象池通用实现

## 一、完整代码
...
```

**正确**：
```markdown
# 【代码片段】对象池通用实现

## 文档定位

本文档提供开箱即用的对象池代码实现。

**设计原理**：参见 [[设计原理-对象池本质]]
**性能数据**：参见 [[../30_性能优化/32_内存管理/【性能数据】对象池vs实例化]]

## 一、完整代码
...
```
