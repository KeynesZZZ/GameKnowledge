## 一、问题背景：没有命令模式会怎样？

### 1.1 场景：技能系统

```csharp
// ❌ 直接调用：技能逻辑散落各处
public class Player : MonoBehaviour
{
    public void UseSkill(int skillId)
    {
        switch (skillId)
        {
            case 1: // 火球术
                var fireball = Instantiate(fireballPrefab, castPoint.position, transform.rotation);
                fireball.GetComponent<Fireball>().damage = 50;
                fireball.GetComponent<Fireball>().speed = 10;
                PlayCastAnimation();
                PlayCastSound();
                break;

            case 2: // 治疗
                health += 30;
                PlayHealEffect();
                break;

            case 3: // 闪现
                transform.position += transform.forward * 5;
                PlayBlinkEffect();
                break;
            // ... 更多技能
        }
    }
}
```

### 1.2 这种代码的问题

```
┌─────────────────────────────────────────────────────────────┐
│                    直接调用的问题                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 无法撤销                                                │
│     └─ 技能释放后无法回滚                                   │
│     └─ 回合制游戏需要撤销功能                               │
│                                                             │
│  2. 无法记录/重放                                           │
│     └─ 无法保存操作历史                                     │
│     └─ 无法实现录像回放功能                                 │
│                                                             │
│  3. 无法排队/延迟执行                                       │
│     └─ 技能无法放入队列                                     │
│     └─ 无法实现"延迟释放"                                   │
│                                                             │
│  4. 难以扩展                                                │
│     └─ 加新技能要改Player类                                 │
│     └─ switch越来越长                                       │
│                                                             │
│  5. 难以组合                                                │
│     └─ 无法组合多个技能                                     │
│     └─ 无法实现"连招"                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、命令模式的核心原理

### 2.1 本质思想

```
┌─────────────────────────────────────────────────────────────┐
│                    命令模式本质                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  核心思想：                                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  将"请求"封装为对象，使你可以用不同的请求             │   │
│  │  对客户进行参数化、排队、记录日志、撤销操作           │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  关键：                                                     │
│  ├─ 动作（做什么）被封装成对象                              │
│  ├─ 调用者（谁发起）不需要知道接收者（谁执行）              │
│  └─ 可以存储、传递、排队、撤销                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 结构示意

```
┌─────────────────────────────────────────────────────────────┐
│                    命令模式结构                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐   │
│  │  Invoker    │────▶│  Command    │────▶│  Receiver   │   │
│  │  (调用者)   │     │  (命令)     │     │  (接收者)   │   │
│  └─────────────┘     └─────────────┘     └─────────────┘   │
│         │                   │                   │          │
│         │                   │                   │          │
│         │            ┌──────┴──────┐            │          │
│         │            │             │            │          │
│         │     ┌──────┴──────┐     │            │          │
│         │     │             │     │            │          │
│      按下按钮   Execute()   Undo()    实际执行动作        │
│                 保存状态                                 │
│                                                             │
│  示例：                                                     │
│  ┌─────────────┐     ┌─────────────┐     ┌─────────────┐   │

```

### 2.3 基础实现

```csharp
// 命令接口
public interface ICommand
{
    void Execute();
    void Undo();
}

// 具体命令：火球术
public class FireballCommand : ICommand
{
    private readonly Player player;
    private readonly Vector3 targetPosition;
    private GameObject fireballInstance;

    public FireballCommand(Player player, Vector3 target)
    {
        this.player = player;
        this.targetPosition = target;
    }

    public void Execute()
    {
        // 执行技能
        fireballInstance = player.CastFireball(targetPosition);
    }

    public void Undo()
    {
        // 撤销：销毁火球
        if (fireballInstance != null)
        {
            Object.Destroy(fireballInstance);
        }
    }
}

// 具体命令：治疗
public class HealCommand : ICommand
{
    private readonly Player player;
    private readonly float healAmount;
    private float previousHealth;

    public HealCommand(Player player, float amount)
    {
        this.player = player;
        this.healAmount = amount;
    }

    public void Execute()
    {
        previousHealth = player.Health;
        player.Heal(healAmount);
    }

    public void Undo()
    {
        player.SetHealth(previousHealth);
    }
}

// 调用者：输入处理器
public class InputHandler : MonoBehaviour
{
    private readonly Stack<ICommand> commandHistory = new Stack<ICommand>();
    private Player player;

    public void ExecuteCommand(ICommand command)
    {
        command.Execute();
        commandHistory.Push(command);
    }

    public void UndoLastCommand()
    {
        if (commandHistory.Count > 0)
        {
            var command = commandHistory.Pop();
            command.Undo();
        }
    }
}
```

## 三、命令模式解决了什么？

### 3.1 撤销/重做

```csharp
public class CommandManager
{
    private readonly Stack<ICommand> undoStack = new Stack<ICommand>();
    private readonly Stack<ICommand> redoStack = new Stack<ICommand>();

    public void ExecuteCommand(ICommand command)
    {
        command.Execute();
        undoStack.Push(command);
        redoStack.Clear();  // 新操作清空重做栈
    }

    public void Undo()
    {
        if (undoStack.Count > 0)
        {
            var command = undoStack.Pop();
            command.Undo();
            redoStack.Push(command);
        }
    }

    public void Redo()
    {
        if (redoStack.Count > 0)
        {
            var command = redoStack.Pop();
            command.Execute();
            undoStack.Push(command);
        }
    }
}
```

### 3.2 技能队列

```csharp
public class SkillQueue : MonoBehaviour
{
    private readonly Queue<ICommand> skillQueue = new Queue<ICommand>();
    private bool isExecuting;

    public void EnqueueSkill(ICommand skill)
    {
        skillQueue.Enqueue(skill);

        if (!isExecuting)
        {
            StartCoroutine(ExecuteQueue());
        }
    }

    private IEnumerator ExecuteQueue()
    {
        isExecuting = true;

        while (skillQueue.Count > 0)
        {
            var skill = skillQueue.Dequeue();
            skill.Execute();
            yield return new WaitForSeconds(0.5f);  // 技能间隔
        }

        isExecuting = false;
    }
}
```

### 3.3 录像/回放

```csharp
public class GameRecorder
{
    private readonly List<RecordedCommand> recordedCommands = new List<RecordedCommand>();

    private struct RecordedCommand
    {
        public ICommand command;
        public float timestamp;
    }

    public void Record(ICommand command, float time)
    {
        recordedCommands.Add(new RecordedCommand
        {
            command = command,
            timestamp = time
        });
    }

    public IEnumerator Replay()
    {
        float startTime = Time.time;

        foreach (var recorded in recordedCommands)
        {
            float waitTime = recorded.timestamp - (Time.time - startTime);
            if (waitTime > 0)
            {
                yield return new WaitForSeconds(waitTime);
            }
            recorded.command.Execute();
        }
    }
}
```

### 3.4 技能组合/连招

```csharp
// 组合命令
public class ComboCommand : ICommand
{
    private readonly List<ICommand> commands;

    public ComboCommand(params ICommand[] commands)
    {
        this.commands = new List<ICommand>(commands);
    }

    public void Execute()
    {
        foreach (var cmd in commands)
        {
            cmd.Execute();
        }
    }

    public void Undo()
    {
        // 逆序撤销
        for (int i = commands.Count - 1; i >= 0; i--)
        {
            commands[i].Undo();
        }
    }
}

// 使用：连招
var combo = new ComboCommand(
    new DashCommand(player, forward),
    new AttackCommand(player, target),
    new JumpCommand(player)
);

commandManager.ExecuteCommand(combo);
```

## 四、命令模式的权衡

### 4.1 优势

| 优势 | 说明 |
|------|------|
| **撤销/重做** | 天然支持操作历史 |
| **队列/延迟** | 可以存储后执行 |
| **记录/回放** | 可以保存和重现操作 |
| **解耦** | 调用者和接收者分离 |
| **组合** | 可以组合多个命令 |
| **扩展** | 新命令不影响现有代码 |

### 4.2 代价

```
代价：
├─ 类数量增加（每个命令一个类）
├─ 复杂度上升（需要管理命令生命周期）
├─ 内存开销（存储命令历史）
└─ 简单操作可能过度设计
```

### 4.3 什么时候使用

```
适用场景：
├─ 需要撤销/重做功能
├─ 需要操作记录/回放
├─ 需要操作队列
├─ 技能系统
├─ 输入系统（可重映射）
└─ 宏/脚本系统

不适用场景：
├─ 简单的一次性操作
├─ 不需要撤销
├─ 不需要记录
└─ 性能敏感的高频操作
```

## 五、Unity中的应用场景

### 5.1 技能系统

```csharp
// 技能基类
public abstract class SkillCommand : ICommand
{
    protected readonly Character caster;
    protected float cooldown;
    protected float lastCastTime;

    public bool CanExecute => Time.time >= lastCastTime + cooldown;

    public void Execute()
    {
        if (!CanExecute) return;

        OnExecute();
        lastCastTime = Time.time;
    }

    protected abstract void OnExecute();
    public abstract void Undo();
}

// 具体技能
public class FireballSkill : SkillCommand
{
    public FireballSkill(Character caster) : base(caster)
    {
        cooldown = 2f;
    }

    protected override void OnExecute()
    {
        // 释放火球
        Object.Instantiate(fireballPrefab, caster.CastPoint.position, caster.transform.rotation);
    }

    public override void Undo()
    {
        // 撤销逻辑
    }
}
```

### 5.2 输入系统

```csharp
// 可重映射的输入命令
public class InputManager : MonoBehaviour
{
    private Dictionary<string, ICommand> keyBindings = new Dictionary<string, ICommand>();

    public void BindKey(string key, ICommand command)
    {
        keyBindings[key] = command;
    }

    public void HandleInput(string key)
    {
        if (keyBindings.TryGetValue(key, out var command))
        {
            command.Execute();
        }
    }
}

// 玩家可以自定义按键
inputManager.BindKey("Q", new FireballSkill(player));
inputManager.BindKey("W", new HealSkill(player));
inputManager.BindKey("E", new DashSkill(player));
```
