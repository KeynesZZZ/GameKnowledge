# 【设计原理】策略模式本质

> 核心问题：如何让算法/行为可以动态切换，且易于扩展？

本文档从**底层机制角度**深入讲解策略模式本质的本质原理。

## 一、问题背景：没有策略模式会怎样？

### 1.1 场景：敌人AI行为

```csharp
// ❌ 硬编码的AI行为
public class Enemy : MonoBehaviour
{
    public string aiType;

    private void Update()
    {
        if (aiType == "Aggressive")
        {
            // 激进AI：直接冲向玩家
            var player = FindPlayer();
            if (player != null)
            {
                transform.position = Vector3.MoveTowards(
                    transform.position,
                    player.position,
                    speed * Time.deltaTime
                );
            }
        }
        else if (aiType == "Defensive")
        {
            // 防守AI：保持距离，玩家靠近才攻击
            var player = FindPlayer();
            if (player != null)
            {
                float distance = Vector3.Distance(transform.position, player.position);
                if (distance < 3f)
                {
                    // 攻击
                }
                else if (distance > 5f)
                {
                    // 靠近一点
                }
                // 否则保持位置
            }
        }
        else if (aiType == "Patrol")
        {
            // 巡逻AI：在固定点之间移动
            // ...
        }
        else if (aiType == "HitAndRun")
        {
            // 打了就跑
            // ...
        }
        // 加新AI类型就要加更多else if
    }
}
```

### 1.2 这种代码的问题

```
┌─────────────────────────────────────────────────────────────┐
│                    硬编码的问题                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 违反开闭原则                                            │
│     └─ 加新AI类型要修改Enemy类                              │
│     └─ switch/if-else 越来越长                              │
│                                                             │
│  2. 难以维护                                                │
│     └─ 所有AI逻辑混在一起                                   │
│     └─ 修改一个可能影响其他                                 │
│                                                             │
│  3. 难以测试                                                │
│     └─ 要测试所有AI类型                                     │
│     └─ 无法单独测试某个策略                                 │
│                                                             │
│  4. 无法运行时切换                                          │
│     └─ 敌人从"巡逻"变成"追击"需要改代码                     │
│                                                             │
│  5. 难以复用                                                │
│     └─ 同样的AI逻辑无法用于不同敌人类型                     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

## 二、策略模式的核心原理

### 2.1 本质思想

```
┌─────────────────────────────────────────────────────────────┐
│                    策略模式本质                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  核心思想：                                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  定义一系列算法，把它们封装起来，使它们可以互换       │   │
│  │  策略让算法独立于使用它的客户端而变化                 │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  关键：                                                     │
│  ├─ 每个"策略"是一个独立的类                                │
│  ├─ 所有策略实现相同的接口                                  │
│  ├─ 客户端只依赖接口，不依赖具体策略                        │
│  └─ 可以在运行时切换策略                                    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 结构示意

```
┌─────────────────────────────────────────────────────────────┐
│                    策略模式结构                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │                   <<interface>>                      │   │
│  │                    IStrategy                         │   │
│  │                 + Execute()                          │   │
│  └───────────────────────┬─────────────────────────────┘   │
│                          │                                  │
│          ┌───────────────┼───────────────┐                 │
│          │               │               │                 │
│          ▼               ▼               ▼                 │
│  ┌─────────────┐ ┌─────────────┐ ┌─────────────┐          │
│  │ StrategyA   │ │ StrategyB   │ │ StrategyC   │          │
│  │ (激进)      │ │ (防守)      │ │ (巡逻)      │          │
│  └─────────────┘ └─────────────┘ └─────────────┘          │
│          ▲               ▲               ▲                 │
│          │               │               │                 │
│          └───────────────┼───────────────┘                 │
│                          │                                  │
│  ┌───────────────────────┴─────────────────────────────┐   │
│  │                     Context                          │   │
│  │                   (Enemy/AI)                         │   │
│  │              - strategy: IStrategy                   │   │
│  │              + SetStrategy(IStrategy)                │   │
│  │              + ExecuteStrategy()                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 2.3 基础实现

```csharp
// 策略接口
public interface IBehaviorStrategy
{
    void Execute(Enemy enemy);
}

// 具体策略：激进
public class AggressiveStrategy : IBehaviorStrategy
{
    public void Execute(Enemy enemy)
    {
        var player = enemy.FindPlayer();
        if (player != null)
        {
            enemy.MoveTowards(player.position);
        }
    }
}

// 具体策略：防守
public class DefensiveStrategy : IBehaviorStrategy
{
    public void Execute(Enemy enemy)
    {
        var player = enemy.FindPlayer();
        if (player != null)
        {
            float distance = Vector3.Distance(enemy.Position, player.position);

            if (distance < 3f)
            {
                enemy.Attack();
            }
            else if (distance > 5f)
            {
                enemy.MoveTowards(player.position);
            }
            // 否则保持位置
        }
    }
}

// 具体策略：巡逻
public class PatrolStrategy : IBehaviorStrategy
{
    private int currentPoint;
    private readonly List<Vector3> patrolPoints;

    public PatrolStrategy(List<Vector3> points)
    {
        patrolPoints = points;
    }

    public void Execute(Enemy enemy)
    {
        var target = patrolPoints[currentPoint];
        enemy.MoveTowards(target);

        if (Vector3.Distance(enemy.Position, target) < 0.5f)
        {
            currentPoint = (currentPoint + 1) % patrolPoints.Count;
        }
    }
}

// 上下文：敌人
public class Enemy : MonoBehaviour
{
    private IBehaviorStrategy strategy;

    public void SetStrategy(IBehaviorStrategy newStrategy)
    {
        strategy = newStrategy;
    }

    private void Update()
    {
        strategy?.Execute(this);
    }

    // 供策略使用的方法
    public void MoveTowards(Vector3 target) { /* ... */ }
    public void Attack() { /* ... */ }
    public Transform FindPlayer() { /* ... */ }
    public Vector3 Position => transform.position;
}
```

## 三、策略模式解决了什么？

### 3.1 运行时切换行为

```csharp
public class Enemy : MonoBehaviour
{
    private IBehaviorStrategy currentStrategy;

    // 不同策略实例
    private IBehaviorStrategy patrolStrategy;
    private IBehaviorStrategy chaseStrategy;
    private IBehaviorStrategy attackStrategy;

    private void Start()
    {
        patrolStrategy = new PatrolStrategy(patrolPoints);
        chaseStrategy = new AggressiveStrategy();
        attackStrategy = new DefensiveStrategy();

        currentStrategy = patrolStrategy;
    }

    private void Update()
    {
        // 根据条件切换策略
        float playerDistance = GetPlayerDistance();

        if (playerDistance < 2f)
        {
            SetStrategy(attackStrategy);
        }
        else if (playerDistance < 10f)
        {
            SetStrategy(chaseStrategy);
        }
        else
        {
            SetStrategy(patrolStrategy);
        }

        currentStrategy.Execute(this);
    }
}
```

### 3.2 策略组合

```csharp
// 复合策略
public class CompositeStrategy : IBehaviorStrategy
{
    private readonly List<IBehaviorStrategy> strategies;

    public CompositeStrategy(params IBehaviorStrategy[] strategies)
    {
        this.strategies = new List<IBehaviorStrategy>(strategies);
    }

    public void Execute(Enemy enemy)
    {
        foreach (var strategy in strategies)
        {
            strategy.Execute(enemy);
        }
    }
}

// 使用：先检查再攻击
var strategy = new CompositeStrategy(
    new DetectPlayerStrategy(),
    new ChaseStrategy(),
    new AttackStrategy()
);
```

### 3.3 配置驱动

```csharp
// 从配置创建策略
public static class StrategyFactory
{
    public static IBehaviorStrategy Create(string type, params object[] args)
    {
        return type switch
        {
            "Aggressive" => new AggressiveStrategy(),
            "Defensive" => new DefensiveStrategy(),
            "Patrol" => new PatrolStrategy((List<Vector3>)args[0]),
            "HitAndRun" => new HitAndRunStrategy(),
            _ => throw new ArgumentException($"Unknown strategy: {type}")
        };
    }
}

// 从ScriptableObject配置
[CreateAssetMenu(fileName = "EnemyConfig", menuName = "Game/EnemyConfig")]
public class EnemyConfig : ScriptableObject
{
    public string behaviorType;
    public float attackRange;
    public float chaseRange;
    // ...
}

// 使用
public class Enemy : MonoBehaviour
{
    [SerializeField] private EnemyConfig config;

    private void Start()
    {
        var strategy = StrategyFactory.Create(config.behaviorType);
        SetStrategy(strategy);
    }
}
```

## 四、策略模式的权衡

### 4.1 优势

| 优势 | 说明 |
|------|------|
| **开闭原则** | 新增策略不改现有代码 |
| **运行时切换** | 可以动态改变行为 |
| **易于测试** | 每个策略独立测试 |
| **消除条件** | 避免大量if-else/switch |
| **复用性** | 策略可被多个类使用 |

### 4.2 代价

```
代价：
├─ 类数量增加（每个策略一个类）
├─ 客户端需要知道有哪些策略
├─ 策略和上下文之间的通信
└─ 简单场景可能过度设计
```

### 4.3 什么时候使用

```
适用场景：
├─ 有多种完成同一任务的方式
├─ 需要在运行时切换算法
├─ 有大量条件判断选择行为
├─ 算法可能经常变化或扩展
└─ 需要隔离复杂算法的实现

不适用场景：
├─ 只有1-2种固定行为
├─ 算法很简单
├─ 不需要切换
└─ 性能极度敏感
```

## 五、Unity中的应用场景

### 5.1 AI行为系统

```csharp
// 敌人AI状态
public enum AIState { Patrol, Chase, Attack, Flee }

public class AIController : MonoBehaviour
{
    private Dictionary<AIState, IBehaviorStrategy> strategies;
    private AIState currentState;

    private void Awake()
    {
        strategies = new Dictionary<AIState, IBehaviorStrategy>
        {
            { AIState.Patrol, new PatrolStrategy(patrolPoints) },
            { AIState.Chase, new ChaseStrategy() },
            { AIState.Attack, new AttackStrategy() },
            { AIState.Flee, new FleeStrategy() }
        };

        currentState = AIState.Patrol;
    }

    public void ChangeState(AIState newState)
    {
        currentState = newState;
    }

    private void Update()
    {
        if (strategies.TryGetValue(currentState, out var strategy))
        {
            strategy.Execute(this);
        }
    }
}
```

### 5.2 技能效果系统

```csharp
// 技能效果策略
public interface ISkillEffect
{
    void Apply(Character target);
}

public class DamageEffect : ISkillEffect
{
    private readonly float damage;

    public DamageEffect(float damage) => this.damage = damage;

    public void Apply(Character target)
    {
        target.TakeDamage(damage);
    }
}

public class HealEffect : ISkillEffect
{
    private readonly float healAmount;

    public HealEffect(float amount) => healAmount = amount;

    public void Apply(Character target)
    {
        target.Heal(healAmount);
    }
}

public class SlowEffect : ISkillEffect
{
    private readonly float slowPercent;
    private readonly float duration;

    public void Apply(Character target)
    {
        target.ApplySlow(slowPercent, duration);
    }
}

// 技能可以组合多个效果
public class Skill : ScriptableObject
{
    public List<ISkillEffect> effects = new List<ISkillEffect>();

    public void Cast(Character target)
    {
        foreach (var effect in effects)
        {
            effect.Apply(target);
        }
    }
}

## 六、策略模式 vs 状态模式

```
┌─────────────────────────────────────────────────────────────┐
│                  策略 vs 状态                               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  策略模式：                                                 │
│  ├─ 关注"怎么做"（算法的实现方式）                          │
│  ├─ 客户端选择使用哪个策略                                  │
│  ├─ 策略之间通常独立                                        │
│  └─ 示例：排序算法、支付方式、压缩算法                      │
│                                                             │
│  状态模式：                                                 │
│  ├─ 关注"处于什么状态"（对象的状态）                        │
│  ├─ 状态自己决定何时转换                                    │
│  ├─ 状态之间有转换关系                                      │
│  └─ 示例：角色状态、游戏流程、文档状态                      │
│                                                             │
│  关键区别：                                                 │
│  ├─ 策略：客户端控制切换                                    │
│  └─ 状态：状态自己控制转换                                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────┐

│  ├─ 运行时切换行为                                          │
