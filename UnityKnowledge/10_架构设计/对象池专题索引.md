## 关键概念

### 对象池核心思想

```
传统方式：
需要 → 创建 → 使用 → 销毁 → GC触发 → 卡顿

对象池方式：
启动时 → 预创建对象池
运行时 → 取出 → 使用 → 归还 → 取出 → 使用 → 归还
         ↑ 快速   ↑ 无GC
```

### 适用场景

✅ **适合对象池**：
- 子弹、特效（高频创建销毁）
- 敌人、NPC（频繁生成）
- UI元素（弹窗复用）
- 列表项（虚拟列表）

❌ **不适合对象池**：
- Boss、唯一对象（很少创建）
- 状态复杂的对象（重置成本高）
- 内存紧张时（占用常驻内存）

## 常见问题

### Q1: 对象池一定比Instantiate快吗？

**A**: 不一定。首次获取（池为空）时，对象池需要创建对象，比直接Instantiate慢。但后续获取会快很多（74x）。对象池的优势在于**重复使用**。

参见：[[../../30_性能优化/32_内存管理/【性能数据】对象池vs实例化#测试1]]

### Q2: 所有东西都应该池化吗？

**A**: 不应该。判断标准：
- 每秒创建/销毁超过10次？→ 考虑池化
- 创建成本超过0.1ms？→ 考虑池化
- 对象状态简单？→ 适合池化

参见：[[设计原理-对象池本质#3.3]]

### Q3: 如何避免状态污染？

**A**: 归还时必须重置状态：
```csharp
public void Return(GameObject obj)
{
    var bullet = obj.GetComponent<Bullet>();
    bullet.Reset();  // 重置速度、目标等
    pool.Push(obj);
}
```

参见：[[教程-对象池模式#5]]

### Q4: 对象池占用多少内存？

**A**: 取决于池大小和对象复杂度。合理设置池大小：
```
int optimalPoolSize = expectedMaxUsage * 1.2f;
```
