# 单例模式的多种实现方式

> 第2课 | 脚本与架构模块

## 1. 单例模式的核心概念

**定义**：确保一个类只有一个实例，并提供全局访问点。

**在Unity中的应用场景**：
- 游戏管理器（GameManager）
- 音频管理器（AudioManager）
- 资源加载器（ResourceManager）
- 配置数据管理（ConfigManager）

**三种实现方式的区别**：

| 类型 | 继承MonoBehaviour | 跨场景持久化 | 数据存储方式 |
|------|-------------------|--------------|--------------|
| 泛型单例 | ❌ | ❌ | 内存 |
| MonoBehaviour单例 | ✅ | ✅ | 内存 + 场景 |
| ScriptableObject单例 | ❌ | ✅ | 资源文件 |

## 2. 泛型单例类（非MonoBehaviour）

**适用场景**：纯数据管理、不需要Unity生命周期、不需要挂载到GameObject

```csharp
/// <summary>
/// 泛型单例基类 - 非MonoBehaviour版本
/// </summary>
public abstract class Singleton<T> where T : class, new()
{
    private static T instance;
    private static readonly object lockObj = new object();

    public static T Instance
    {
        get
        {
            // 双重检查锁定（线程安全）
            if (instance == null)
            {
                lock (lockObj)
                {
                    if (instance == null)
                    {
                        instance = new T();
                    }
                }
            }
            return instance;
        }
    }

    protected Singleton()
    {
        // 防止通过new创建多个实例
        if (instance != null)
        {
            throw new System.Exception($"单例 {typeof(T)} 已存在实例！");
        }
    }
}

// 使用示例
public class AudioManager : Singleton<AudioManager>
{
    private float masterVolume = 1f;

    public void PlayBGM(string clipName) { }
    public void PlaySFX(string clipName) { }
    public void SetMasterVolume(float volume) => masterVolume = volume;
}

// 调用方式
AudioManager.Instance.PlayBGM("bgm_battle");
```

**优点**：
- 线程安全
- 不依赖Unity场景
- 零GC分配（创建后）

**缺点**：
- 不能使用MonoBehaviour功能（协程、生命周期等）
- 不能在Inspector中配置

## 3. MonoBehaviour单例（场景持久化）

**适用场景**：需要MonoBehaviour功能、需要跨场景持久化

```csharp
/// <summary>
/// 泛型单例基类 - MonoBehaviour版本
/// </summary>
public abstract class MonoSingleton<T> : MonoBehaviour where T : MonoSingleton<T>
{
    private static T instance;
    private static readonly object lockObj = new object();
    private static bool applicationIsQuitting = false;

    public static T Instance
    {
        get
        {
            if (applicationIsQuitting)
            {
                Debug.LogWarning($"[MonoSingleton] 应用正在退出，返回null: {typeof(T)}");
                return null;
            }

            lock (lockObj)
            {
                if (instance == null)
                {
                    // 在场景中查找现有实例
                    instance = FindObjectOfType<T>();

                    if (instance == null)
                    {
                        // 创建新的GameObject
                        var singleton = new GameObject($"[{typeof(T).Name}]");
                        instance = singleton.AddComponent<T>();
                        DontDestroyOnLoad(singleton);

                        Debug.Log($"[MonoSingleton] 创建新实例: {typeof(T)}");
                    }
                }

                return instance;
            }
        }
    }

    protected virtual void Awake()
    {
        if (instance == null)
        {
            instance = this as T;
            DontDestroyOnLoad(gameObject);
        }
        else if (instance != this)
        {
            Debug.LogWarning($"[MonoSingleton] 检测到重复实例，销毁: {gameObject.name}");
            Destroy(gameObject);
        }
    }

    protected virtual void OnDestroy()
    {
        if (instance == this)
        {
            instance = null;
        }
    }

    protected virtual void OnApplicationQuit()
    {
        applicationIsQuitting = true;
    }
}

// 使用示例
public class GameManager : MonoSingleton<GameManager>
{
    public int CurrentLevel { get; private set; }
    public GameState State { get; private set; }

    protected override void Awake()
    {
        base.Awake();
        // 初始化游戏数据
        CurrentLevel = 1;
        State = GameState.Menu;
    }

    public void StartGame()
    {
        State = GameState.Playing;
        StartCoroutine(GameLoop());
    }

    private IEnumerator GameLoop()
    {
        while (State == GameState.Playing)
        {
            yield return null;
        }
    }
}

// 调用方式
GameManager.Instance.StartGame();
```

### 关键点解析

| 技术点 | 说明 |
|--------|------|
| `DontDestroyOnLoad` | 切换场景时不销毁 |
| `FindObjectOfType` | 查找场景中已存在的实例 |
| `applicationIsQuitting` | 防止退出时访问已销毁的实例 |
| `lock` | 线程安全保护 |

### 常见陷阱

```csharp
// ❌ 错误：在Awake中直接访问Instance可能导致死锁
protected override void Awake()
{
    base.Awake();
    var gm = GameManager.Instance; // 如果GameManager也这样做...
}

// ✅ 正确：在Start中访问其他单例
protected virtual void Start()
{
    var gm = GameManager.Instance; // 安全
}
```

## 4. ScriptableObject单例（配置数据）

**适用场景**：配置数据、运行时数据持久化、需要Inspector配置

```csharp
/// <summary>
/// ScriptableObject单例基类
/// </summary>
public abstract class SOSingleton<T> : ScriptableObject where T : SOSingleton<T>
{
    private static T instance;

    public static T Instance
    {
        get
        {
            if (instance == null)
            {
                // 从Resources加载
                instance = Resources.Load<T>($"Data/{typeof(T).Name}");

                if (instance == null)
                {
                    Debug.LogError($"[SOSingleton] 未找到资源文件: Data/{typeof(T).Name}");
                }
            }
            return instance;
        }
    }

    /// <summary>
    /// 运行时创建实例（用于测试或动态生成）
    /// </summary>
    public static T CreateRuntimeInstance()
    {
        if (instance == null)
        {
            instance = CreateInstance<T>();
        }
        return instance;
    }
}

// 使用示例：游戏配置
[CreateAssetMenu(fileName = "GameConfig", menuName = "Match3/GameConfig")]
public class GameConfig : SOSingleton<GameConfig>
{
    [Header("棋盘设置")]
    public int boardWidth = 7;
    public int boardHeight = 7;

    [Header("消除设置")]
    public int minMatchCount = 3;
    public float matchAnimDuration = 0.3f;

    [Header("伤害系数")]
    public float damageMultiplier3 = 1f;
    public float damageMultiplier4 = 1.2f;
    public float damageMultiplier5 = 1.5f;

    public float GetDamageMultiplier(int matchCount)
    {
        return matchCount switch
        {
            >= 5 => damageMultiplier5,
            4 => damageMultiplier4,
            _ => damageMultiplier3
        };
    }
}

// 调用方式
var config = GameConfig.Instance;
Debug.Log($"棋盘大小: {config.boardWidth}x{config.boardHeight}");
```

### 资源目录结构

```
Assets/
└── Resources/
    └── Data/
        ├── GameConfig.asset
        ├── HeroConfig.asset
        └── SkillConfig.asset
```

**优点**：
- 可在Inspector中配置
- 数据与代码分离
- 运行时修改的数据会保留（编辑器模式）

**缺点**：
- 需要放在Resources目录（或使用Addressables）
- 运行时创建的实例不会持久化到磁盘

## 5. 单例的优缺点与替代方案

### 优点
- 全局访问，使用方便
- 延迟初始化
- 避免重复创建

### 缺点
- **全局状态**：难以追踪数据变化
- **紧耦合**：代码难以测试
- **生命周期问题**：销毁顺序不可控
- **隐藏依赖**：不明显的依赖关系

### 替代方案：依赖注入

```csharp
// 使用Zenject/VContainer替代单例

// 1. 定义接口
public interface IAudioManager
{
    void PlayBGM(string clipName);
    void PlaySFX(string clipName);
}

// 2. 实现类
public class AudioManager : IAudioManager
{
    public void PlayBGM(string clipName) { }
    public void PlaySFX(string clipName) { }
}

// 3. 注册依赖（Installer）
public class GameInstaller : MonoInstaller
{
    public override void InstallBindings()
    {
        Container.Bind<IAudioManager>()
                 .To<AudioManager>()
                 .AsSingle();
    }
}

// 4. 注入使用
public class Player : MonoBehaviour
{
    private IAudioManager audioManager;

    [Inject]  // Zenject注入
    public void Construct(IAudioManager audioManager)
    {
        this.audioManager = audioManager;
    }

    void OnAttack()
    {
        audioManager.PlaySFX("attack");
    }
}
```

### 替代方案：服务定位器模式

```csharp
/// <summary>
/// 服务定位器 - 更灵活的替代方案
/// </summary>
public static class ServiceLocator
{
    private static readonly Dictionary<Type, object> services = new();

    public static void Register<T>(T service)
    {
        var type = typeof(T);
        if (services.ContainsKey(type))
        {
            Debug.LogWarning($"服务 {type.Name} 已注册，将被覆盖");
        }
        services[type] = service;
    }

    public static void Unregister<T>()
    {
        services.Remove(typeof(T));
    }

    public static T Get<T>()
    {
        if (services.TryGetValue(typeof(T), out var service))
        {
            return (T)service;
        }
        throw new Exception($"服务 {typeof(T).Name} 未注册");
    }

    public static bool TryGet<T>(out T service)
    {
        if (services.TryGetValue(typeof(T), out var obj))
        {
            service = (T)obj;
            return true;
        }
        service = default;
        return false;
    }

    public static void Clear()
    {
        services.Clear();
    }
}

// 使用示例
// 注册服务
ServiceLocator.Register<IAudioManager>(new AudioManager());
ServiceLocator.Register<IGameManager>(GameManager.Instance);

// 获取服务
var audio = ServiceLocator.Get<IAudioManager>();
audio.PlaySFX("click");
```

## 6. 实战应用：三消游戏的管理器设计

```csharp
/// <summary>
/// 游戏管理器 - 使用MonoBehaviour单例
/// </summary>
public class Match3GameManager : MonoSingleton<Match3GameManager>
{
    [SerializeField] private GameConfig config;

    public GameConfig Config => config;
    public int CurrentLevel { get; private set; }
    public int Score { get; private set; }

    protected override void Awake()
    {
        base.Awake();
        InitializeGame();
    }

    private void InitializeGame()
    {
        CurrentLevel = 1;
        Score = 0;

        // 加载配置（如果未在Inspector中设置）
        if (config == null)
        {
            config = GameConfig.Instance;
        }
    }

    public void AddScore(int matchCount)
    {
        var multiplier = config.GetDamageMultiplier(matchCount);
        Score += (int)(matchCount * 100 * multiplier);
    }

    public void NextLevel()
    {
        CurrentLevel++;
        // 加载下一关...
    }
}

/// <summary>
/// 棋盘管理器 - 使用MonoBehaviour单例
/// </summary>
public class BoardManager : MonoSingleton<BoardManager>
{
    private Board board;

    public Board Board => board;

    public void Initialize(int width, int height)
    {
        board = new Board(width, height);
    }

    public void ClearBoard()
    {
        board?.Clear();
    }
}

/// <summary>
/// 使用示例
/// </summary>
public class MatchDetector : MonoBehaviour
{
    public void ProcessMatch(List<Match> matches)
    {
        foreach (var match in matches)
        {
            // 通过单例访问
            Match3GameManager.Instance.AddScore(match.Count);
        }

        // 通知棋盘管理器
        BoardManager.Instance.Board.ApplyMatches(matches);
    }
}
```

## 本课小结

### 单例实现对比

| 实现方式 | 适用场景 | 关键技术 |
|----------|----------|----------|
| 泛型单例 | 纯数据、无Unity依赖 | `new()`约束、双重检查锁 |
| MonoBehaviour单例 | 需要生命周期、跨场景 | `DontDestroyOnLoad`、`FindObjectOfType` |
| ScriptableObject单例 | 配置数据、Inspector配置 | `Resources.Load`、`CreateInstance` |

### 替代方案对比

| 替代方案 | 优点 | 适用场景 |
|----------|------|----------|
| 依赖注入 | 解耦、易测试 | 大型项目、团队协作 |
| 服务定位器 | 灵活、可替换 | 中型项目、模块化设计 |
