# 【设计原理】状态机本质

> 核心问题：如何优雅地管理对象的不同状态及其转换？

## 问题背景：没有状态机会怎样？

### 场景：角色行为控制

```csharp
// ❌ 用大量布尔变量控制行为
public class Player : MonoBehaviour
{
    private bool isIdle;
    private bool isWalking;
    private bool isRunning;
    private bool isJumping;
    private bool isAttacking;
    private bool isDead;
    private bool isHurt;

    private void Update()
    {
        // 意大利面条式逻辑
        if (isDead)
        {
            // 死亡逻辑
        }
        else if (isHurt)
        {
            // 受伤逻辑
        }
        else if (isAttacking)
        {
            // 攻击逻辑
        }
        else if (isJumping)
        {
            if (Input.GetKey(KeyCode.LeftShift))
            {
                isRunning = true;
                isWalking = false;
            }
            // 跳跃中还能攻击？
            if (Input.GetMouseButtonDown(0))
            {
                isAttacking = true;
                // 但还在跳跃...
            }
        }
        else if (isRunning)
        {
            // ...
        }
        // ... 更多嵌套
    }
}
```

## 状态机的本质

### 核心思想

```
┌─────────────────────────────────────────────────────────────┐
│                     状态机本质                              │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   任意时刻，对象只能处于【唯一确定】的一个状态              │
│                                                             │
│   ┌─────────────────────────────────────────────────────┐  │
│   │                                                     │  │
│   │   状态A ──[条件]──▶ 状态B ──[条件]──▶ 状态C        │  │
│   │     ▲                  │                           │  │
│   │     └──────[条件]──────┘                           │  │
│   │                                                     │  │
│   │   关键约束：                                        │  │
│   │   1. 状态数量有限                                   │  │
│   │   2. 同一时刻只有一个状态                           │  │
│   │   3. 状态转换有明确条件                             │  │
│   │                                                     │  │
│   └─────────────────────────────────────────────────────┘  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 数学本质

```
状态机 = (S, Σ, δ, s0, F)

S  = 有限状态集合 {Idle, Walk, Run, Jump, Attack...}
Σ  = 输入事件集合 {按键, 碰撞, 计时器...}
δ  = 状态转移函数 S × Σ → S
s0 = 初始状态 Idle
F  = 终止状态（可选）{Dead}

示例：
δ(Idle, 按W键) = Walk
δ(Walk, 按Shift) = Run
δ(Run, 松开Shift) = Walk
δ(任意状态, 按Space) = Jump
```

### 具体案例

| 游戏类型 | AI架构 | 理由 |
|----------|--------|------|
| **马里奥** | 状态机 | 敌人行为简单：巡逻→发现→追击 |
| **刺客信条** | 行为树 | NPC有复杂日常行为 |
| **红警** | 规则引擎 | 单位需要自动寻路、攻击、生产决策 |
| **空洞骑士** | 状态机 | Boss行为固定，状态清晰 |
| **求生之路** | 行为树 | 感染者有复杂协作行为 |
| **文明** | 规则引擎 | AI需要长期规划和策略 |

## 状态机实现演进

### Level 1: 枚举 + Switch

```csharp
// 最简单的状态机
public enum State { Idle, Walk, Jump }

public class SimpleFSM : MonoBehaviour
{
    private State currentState;

    private void Update()
    {
        switch (currentState)
        {
            case State.Idle:
                UpdateIdle();
                break;
            case State.Walk:
                UpdateWalk();
                break;
            case State.Jump:
                UpdateJump();
                break;
        }
    }

    private void UpdateIdle()
    {
        if (Input.GetKey(KeyCode.W))
        {
            TransitionTo(State.Walk);
        }
        if (Input.GetKeyDown(KeyCode.Space))
        {
            TransitionTo(State.Jump);
        }
    }

    private void TransitionTo(State newState)
    {
        // 退出当前状态
        OnExit(currentState);

        // 切换状态
        currentState = newState;

        // 进入新状态
        OnEnter(newState);
    }
}
```

**优点**：简单直观
**缺点**：状态多了switch巨大，难以维护

### Level 2: 状态模式

```csharp
// 每个状态是一个类
public interface IState
{
    void Enter();
    void Update();
    void Exit();
}

public class IdleState : IState
{
    private readonly Player player;

    public IdleState(Player player) => this.player = player;

    public void Enter()
    {
        player.Animator.Play("Idle");
    }

    public void Update()
    {
        if (Input.GetKey(KeyCode.W))
        {
            player.StateMachine.TransitionTo(player.WalkState);
        }
    }

    public void Exit() { }
}

public class StateMachine
{
    private IState currentState;

    public void TransitionTo(IState newState)
    {
        currentState?.Exit();
        currentState = newState;
        currentState.Enter();
    }

    public void Update()
    {
        currentState?.Update();
    }
}
```

**优点**：状态逻辑隔离，易扩展
**缺点**：类数量多，状态转换分散

### Level 3: 分层状态机 (HFSM)

```
问题：Walk和Run都有"移动中"的共同逻辑

解决方案：状态嵌套

            ┌─────────────────────────────────┐
            │          Move (父状态)           │
            │  ┌───────────┐  ┌───────────┐   │
            │  │   Walk    │  │    Run    │   │
            │  └───────────┘  └───────────┘   │
            │                                 │
            │  共同逻辑：                      │
            │  • 面向移动方向                  │
            │  • 播放脚步声                    │
            │  • 检测碰撞                      │
            └─────────────────────────────────┘
```

```csharp
public class HierarchicalState : IState
{
    private readonly IState parentState;
    private readonly List<IState> subStates;

    public void Update()
    {
        // 先执行父状态逻辑
        parentState?.Update();

        // 再执行当前子状态逻辑
        currentSubState?.Update();
    }
}
```

**优点**：减少重复代码，逻辑更清晰
**缺点**：实现复杂

### Level 4: 并行状态机

```
问题：角色可以"边跑边攻击"

解决方案：多状态机并行

┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 移动状态机   │     │ 动作状态机   │     │ 装备状态机   │

                    Run + Attack + Sword
```

## 状态机常见陷阱

### 陷阱1：忘记Exit/Enter处理

```csharp
// ❌ 直接改状态，不处理转换
currentState = State.Jump;

// ✅ 通过转换函数
TransitionTo(State.Jump);

private void TransitionTo(State newState)
{
    OnExit(currentState);  // 清理旧状态
    currentState = newState;
    OnEnter(newState);     // 初始化新状态
}
```

### 陷阱2：在状态内直接跳转

```csharp
// ❌ 状态类直接引用其他状态
public class IdleState : IState
{
    public void Update()
    {
        if (condition)
        {
            player.StateMachine.currentState = new WalkState();  // 耦合！
        }
    }
}

// ✅ 通过转换请求解耦
public class IdleState : IState
{
    public void Update()
    {
        if (condition)
        {
            stateMachine.RequestTransition("Walk");  // 解耦
        }
    }
}
```

### 陷阱3：状态爆炸

```
问题：每个技能一个状态

Attack1 → Attack2 → Attack3 → SpecialAttack → UltimateAttack
    ↓
10个攻击状态 + 10×9=90个转换 = 灾难

解决方案：
1. 使用通用"Attack"状态 + 参数
2. 使用动画状态机（Animator）处理连招
3. 考虑行为树
```
