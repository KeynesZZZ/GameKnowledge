# 自动化构建

> Unity自动化构建流程配置

## 文档定位

讲解Unity自动化构建流程的配置方法，包括命令行构建脚本、BuildPipeline API使用、CI/CD集成（GitHub Actions示例）等，确保构建过程的一致性和可重复性。

---

## 概述

自动化构建可以确保构建过程的一致性和可重复性。

## 命令行构建

```bash
# Windows
Unity.exe -quit -batchmode -projectPath "D:\Project" -buildWindowsPlayer "D:\Build\Game.exe"

# Android
Unity.exe -quit -batchmode -projectPath "D:\Project" -executeMethod BuildScript.BuildAndroid
```

## 构建脚本

```csharp
public class BuildScript
{
    [MenuItem("Build/Android")]
    public static void BuildAndroid()
    {
        var options = new BuildPlayerOptions
        {
            scenes = new[] { "Assets/Scenes/Main.unity" },
            locationPathName = "Builds/Android/Game.apk",
            target = BuildTarget.Android,
            options = BuildOptions.None
        };

        BuildPipeline.BuildPlayer(options);
    }
}
```

## CI/CD集成

```yaml
# GitHub Actions示例
name: Build
on: [push]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: game-ci/unity-builder@v2
        with:
          targetPlatform: Android
```
