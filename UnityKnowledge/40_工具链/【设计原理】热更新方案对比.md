# 热更新方案对比

> Unity热更新方案对比分析

## 文档定位

从底层机制角度对比分析Unity主流热更新方案（HybridCLR、ILRuntime、xLua等）的原理、优劣势和适用场景，帮助团队根据项目需求做出合理的技术选型。

---

## 概述

热更新允许在不重新发布应用的情况下更新游戏内容。

## 方案对比

| 方案 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| **AssetBundle** | 资源打包 | 官方支持 | 需要版本管理 |
| **Addressables** | AB封装 | 简化流程 | 学习成本 |
| **Lua** | 脚本热更 | 灵活性高 | 性能损耗 |
| **ILRuntime** | C#解释执行 | 原生C# | 调试困难 |
| **HybridCLR** | 原生AOT补充 | 接近原生 | 侵入性强 |

## Addressables热更流程

```csharp
// 检查更新
public async Task CheckUpdate()
{
    var handle = Addressables.CheckForCatalogUpdates();
    await handle.Task;

    if (handle.Result.Count > 0)
    {
        var updateHandle = Addressables.UpdateCatalogs(handle.Result);
        await updateHandle.Task;
    }

    Addressables.Release(handle);
}
```
