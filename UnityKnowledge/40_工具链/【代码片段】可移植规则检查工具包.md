## 🚀 使用方法

### 方式1: 复制到Unity项目

```bash
# 1. 复制整个工具包目录
cp -r unity-rules-checker /path/to/YourUnityProject/

# 2. 进入工具包目录
cd /path/to/YourUnityProject/unity-rules-checker

# 3. 运行安装脚本
./install.sh          # Linux/Mac
# 或
powershell -File install.ps1    # Windows

# 4. 删除工具包目录（可选）
cd ..
rm -rf unity-rules-checker
```

### 方式2: 打包分发

```bash
# 创建压缩包
tar -czf unity-rules-checker-v1.0.0.tar.gz unity-rules-checker/
# 或
zip -r unity-rules-checker-v1.0.0.zip unity-rules-checker/

# 分发给团队成员或上传到服务器
```

### 方式3: Git子模块（推荐多项目使用）

```bash
# 在Unity项目中添加为子模块
git submodule add https://github.com/your-username/unity-rules-checker.git unity-rules-checker

# 初始化子模块
git submodule update --init --recursive

# 运行安装
cd unity-rules-checker
./install.sh
```

## 🔧 工作原理

### 安装脚本做什么？

1. **检测Unity项目** - 确认在正确的目录
2. **备份现有配置** - 避免覆盖用户文件
3. **复制文件** - 安装SKILL、Hook、文档
4. **设置权限** - Git Hook需要可执行权限
5. **验证安装** - 确认所有文件就位

### SKILL如何工作？

1. **查找规则清单** - 按优先级搜索：
   - `./docs/开发规则清单.md`
   - `unity-rules-checker/docs/开发规则清单.md`
   - 用户全局Knowledge目录

2. **解析规则** - 提取所有60条规则

3. **检查代码** - 根据规则分析C#代码

4. **生成报告** - 显示违规和修复建议

## 🛠️ 高级用法

### 自定义规则

编辑 `docs/开发规则清单.md`，添加项目特定规则。

### 自定义Hook配置

编辑 `.claude/hooks/config.json`：

```json
{
  "block_on": "HIGH",           // 改为HIGH级别
  "exclude_patterns": [
    "**/ThirdParty/**/*.cs"     // 添加排除目录
  ]
}
```

### 团队共享

1. 将工具包放入团队共享仓库
2. 在项目README中说明安装步骤
3. 新成员clone项目后运行安装脚本
