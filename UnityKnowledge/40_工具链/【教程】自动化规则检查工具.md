# 自动化规则检查工具

> Unity开发规则的自动化检查体系 - Claude Code SKILL + Python工具 + CI/CD集成

## 文档定位

本文档从**使用角度**讲解自动化规则检查工具。

**相关文档**：[[【教程】自动化规则检查工具]]

---

## 概述

本工具集提供三层自动化检查能力，确保Unity代码符合60条开发规则：

```
Layer 1: Claude Code SKILL
  ├─ 快速启动、立即可用
  ├─ 覆盖所有60条规则
  └─ 智能修复建议

Layer 2: Python脚本工具
  ├─ CI/CD集成
  ├─ 批量检查
  └─ HTML/JSON报告

Layer 3: Roslyn Analyzer (规划中)
  ├─ IDE实时提示
  └─ 编译时检查
```

## 快速开始

### 1. Claude Code SKILL（立即可用）

#### 检查单个文件

在Claude Code中运行：

```bash
/check-rules Assets/Scripts/PlayerController.cs
```

#### 检查整个目录

```bash
/check-rules Assets/Scripts
```

#### 只检查特定规则类别

```bash
/check-rules Assets/Scripts --rules=GC,MEMORY
```

可用类别：
- `GC` - GC优化规则
- `MEMORY` - 内存管理规则
- `POOL` - 对象池规则
- `ARCH` - 架构规则
- `UI` - UI优化规则
- `PHYSICS` - 物理系统规则
- `RES` - 资源管理规则
- `PERF` - 编译期优化规则
- `SAFE` - 代码安全规则

#### 只检查CRITICAL规则

```bash
/check-rules Assets/Scripts --severity=CRITICAL
```

### 2. Git Hook集成（自动检查）

配置后，每次提交前自动检查：

```bash
git add .
git commit -m "feat: add player controller"
# → 自动触发规则检查
# → CRITICAL违规会阻止提交
```

**详细配置**：见 [Git Hook配置](#git-hook配置)

### 3. Python工具（CI/CD）

```bash
# 安装依赖
pip install -r tools/unity_rules_checker/requirements.txt

# 检查项目
python tools/unity_rules_checker/rules_checker.py analyze Assets/Scripts

# 生成HTML报告
python tools/unity_rules_checker/rules_checker.py analyze Assets/Scripts --report report.html
```

**详细使用**：见 [Python工具文档](#python工具文档)

## Git Hook配置

### 自动安装Hook

```bash
# 复制Hook脚本到.git/hooks
cp .claude/hooks/pre-commit.md .git/hooks/pre-commit

# 设置可执行权限（Linux/Mac）
chmod +x .git/hooks/pre-commit
```

### Hook配置文件

创建 `.claude/hooks/config.json`：

```json
{
  "block_on": "CRITICAL",
  "warn_on": "HIGH",
  "check_patterns": ["Assets/**/*.cs"],
  "exclude_patterns": [
    "**/Generated/**/*.cs",
    "**/Temp/**/*.cs",
    "**/Plugins/**/*.cs"
  ]
}
```

### 跳过Hook检查

```bash
git commit --no-verify -m "提交信息"
```

## Python工具文档

### 安装

```bash
cd tools/unity_rules_checker
pip install -r requirements.txt
```

### 基本使用

```bash
# 检查单个文件
python rules_checker.py analyze Assets/Scripts/PlayerController.cs

# 检查整个项目（递归）
python rules_checker.py analyze Assets/Scripts --recursive

# 只检查特定规则
python rules_checker.py analyze Assets/Scripts --rules GC-001,GC-002,MEM-001

# 只检查特定严重性
python rules_checker.py analyze Assets/Scripts --severity CRITICAL

# 发现违规时退出码为1（CI/CD使用）
python rules_checker.py analyze Assets/Scripts --severity CRITICAL --fail-on-error
```

### 生成报告

```bash
# 生成HTML报告
python rules_checker.py analyze Assets/Scripts --report report.html

# 生成JSON报告
python rules_checker.py analyze Assets/Scripts --report report.json

# 控制台+HTML报告
python rules_checker.py analyze Assets/Scripts --report report.html --console
```

### CI/CD集成

#### GitHub Actions

```yaml
name: Code Quality Check

on: [push, pull_request]

jobs:
  rules-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v2

      - name: Setup Python
        uses: actions/setup-python@v2
        with:
          python-version: '3.9'

      - name: Install dependencies
        run: pip install -r tools/unity_rules_checker/requirements.txt

      - name: Run Rules Checker
        run: python tools/unity_rules_checker/rules_checker.py analyze Assets/Scripts --severity CRITICAL --fail-on-error

      - name: Upload Report
        uses: actions/upload-artifact@v2
        if: always()
        with:
          name: rules-check-report
          path: report.html
```

#### GitLab CI

```yaml
code_quality:
  stage: test
  script:
    - pip install -r tools/unity_rules_checker/requirements.txt
    - python tools/unity_rules_checker/rules_checker.py analyze Assets/Scripts --severity CRITICAL --fail-on-error
  artifacts:
    paths:
      - report.html
    when: always
```

## 工作流示例

### 日常开发流程

```
1. 编写代码
   ↓
2. 本地检查（Claude Code）
   /check-rules Assets/Scripts/NewScript.cs
   ↓
3. 查看违规和建议
   ↓
4. 修复代码
   ↓
5. 提交代码
   git add .
   git commit -m "feat: add new feature"
   ↓
6. Git Hook自动检查
   → CRITICAL违规阻止提交
   → 修复后重新提交
   ↓
7. 推送到远程
   git push
   ↓
8. CI/CD运行Python工具
   → 生成团队报告
```

## 常见问题

### Q1: Hook检查太慢怎么办？

**A**: 只检查暂存的文件，通常 < 5秒。如果仍然慢，可以：

1. 调整配置，只检查CRITICAL规则
2. 排除自动生成的文件
3. 使用 `--no-verify` 跳过（不推荐）

### Q2: 某些规则不适用于项目怎么办？

**A**: 三种方式处理：

1. **配置文件**：在 `config.json` 中排除规则
2. **规则例外**：开发规则清单中已标注例外情况
3. **团队讨论**：提交规则变更建议

### Q3: Python工具检测不到某些违规？

**A**: Python工具基于模式匹配，复杂规则（如架构设计）建议使用Claude Code SKILL。

### Q4: 如何添加自定义规则？

**A**: 三种方式：

1. **修改开发规则清单** - 添加到 `开发规则清单.md`
2. **Python工具** - 在 `rule_matchers/` 中添加自定义匹配器
3. **SKILL扩展** - 在 `check-rules.md` 中添加检查逻辑
