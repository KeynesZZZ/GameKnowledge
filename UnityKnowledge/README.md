## 设计原则

### 编号规则

| 前缀 | 分类 | 说明 |
|------|------|------|
| 00 | 元数据 | 模板、配置、规范 |
| 10-30 | 技术知识 | 按技术领域分类 |
| 40-60 | 工具与库 | 开发工具、平台、第三方库 |
| 90 | 经验总结 | 项目复盘、踩坑记录 |

### 文档规范

1. **使用标签** - 每篇文档至少2个标签（参考[标签体系](00_元数据与模板/标签体系.md)）
2. **代码可运行** - 代码示例必须完整可运行
3. **版本标注** - 注明适用的Unity版本
4. **关联链接** - 添加相关文档的交叉引用

## 与学习目录的区别

| 维度 | 学习/ | UnityKnowledge/ |
|------|-------|-----------------|
| **目的** | 系统学习 | 快速检索 |
| **组织** | 按学习路径 | 按技术领域 |
| **内容** | 教程为主 | 代码片段+最佳实践 |
| **适用** | 初学者 | 有经验开发者 |

## 快速开始

### 查找知识

1. 按目录导航
2. 使用知识库搜索（ChromaDB）
3. 按标签筛选

### 贡献知识

1. 使用对应模板创建文档
2. 添加适当标签
3. 更新相关README
