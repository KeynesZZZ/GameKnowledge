# 【笔记】大规模技能特效方案

> 承 [[【笔记】大规模单位战斗结算]]。单位会动、会打、会死之后，最后一个视觉问题是「打中了要看到效果」。本文讲大规模战斗中技能特效的分层策略与 ECS 事件驱动方案。

## 核心矛盾

| 矛盾 | 棘手之处 |
|------|---------|
| 10w 单位同时交战，命中/死亡事件密集 | 同帧数百~数千个特效触发，ParticleSystem 主线程爆炸 |
| 特效类型多（刀光/命中/爆炸/拖尾/AOE 圈） | 每种特效参数不同，不能用单一粒子系统覆盖 |
| 远景看不清细节 | 10w 单位同屏时大量像素不足 1 个，高精度粒子浪费 |
| 特效生命周期短但创建/销毁频繁 | Instantiate/Destroy 的 GC 和结构性开销 |

**核心思路：分层渲染 + GPU 事件驱动 + ECS 事件桥接 + 池化回收。**

> 本文聚焦 ECS 架构层面如何驱动 VFX Graph。VFX Graph 编辑器内部的效果制作（命中/刀光/爆炸/区域/护盾的具体 Block 配置）见 [[【笔记】VFX Graph效果制作]]。

---

## 一、为什么传统 ParticleSystem 不行

| 方式 | 问题 |
|------|------|
| `ParticleSystem.Play()`（GameObject 体系） | 每个特效一个 GO，数百个 `Play()` 调用 = 主线程瓶颈 |
| `Instantiate` 特效预制体 | 频繁创建销毁 → GC Spike + Entity 结构性变更 |
| `ParticleSystem.Emit(1)` 批量发射 | 仍受限于 CPU 粒子模拟，无法 GPU 驱动 |
| 每单位挂 ParticleSystem | 10w 个 ParticleSystem 组件 = 内存爆炸 + 模拟开销 |

**正解：VFX Graph（GPU 粒子）+ ECS 事件桥接 + 分层 LOD 策略。**

## 二、VFX Graph：GPU 驱动的粒子系统

### 2.1 为什么选 VFX Graph

| 特性 | ParticleSystem | VFX Graph |
|------|---------------|-----------|
| 粒子模拟 | CPU 逐帧 | **GPU Compute Shader** |
| 粒子上限 | ~数万 | **百万级** |
| 多实例 | 每实例独立模拟 | 一个 VFX 控制所有粒子 |
| ECS 兼容 | 差（GameObject 体系） | 好（通过 GraphicsBuffer 桥接） |
| Unity 版本 | 内置 | **Unity 2022.3+ / Unity 6 推荐** |

### 2.2 VFX Graph 事件驱动模型

VFX Graph 支持 **GPU Event**——由代码触发事件，VFX 在 GPU 端自主完成粒子生命周期：

```
C# / ECS Job
    │
    │ 1. 写事件到 GraphicsBuffer（位置/类型/颜色/大小）
    ▼
VFX Graph
    │
    │ 2. GPU Event 接收事件 → Spawn 粒子
    │ 3. GPU 自主模拟（速度/重力/碰撞/衰减）
    │ 4. GPU 渲染（无需回读 CPU）
    ▼
屏幕上的粒子
```

**关键**：CPU 只负责「在哪里、什么时候、什么类型」的事件写入，粒子模拟全在 GPU。

### 2.3 VFX + ECS 桥接代码

```csharp
using Unity.Burst;
using Unity.Collections;
using Unity.Entities;
using Unity.Mathematics;
using Unity.Rendering;
using UnityEngine.VFX;

/// <summary>特效事件数据——由战斗系统产出，VFX 桥接系统消费。</summary>
public struct VfxEvent : IBufferElementData
{
    public float3 Position;
    public quaternion Rotation;
    public int EffectType;    // 0=命中 1=刀光 2=爆炸 3=AOE圈
    public float Scale;
    public float Intensity;   // 伤害值映射到特效强度
}

/// <summary>管理 VFX Graph 的 singleton，持有 GraphicsBuffer 引用。</summary>
public partial class VfxManagerSystem : SystemBase
{
    private VisualEffect m_HitVfx;
    private VisualEffect m_SlashVfx;
    private VisualEffect m_ExplosionVfx;

    // VFX Graph 中对应的 Event 属性名
    const string s_EffectCount = "EventCount";
    const string s_Positions   = "Positions";
    const string s_Scales      = "Scales";
    const string s_Intensities = "Intensities";

    protected override void OnCreate()
    {
        // VFX Graph 通过 GraphicsBuffer 接收批量事件
        // 每个 VisualEffect 实例负责一类特效
    }

    protected override void OnUpdate()
    {
        // 收集所有 VfxEvent → 按 EffectType 分组 → 写入对应 VFX 的 GraphicsBuffer
        // VFX Graph 内部 GPU Event 自动处理粒子的生成、模拟、渲染
    }
}
```

> **Unity 6 增强**：VFX Graph 在 Unity 6 中可直接绑定 `GraphicsBuffer`，无需逐事件 `SendEvent`，一次写入批量事件数据即可。

## 三、ECS 事件驱动：从战斗结算到特效触发

### 3.1 事件管线

特效系统是 [[【笔记】大规模单位战斗结算]] 的事件消费者：

```
CombatTriggerSystem                      CombatResolveSystem
  (碰撞检测)                                (扣血 + 死亡判定)
      │                                         │
      ▼                                         ▼
  DamageEvent buffer                    AnimState.Death 标记
      │                                         │
      └──────────┬──────────────────────────────┘
                 ▼
         VfxEventSystem（新增）
           · 命中事件 → VfxEvent(EffectType=0)
           · 死亡事件 → VfxEvent(EffectType=2)
                 │
                 ▼
         VfxManagerSystem
           · 按类型分组
           · 写入 GraphicsBuffer
           · VFX Graph GPU 播放
```

### 3.2 VfxEventSystem：事件收集

```csharp
using Unity.Burst;
using Unity.Entities;
using Unity.Collections;

/// <summary>监听伤害事件和死亡事件，产出 VfxEvent 写入全局特效队列。</summary>
[BurstCompile]
[UpdateInGroup(typeof(SimulationSystemGroup))]
[UpdateAfter(typeof(CombatResolveSystem))]  // 必须在伤害结算之后
public partial struct VfxEventSystem : ISystem
{
    [BurstCompile]
    public void OnUpdate(ref SystemState state)
    {
        // 方案 A：扫描有 DamageEvent 的单位（命中特效）
        new HitVfxJob { }.ScheduleParallel();

        // 方案 B：扫描刚进入 Death 状态的单位（死亡特效）
        // 用 EntityCommandBuffer 记录已处理标记，避免重复触发
        var ecb = new EntityCommandBuffer(Allocator.TempJob);
        new DeathVfxJob { Ecb = ecb.AsParallelWriter() }.ScheduleParallel();
        state.Dependency.Complete();
        ecb.Playback(state.EntityManager);
        ecb.Dispose();
    }

    /// <summary>命中特效：单位收到伤害 → 在位置生成命中闪光。</summary>
    [BurstCompile]
    partial struct HitVfxJob : IJobEntity
    {
        void Execute(Entity e, in LocalTransform transform,
                     in Health hp, DynamicBuffer<DamageEvent> damageEvents,
                     in VfxTag vfxTag)
        {
            if (damageEvents.Length == 0) return;
            // 累积伤害值 → 映射到特效强度
            float totalDamage = 0f;
            foreach (var dmg in damageEvents) totalDamage += dmg.Amount;

            // 追加到单位的 VfxEvent buffer（由 VfxManagerSystem 统一收集）
            // 注意：这里写 DynamicBuffer 是安全的（IJobEntity 逐实体独立）
        }
    }

    /// <summary>死亡特效：单位刚死亡 → 在位置生成爆裂特效。</summary>
    [BurstCompile]
    partial struct DeathVfxJob : IJobEntity
    {
        public EntityCommandBuffer.ParallelWriter Ecb;
        void Execute(Entity e, [EntityIndexInQuery] int sortKey,
                     in LocalTransform transform, in Health hp, in UnitAnim anim)
        {
            if (hp.Current > 0f || anim.State != AnimState.Death) return;
            // 检查是否已生成过死亡特效（用标记组件避免重复）
            // 首次死亡 → 追加 VfxEvent(EffectType=2, 爆炸) + 添加标记
        }
    }
}

/// <summary>标记该单位需要播特效（enableable，死亡时自动禁用）。</summary>
public struct VfxTag : IComponentData, IEnableableComponent { }
/// <summary>标记已生成死亡特效（防止重复）。</summary>
public struct DeathVfxSpawned : IComponentData, IEnableableComponent { }
```

> 与 [[【笔记】大规模单位战斗结算]] 的 `DamageEvent` buffer 复用——伤害事件天然是特效事件的信号源。

## 四、分层特效策略（LOD）

10w 单位同屏时，不可能也没必要给每个单位播高精度特效。按距离/屏幕占比分三层：

### 4.1 三层策略

```
┌──────────────────────────────────────────────────────────────┐
│                    摄像机                                     │
│                                                               │
│  ┌─────────┐   ┌──────────────┐   ┌────────────────────────┐ │
│  │ 近景层   │   │ 中景层        │   │ 远景层                  │ │
│  │ < 30m   │   │ 30m ~ 80m    │   │ > 80m                  │ │
│  │         │   │              │   │                        │ │
│  │VFX Graph│   │VFX Graph     │   │纯顶点动画 /            │ │
│  │高精度    │   │低精度（减粒子）│   │Shader 特效（无粒子）    │ │
│  │         │   │              │   │                        │ │
│  │50粒子/命中│   │10粒子/命中    │   │1次全屏闪/一组单位       │ │
│  └─────────┘   └──────────────┘   └────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

| 层 | 距离 | 特效方式 | 粒子数/命中 | 说明 |
|----|------|---------|-----------|------|
| 近景 | < 30m | VFX Graph 高精度 | 50~100 | 可见刀光、命中迸溅、拖尾 |
| 中景 | 30~80m | VFX Graph 低精度 | 10~20 | 简化粒子，去掉拖尾 |
| 远景 | > 80m | Shader 闪屏/色彩偏移 | 0 | 无粒子，纯屏幕后效或顶点色脉冲 |

### 4.2 远景替代方案（无粒子）

远景单位完全不用粒子，改用**顶点级特效**——成本极低：

```csharp
// 远景命中：单位顶点色脉冲（Shader 端处理，CPU 零开销）
// ECS 侧只设置一个 float 字段 → Material 属性 → Shader 脉冲
public struct HitFlash : IComponentData
{
    public float Timer;  // > 0 表示正在闪，每帧递减
}

// Shader 端（URP Shader Graph 或 HLSL）：
// float flash = saturate(_HitFlashTimer);
// finalColor = lerp(baseColor, float3(1,1,1), flash * 0.8);
```

**优势**：10w 个远景命中 = 10w 个 float 写入（Burst 并行），零粒子模拟开销。

### 4.3 分层调度 Job

```csharp
[BurstCompile]
partial struct VfxLodJob : IJobEntity
{
    public float3 CameraPos;
    public void Execute(Entity e, in LocalTransform transform,
                        ref VfxEvent vfxEvent, in LodDistance lod)
    {
        float dist = math.distance(transform.Position, CameraPos);

        if (dist > 80f)
        {
            // 远景：改用 HitFlash，不发 VFX 事件
            vfxEvent.EffectType = -1;  // 标记跳过粒子
            // 由 HitFlashSystem 处理（写 HitFlash.Timer）
        }
        else if (dist > 30f)
        {
            // 中景：降级特效类型（命中 → 简化命中）
            vfxEvent.Scale *= 0.5f;    // 缩小尺寸
            vfxEvent.Intensity *= 0.5f;
        }
        // 近景：保持原样
    }
}
```

## 五、特效池化与回收

### 5.1 为什么不能用 Instantiate/Destroy

| 操作 | 10w 战场的代价 |
|------|--------------|
| `EntityManager.CreateEntity`（特效实体） | 每帧数百次结构性变更 → archetype 重排 |
| `VisualEffect` 实例创建 | GPU 资源分配 → 帧卡顿 |
| `DestroyEntity` | 同上，且 GC 压力 |

### 5.2 VFX Graph 天然池化

VFX Graph 的优势在于**一个 VisualEffect 实例处理所有同类粒子**——不需要池化实例：

```
❌ 传统：每个命中 → Instantiate(命中特效预制体) → 100个 GO
✅ VFX：一个 VisualEffect（命中特效）→ GPU Event 批量发射 → 100组粒子在一个系统中
```

VFX Graph 内部的粒子池由 GPU 管理，CPU 只需把"在哪里、多大量级"写进 GraphicsBuffer。

### 5.3 ECS 侧 VfxEvent Buffer 池化

```csharp
/// <summary>全局特效事件队列——预分配，永不扩容。</summary>
public struct VfxEventQueue : IComponentData
{
    public NativeList<VfxEvent> Events;  // 预分配 Capacity = 4096
}

// 每帧流程：
// 1. OnUpdate 开头 Clear()（NativeList Clear 不释放内存）
// 2. 各系统 Append 事件
// 3. VfxManagerSystem 一次性消费 → 写 GraphicsBuffer
// 4. VFX Graph GPU 播放
```

## 六、AOE / 范围技能特效

### 6.1 大范围技能（屏幕级）

AOE 技能（如陨石雨、冰风暴）的特效**不需要按单位触发**，而是**按技能释放触发**：

```csharp
// 技能释放时 → 一个全局 VfxEvent(EffectType=3, 位置=技能中心, 范围=radius)
// VFX Graph 播放大范围 AOE 特效（地面圈 + 爆炸 + 烟雾）
// 伤害由 CombatResolveSystem 的 AOESystem 独立计算（见战斗结算笔记第五节）
```

### 6.2 区域填充型特效（毒雾/火墙）

持续型区域特效用 VFX Graph 的 **GPU 碰撞 + 区域填充**：

```
VFX Graph 设置：
  · Spawn: 按面积密度发射粒子
  · Collision: 与地面/高度场碰撞
  · Lifetime: 匹配技能持续时间
  · 颜色/大小: 按技能类型配置

ECS 侧：
  · 只维护"活跃区域列表"（中心 + 半径 + 剩余时间）
  · 区域消失时通知 VFX 停止 Spawn
```

## 七、刀光/拖尾特效

刀光和拖尾是实时战斗中最频繁的特效类型。

### 7.1 方案选型

| 方案 | 适用 | 10w 评价 |
|------|------|---------|
| TrailRenderer（GO 体系） | 主角/Boss | ❌ 10w 个 TrailRenderer = 渲染批次爆炸 |
| VFX Graph Trail（GPU 拖尾） | 大规模 | ✅ GPU 生成，CPU 只传起点/终点 |
| Mesh Ribbon（自定义条带） | 中等规模 | ⚠️ 需自建 mesh 更新 |
| VAT 拖尾（烘焙进顶点动画） | 固定动作 | ✅ 零运行时成本（配合 [[【笔记】大规模单位动画方案]]） |

### 7.2 大规模刀光：VFX Graph GPU 拖尾

```csharp
// 近战攻击时 → 写入刀光事件（起点 + 朝向 + 武器长度）
// VFX Graph 的 Trail 模块自动生成 GPU 条带

public struct SlashVfxEvent : IBufferElementData
{
    public float3 StartPos;     // 武器起点
    public float3 Direction;    // 挥砍方向
    public float  WeaponLength; // 武器长度
    public float  Intensity;    // 攻击力映射
}
```

VFX Graph 侧配置：
```
Trail Output:
  · Position: 从 StartPos 沿 Direction 弧线
  · Ribbon Width: WeaponLength * 0.3
  · Color: 根据 Intensity 热力图（弱=蓝/强=红白）
  · Lifetime: 0.3s（快闪）
```

### 7.3 远景替代：刀光 = VAT 动画的一部分

远景单位的刀光直接烘焙进 VAT 纹理（攻击动作自带刀光形变），无需运行时粒子。呼应 [[【笔记】大规模单位动画方案]] 的 VAT 方案。

## 八、系统时序

```
FixedStepSimulationSystemGroup
  ├─ StepPhysicsWorld
  ├─ CombatTriggerSystem          (碰撞 → 伤害事件)
  └─ CombatResolveSystem          (扣血 + 死亡判定)

SimulationSystemGroup
  ├─ VfxEventSystem               (消费 DamageEvent/Death → 写 VfxEvent)   ← 新增
  ├─ VfxLodSystem                 (按距离分层，降级/跳过远景特效)            ← 新增
  ├─ VfxManagerSystem             (收集 VfxEvent → 写 GraphicsBuffer → VFX)  ← 新增
  ├─ DeathRecycleSystem           (死亡动画到时间 → 回收)
  ├─ UnitAnimSystem               (动画状态机)
  └─ UnitMovementSystem / SteeringSystem
```

- **VfxEventSystem** 必须在 `CombatResolveSystem` 之后（伤害/死亡已结算）
- **VfxLodSystem** 在事件产出后、VFX 播放前做 LOD 筛选
- **VfxManagerSystem** 最后统一写入 GraphicsBuffer，一次性提交

## 九、性能预算参考

以 10w 单位、1% 同时交战（1000 次/帧命中）为基准：

| 环节 | CPU 耗时 | GPU 耗时 | 备注 |
|------|---------|---------|------|
| VfxEventSystem（事件收集） | 0.1ms | - | Burst 并行扫描 buffer |
| VfxLodSystem（LOD 筛选） | 0.05ms | - | 逐实体距离判定 |
| VfxManagerSystem（写 GraphicsBuffer） | 0.1ms | - | 一次 `SetData` |
| VFX Graph 粒子模拟 | - | 0.3~0.8ms | GPU Compute |
| VFX Graph 粒子渲染 | - | 0.2~0.5ms | 透明物合批 |
| 远景 HitFlash（10w） | 0.1ms | - | Burst 写 float |
| **总计** | **~0.35ms** | **~0.5~1.3ms** | 远优于 ParticleSystem |

对比：传统 ParticleSystem 1000 次 `Play()` ≈ 5~15ms（主线程）。

## 十、避坑

| 现象 | 根因 / 处置 |
|------|------------|
| VFX Graph 事件不触发 | `GraphicsBuffer` 格式与 VFX Graph 的 `GraphicsBuffer` 节点类型不匹配 → 检查 stride/count/格式 |
| 远景单位看不到命中反馈 | VFX 粒子太小/太短 → 远景改用 HitFlash（顶点色脉冲），不用粒子 |
| GPU 粒子数爆炸（爆显存） | VFX Graph 的容量（Capacity）设太高 → 按同屏最大粒子数设上限，超出用事件丢弃 |
| `VisualEffect` 组件不在 Entity 上 | VFX 是 GameObject 体系，不能直接挂 Entity → 用 singleton 管理 VisualEffect 引用 |
| 死亡特效重复播放 | 死亡状态跨帧，每帧都触发 → 加 `DeathVfxSpawned` 标记组件，触发后启用 |
| AOE 特效与伤害不同步 | 特效和伤害在不同系统 → 确保 VfxEventSystem 和 AOESystem 读同一份技能释放数据 |
| 刀光抖动/断裂 | Trail 采样率不足或位置更新不同步 → VFX Trail 用 GPU 端插值，不要逐帧 CPU 写位置 |
| 移动端 GPU 粒子性能差 | 移动端 GPU Compute 限制 → 移动端降级为简单 Sprite + VAT 拖尾，不用 VFX Graph |

## 速查清单

- [ ] 用 VFX Graph（GPU 粒子）替代 ParticleSystem，一个 VFX 实例处理所有同类特效
- [ ] ECS 桥接：战斗结算 → `VfxEvent` buffer → VFX GraphicsBuffer → GPU 播放
- [ ] 分层 LOD：近景 VFX 高精度 / 中景低精度 / 远景 HitFlash 无粒子
- [ ] 特效事件池化：预分配 `NativeList<VfxEvent>`，Clear 不释放内存
- [ ] AOE 特效按技能释放触发（不按单位），与 AOE 伤害共用技能数据
