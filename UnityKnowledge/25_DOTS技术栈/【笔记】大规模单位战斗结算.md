# 【笔记】大规模单位战斗结算

> 承 [[【笔记】大规模单位AI决策与寻路]]。单位会动、会避障之后，最后一个问题是「打中谁、扣多少血、怎么死」。本文讲大规模战斗结算的 DOTS 事件化方案。

## 一、为什么 OnTriggerEnter / SendMessage 不行

| 方式 | 问题 |
|------|------|
| `OnTriggerEnter`（MonoBehaviour） | 10w 单位 = 10w 个回调，主线程爆炸；无法 Burst |
| `SendMessage`/`UnityEvent` | 字符串/委托派发，GC + 反射，性能不可控 |
| 直接在碰撞 job 里扣血 | 写竞争（同一帧被多源命中）、与渲染/动画系统时序耦合 |

**正解：碰撞只「产出事件」，伤害由独立系统「消费事件」应用**——解耦 + 无竞争 + 可 Burst。

## 二、碰撞检测：两条路线

### 路线 A：Unity.Physics（`com.unity.physics`，推荐）

DOTS 原生物理，broadphase + trigger events 全 Burst/Job 化，适合海量触发器。

```csharp
using Unity.Burst;
using Unity.Collections;
using Unity.Entities;
using Unity.Physics;
using Unity.Physics.Systems;

public struct ProjectileTag : IComponentData { public float Damage; }
public struct DamageEvent : IBufferElementData { public float Amount; public Entity Source; }

/// <summary>收集投射物 ↔ 单位的触发事件到并发安全的队列。</summary>
[UpdateInGroup(typeof(FixedStepSimulationSystemGroup))]
[UpdateAfter(typeof(StepPhysicsWorld))]
[BurstCompile]
public partial struct CombatTriggerSystem : ISystem
{
    private NativeQueue<HitEvent> m_Hits;

    struct HitEvent { public Entity Target; public Entity Source; public float Damage; }

    [BurstCompile] public void OnCreate(ref SystemState state)
    {
        m_Hits = new NativeQueue<HitEvent>(Allocator.Persistent);
        state.RequireForUpdate<ProjectileTag>();
    }
    [BurstCompile] public void OnDestroy(ref SystemState state) => m_Hits.Dispose();

    [BurstCompile] public void OnUpdate(ref SystemState state)
    {
        m_Hits.Clear();
        var sim = SystemAPI.GetSingleton<SimulationSingleton>();
        state.Dependency = new TriggerJob {
            Hits = m_Hits.AsParallelWriter(),
            UnitLookup = state.GetComponentLookup<Health>(isReadOnly: true),
            ProjLookup = state.GetComponentLookup<ProjectileTag>(isReadOnly: true)
        }.Schedule(sim, state.Dependency);

        // 主线程回放：并行队列 → 逐事件 AppendToBuffer（无竞争）
        state.Dependency.Complete();
        while (m_Hits.TryDequeue(out var hit))
        {
            // ECB 或直接 buffer.Add；用 SystemAPI 的 ECB 更安全（见下 ResolveSystem）
            // 这里用 buffer lookup 直接加（单线程，安全）
        }
    }

    [BurstCompile]
    struct TriggerJob : ITriggerEventsJob
    {
        public NativeQueue<HitEvent>.ParallelWriter Hits;
        [ReadOnly] public ComponentLookup<Health> UnitLookup;
        [ReadOnly] public ComponentLookup<ProjectileTag> ProjLookup;
        public void Execute(TriggerEvent ev)
        {
            // 双向判定：谁是投射物谁是单位
            Resolve(ev.EntityA, ev.EntityB);
            Resolve(ev.EntityB, ev.EntityA);
        }
        void Resolve(Entity a, Entity b)
        {
            if (ProjLookup.HasComponent(a) && UnitLookup.HasComponent(b))
                Hits.Enqueue(new HitEvent { Target = b, Source = a, Damage = ProjLookup[a].Damage });
        }
    }
}
```

> 关键：**`ITriggerEventsJob` 只读 + 产出事件**，绝不在此扣血（写竞争）。事件入 `NativeQueue.ParallelWriter`（并发安全），主线程回放写入伤害 buffer。

### 路线 B：自建 UniformGrid 重叠（无物理依赖）

若不引 Unity.Physics：用 [[【笔记】大规模单位AI决策与寻路]] 的 `NativeMultiHashMap<int3, Entity>` 网格，一个 Job 扫「投射物/攻击者」所在格的单位，距离判定命中，同样产出事件队列。近战范围攻击/AOE 也走这条路。优点是无物理引擎开销，缺点是要自己处理连续碰撞（隧道效应）。

## 三、伤害应用：DynamicBuffer + ECB（标准 DOTS 事件模式）

```csharp
using Unity.Burst;
using Unity.Entities;

/// <summary>把伤害事件沉淀到目标的 DynamicBuffer，统一结算。
/// 替代上面「主线程直接 buffer.Add」，更推荐用 ECB 回放。</summary>
[BurstCompile]
public partial struct CombatResolveSystem : ISystem
{
    [BurstCompile] public void OnUpdate(ref SystemState state)
    {
        var ecb = new EntityCommandBuffer(Allocator.TempJob);
        // 消费 CombatTriggerSystem 入队的事件（或直接由 trigger job 写 ECB.ParallelWriter）
        // 这里假设 trigger 阶段已把事件写进各目标的 DamageEvent buffer
        new ApplyDamageJob { Ecb = ecb.AsParallelWriter() }.ScheduleParallel();
        state.Dependency.Complete();
        ecb.Playback(state.EntityManager);
        ecb.Dispose();
    }

    /// <summary>逐单位：累加 buffer 伤害 → 扣血 → 清 buffer → 死亡判定。</summary>
    [BurstCompile]
    partial struct ApplyDamageJob : IJobEntity
    {
        public EntityCommandBuffer.ParallelWriter Ecb;
        void Execute(Entity e, [EntityIndexInQuery] int sortKey, ref Health hp, ref UnitAnim anim,
                     DynamicBuffer<DamageEvent> events)
        {
            if (events.Length == 0 || hp.Current <= 0) { events.Clear(); return; }
            float total = 0f;
            foreach (var ev in events) total += ev.Amount;
            hp.Current -= total;
            events.Clear();

            if (hp.Current <= 0f) {
                hp.Current = 0f;
                // 切死亡动画（动画系统会据此播放 death VAT）
                if (anim.State != AnimState.Death) { anim.State = AnimState.Death; anim.Time = 0f; }
                // 软禁用碰撞/逻辑（enableable 组件），动画播完再回收（见下）
                Ecb.SetComponentEnabled<AliveTag>(sortKey, e, false);
            }
        }
    }
}
```

> `DynamicBuffer<DamageEvent>` + ECB 是 DOTS 事件的标准解法：buffer 天然支持「一帧多源命中」（同名 buffer 多元素），ECB 解决「job 内不能结构性变更」。详见 [[【教程】ECS架构入门]] 的 EntityCommandBuffer。

## 四、死亡回收：IEnableableComponent + 对象池

死亡不能 `DestroyEntity`（10w 规模结构性变更昂贵），分两步：

1. **瞬时**：扣血 ≤0 → `AnimState.Death`（动画系统播放死亡 VAT）+ `SetComponentEnabled<AliveTag>(false)` 软禁用逻辑/碰撞（bit 开关，无结构性变更）。
2. **延迟**：死亡动画播完（`anim.Time > deathClipLength`）→ 回对象池（重置血量/位置 + `AliveTag` 禁用 + 移出渲染查询）而非销毁。

```csharp
public struct AliveTag : IComponentData, IEnableableComponent { }  // 软开关：死亡时禁用

[BurstCompile]
public partial struct DeathRecycleSystem : ISystem
{
    [BurstCompile] public void OnUpdate(ref SystemState state)
    {
        float deadLen = 1.2f; // death 动画时长（秒，从 VAT LUT 取）
        var ecb = new EntityCommandBuffer(Allocator.TempJob);
        new RecycleJob { Ecb = ecb.AsParallelWriter(), DeadLen = deadLen }.ScheduleParallel();
        state.Dependency.Complete();
        ecb.Playback(state.EntityManager);
        ecb.Dispose();
    }

    [BurstCompile]
    partial struct RecycleJob : IJobEntity
    {
        public EntityCommandBuffer.ParallelWriter Ecb;
        public float DeadLen;
        void Execute(Entity e, [EntityIndexInQuery] int k, in UnitAnim anim, in Health hp, in LocalTransform t)
        {
            if (hp.Current > 0f || anim.State != AnimState.Death) return;
            if (anim.Time >= DeadLen) {
                // 回对象池：移到池位置 + 标记可复用（实际由 PoolSystem 处理；这里示意）
                // Ecb.SetComponent(k, e, new Health { Current = Max, Max = Max });
                // Ecb.SetComponentEnabled<AliveTag>(k, e, true);  // 下次复用时再启用
            }
        }
    }
}
```

> 对象池细节（复用 Entity、重置组件、入池查询排除）超出本文，核心是 **`IEnableableComponent` 做软开关 + ECB 做结构性变更**，呼应 [[【笔记】大规模单位动画方案]] 的避坑。

## 五、AOE / 范围伤害

范围技能不做逐投射物碰撞，而是**事件触发时做一次空间查询**：
1. 释放技能 → 记录 `(center, radius, damage)` 到一个 singleton/事件 buffer；
2. `AOESystem`（Burst 并行）扫 `UniformGrid` 中半径内单位 → 写 `DamageEvent` 到各单位 buffer；
3. 后续走第三节的 `ApplyDamageJob` 统一结算。

这样 AOE 与单体命中**共用同一套伤害结算**，逻辑统一、无竞争。

## 六、系统时序（关键，否则读半更新数据）

```
FixedStepSimulationSystemGroup
  ├─ BuildPhysicsWorld / StepPhysicsWorld   (Unity.Physics 步进)
  ├─ CombatTriggerSystem                    (ITriggerEventsJob → NativeQueue)
  └─ CombatResolveSystem                    (ApplyDamageJob：扣血 + 死亡判定)
SimulationSystemGroup
  ├─ DeathRecycleSystem                     (死亡动画到时间 → 回收)
  ├─ UnitAnimSystem                         (据 Health/State 切动画，见动画笔记)
  └─ UnitMovementSystem / SteeringSystem
```

- **碰撞 → 伤害** 必须 `[UpdateAfter]`，且 trigger job 写 queue、resolve job 读 queue，中间 `Complete()`。
- **死亡标记 → 动画系统** 读 `AnimState.Death`，要 `[UpdateAfter]` resolve。
- `ComponentLookup`/`BufferLookup` 跨系统用要每帧 `Update()` 刷新。

## 七、避坑

| 现象 | 根因 / 处置 |
|------|------------|
| 同一帧扣血异常 | trigger job 里直接写 Health（竞争）→ 改为产出事件，resolve 单点结算 |
| 死亡单位仍在渲染/参与碰撞 | 没软禁用 `AliveTag`，渲染/碰撞 query 没排除死亡 → query 加 `Exclude<AliveTag>` 或用 enableable |
| ECS 里物理事件不触发 | Unity.Physics 需配 `PhysicsCollider` + trigger 状态；单位用 kinematic（`PhysicsMass` + 无 velocity 由你自控） |
| 隧道（投射物穿过去） | 离散触发 + 高速投射物 → 用 Unity.Physics 连续碰撞或自建扫掠检测 |
| ECB 帧末开销大 | ECB 不要逐 entity 单写，用 ParallelWriter + 合理 sortKey；结构性变更多的系统考虑结构性缓冲 |
| Lookup 报旧数据 | 跨系统 `ComponentLookup` 没每帧刷新 → `state.GetComponentLookup<T>()` 放 OnUpdate 顶部 |

## 速查清单

- [ ] 碰撞只产出事件（NativeQueue/ECB），不在此扣血
- [ ] 伤害走 `DynamicBuffer<DamageEvent>` + ECB 单点结算，无写竞争
- [ ] 死亡用 `IEnableableComponent`(AliveTag) 软禁用 + ECB 结构性回收，禁每帧 DestroyEntity
- [ ] AOE 复用同一伤害结算管线（空间查询 → 写 DamageEvent buffer）
- [ ] 系统时序：碰撞 → 伤害 → 死亡回收 → 动画，`[UpdateAfter]` + Lookup 每帧刷新
- [ ] Unity.Physics 配 kinematic + trigger；高速投射物防隧道
