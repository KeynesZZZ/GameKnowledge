# Inbox临时笔记处理流程

> `01_Inbox` 用来接住还没想清楚归类的内容，避免因为分类压力导致知识不进入系统。

## 放什么

适合放入 Inbox 的内容：

- 临时问题记录
- 未验证的技术想法
- 外部文章摘要
- AI 对话中有价值但未整理的结论
- 项目中刚踩到、但还没复盘清楚的问题
- 准备以后整理成教程、踩坑记录、最佳实践的草稿

不适合长期留在 Inbox 的内容：

- 已经验证过的完整方案
- 可以直接归类到架构、性能、工具链的文档
- 没有上下文的链接收藏
- 过期的临时状态记录

## 文件命名

推荐命名：

```text
YYYY-MM-DD-主题.md
```

示例：

```text
2026-05-29-Addressables重复加载问题.md
2026-05-29-AI生成Unity测试用例想法.md
```

## 最小结构

Inbox 文档可以很轻，只需要写清楚问题和下一步：

```markdown
# 主题

## 背景

## 当前结论

## 待验证

## 后续归类
```

## 每周清理规则

每周处理一次 Inbox：

| 状态 | 处理方式 |
|------|----------|
| 已经解决的问题 | 整理为 `【踩坑记录】` |
| 可复用做法 | 整理为 `【最佳实践】` |
| 可复制代码/命令 | 整理为 `【代码片段】` |
| 方案对比 | 整理为 `【架构决策】` |
| 暂无价值 | 删除或归档 |
