# 【代码片段】C# 泛型对象池通用实现

## 一、完整代码

```csharp
using System;
using System.Collections.Generic;

/// <summary>
/// 通用泛型对象池 - 减少实例化开销和GC分配
/// </summary>
/// <typeparam name="T">池化对象类型，需为引用类型</typeparam>
public class ObjectPool<T> where T : class
{
    private readonly Stack<T> pool;
    private readonly Func<T> createFunc; // 对象创建工厂
    private readonly Action<T> onReturn; // 归还时重置状态
    public int CountInactive => pool.Count; // 池中可用对象数

    public ObjectPool(Func<T> createFunc, Action<T> onReturn = null, int maxSize = 1000)
    {
        this.createFunc = createFunc ?? throw new ArgumentNullException(nameof(createFunc));
        this.onReturn = onReturn;
        pool = new Stack<T>(maxSize);
    }

    // 从池中获取对象
    public T Get() => pool.Count > 0 ? pool.Pop() : createFunc();

    // 归还对象到池中
    public void Return(T item)
    {
        if (item == null) return;
        onReturn?.Invoke(item);
        if (pool.Count < maxSize) pool.Push(item);
    }
}
```

## 二、使用示例

```csharp
// 子弹池使用示例
var bulletPool = new ObjectPool<Bullet>(() => new Bullet(), bullet => bullet.ResetState());
var bullet = bulletPool.Get();
// 使用后归还
bulletPool.Return(bullet);
```

## 三、关键要点

1. **泛型约束**：`where T : class` 保证为引用类型，避免值类型装箱
2. **状态重置**：通过 `onReturn` 回调在归还时重置对象状态（如子弹位置、速度）
3. **容量限制**：设置 `maxSize` 防止池过大导致内存泄漏
4. **适配 Unity**：可结合 `MonoBehaviour` 实现 GameObject 专用池，重写 `OnSpawn/OnDespawn`

| GC分配 | 0B | 4.8KB | **100%** |
