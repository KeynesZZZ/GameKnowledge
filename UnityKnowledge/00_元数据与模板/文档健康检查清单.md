# 文档健康检查清单

> 健康文档应该能被找到、能被读懂、能被验证、能被复用。

## 元数据检查

- [ ] 有 YAML Frontmatter
- [ ] `title` 包含正确文档类型前缀
- [ ] `tags` 至少 2 个，且不带 `#`
- [ ] `category` 属于标准分类路径
- [ ] `created` 和 `updated` 格式一致
- [ ] `description` 简洁，能说明核心价值
- [ ] `status` 可判断维护状态
- [ ] `validation` 可判断可信度

## 结构检查

- [ ] 有 `文档定位`
- [ ] 有 `相关链接`
- [ ] 章节符合文档类型
- [ ] 没有为了模板硬凑的空章节
- [ ] 没有把多个文档类型混成一篇

## 内容检查

- [ ] 说明了适用场景
- [ ] 说明了不适用或风险场景
- [ ] 关键结论有理由、示例或验证
- [ ] 代码片段能独立复制或明确依赖
- [ ] 性能结论包含测试环境和数据
- [ ] 架构决策包含备选方案和取舍

## 链接检查

- [ ] 内部链接能跳转到真实文档
- [ ] 相关链接和正文确实相关
- [ ] 已过时文档链接到替代文档
- [ ] 外部链接仍可访问

## 过期检查

- [ ] Unity 版本仍适用
- [ ] 第三方库版本仍适用
- [ ] 推荐方案仍是当前最佳选择
- [ ] `updated` 不是长期停留在旧日期
- [ ] `已过时` 文档不再被新文档当作推荐方案引用

## 每周检查

- [ ] 清理 `01_Inbox`
- [ ] 检查新增文档是否符合结构规范
- [ ] 修复断链或错误分类
- [ ] 把成熟草稿升级为正式文档

## 每月检查

- [ ] 复查高频文档
- [ ] 合并重复主题
- [ ] 标记过时文档
- [ ] 更新学习路径和专题索引
