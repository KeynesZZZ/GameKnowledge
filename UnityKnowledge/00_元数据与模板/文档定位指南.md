# 文档定位指南

> UnityKnowledge 文档类型说明与使用指南

## 文档类型体系

UnityKnowledge 使用 **8种文档类型** 来组织知识，每种类型有明确的定位和用途。

### 文档类型一览表

| 文档类型 | 标题前缀 | 核心问题 | 定位 | 典型内容 |
|---------|---------|---------|------|---------|
| **设计原理** | 【设计原理】 | Why | 底层原理、本质思考、设计权衡 | 为什么用、为什么这样设计 |
| **教程** | 【教程】 | How to use | 使用方法、API说明、学习路径 | 如何使用、如何实现 |
| **代码片段** | 【代码片段】 | Quick copy | 可复用代码、模板、工具类 | 复制即用的代码 |
| **性能数据** | 【性能数据】 | How fast | 性能测试、benchmark、对比数据 | 性能数据、测试结果 |
| **最佳实践** | 【最佳实践】 | Best practice | 推荐做法、经验总结 | 应该这样做 |
| **踩坑记录** | 【踩坑记录】 | Pitfalls | 常见问题、解决方案、错误案例 | 不要这样做、如何避坑 |
| **架构决策** | 【架构决策】 | Which | 方案对比、选择标准、决策依据 | A方案vs B方案 |
| **系统架构** | 【系统架构】 | What | 系统设计、模块关系、架构图 | 整体架构、模块划分 |

---

## 各类型详细说明

### 1. 设计原理 【设计原理】

**定位**：解释"为什么"，深入理解底层机制和设计思想

**适合内容**：
- 某技术的本质原理（如：对象池本质）
- Unity系统的底层机制（如：场景加载底层机制）
- 设计模式的核心思想（如：策略模式本质）
- 技术选择的权衡考量（如：ECS为什么快）

**示例**：
- `【设计原理】对象池本质` - 为什么对象池能解决GC问题
- `【设计原理】场景加载底层机制` - Unity如何加载场景
- `【设计原理】ECS为什么快` - ECS架构的底层优势

**写作要点**：
- 从问题出发，引出原理
- 用图示解释机制
- 说明适用场景和不适用场景
- 避免过多的实现细节（留给"代码片段"）

**相关文档**：
- → 教程（如何使用）
- → 代码片段（具体实现）
- → 性能数据（性能表现）

---

### 2. 教程 【教程】

**定位**：讲解"如何使用"，循序渐进的学习指南

**适合内容**：
- 技术入门教程
- API使用方法
- 系统性学习路径
- 功能实现步骤

**示例**：
- `【教程】对象池模式` - 如何在项目中使用对象池
- `【教程】DOTween快速入门` - 从零开始学习DOTween
- `【教程】动画系统学习路径` - 动画系统学习顺序

**写作要点**：
- 从零开始，循序渐进
- 提供完整示例
- 说明常见错误
- 链接到相关资源

**相关文档**：
- ← 设计原理（理解原理）
- → 代码片段（复制代码）
- → 最佳实践（注意事项）

---

### 3. 代码片段 【代码片段】

**定位**：提供"可复制代码"，开箱即用的解决方案

**适合内容**：
- 完整的类实现
- 工具函数库
- 代码模板
- 常用代码模式

**示例**：
- `【代码片段】对象池通用实现` - 可直接使用的对象池类
- `【代码片段】事件总线` - 完整的事件总线实现
- `【代码片段】有限状态机` - FSM模板代码

**写作要点**：
- 代码可直接复制使用
- 包含完整的注释
- 提供使用示例
- 说明依赖和配置

**相关文档**：
- ← 设计原理（理解原理）
- ← 教程（如何使用）
- → 性能数据（性能表现）

---

### 4. 性能数据 【性能数据】

**定位**：提供"性能数据"，基于测试的客观证据

**适合内容**：
- 性能测试结果
- Benchmark对比
- 内存分配分析
- 帧率影响测试

**示例**：
- `【性能数据】对象池vs实例化` - 74x性能提升
- `【性能数据】foreach vs for` - 循环性能对比
- `【性能数据】UGUI DrawCall影响因素` - UI性能测试

**写作要点**：
- 包含测试环境
- 提供测试代码
- 数据可视化（表格、图表）
- 基于数据给出建议

**相关文档**：
- ← 设计原理（理解原理）
- ← 最佳实践（优化建议）
- ← 代码片段（优化后代码）

---

### 5. 最佳实践 【最佳实践】

**定位**：总结"应该这样做"，基于经验的推荐做法

**适合内容**：
- 推荐的编程方式
- 经验总结
- 使用建议
- 优化技巧

**示例**：
- `【最佳实践】依赖注入使用指南` - 如何正确使用DI
- `【最佳实践】GC优化清单` - 避免GC的检查清单
- `【最佳实践】单例模式` - 单例的多种实现方式

**写作要点**：
- 给出明确的DO和DON'T
- 说明理由
- 提供代码示例
- 链接到相关资源

**相关文档**：
- ← 设计原理（理解原理）
- ← 踩坑记录（反面教材）
- → 代码片段（正确实现）

---

### 6. 踩坑记录 【踩坑记录】

**定位**：记录"不要这样做"，常见问题和解决方案

**适合内容**：
- 常见错误
- 易踩的坑
- 问题排查过程
- 解决方案

**示例**：
- `【踩坑记录】UGUI DrawCall飙升` - UI性能问题排查
- `【踩坑记录】Android常见问题清单` - Android平台坑点
- `【踩坑记录】内存泄漏模式` - 内存泄漏的常见原因

**写作要点**：
- 描述问题现象
- 分析问题原因
- 给出解决方案
- 说明如何避免

**相关文档**：
- ← 最佳实践（正确做法）
- ← 设计原理（理解原因）

---

### 7. 架构决策 【架构决策】

**定位**：对比"A方案vs B方案"，帮助做出技术选择

**适合内容**：
- 技术方案对比
- 实现方式选择
- 技术栈决策
- 架构权衡

**示例**：
- `【架构决策】单例vs依赖注入` - 两种方式深度对比
- `【架构决策】UI架构-MVP vs MVVM` - UI架构选择
- `【架构决策】组件化vs继承` - 代码组织方式

**写作要点**：
- 多维度对比
- 给出选择建议
- 说明适用场景
- 保持客观中立

**相关文档**：
- ← 设计原理（理解原理）
- → 最佳实践（使用建议）
- → 代码片段（具体实现）

---

### 8. 系统架构 【系统架构】

**定位**：描述"整体架构"，系统设计和模块关系

**适合内容**：
- 系统架构设计
- 模块划分和职责
- 架构图
- 跨模块交互

**示例**：
- `【系统架构】场景管理` - 场景加载系统完整架构
- `【系统架构】存档系统` - 存档系统设计
- `【系统架构】输入管理` - 输入系统架构

**写作要点**：
- 提供架构图
- 说明模块职责
- 描述交互流程
- 链接到实现细节

**相关文档**：
- ← 架构决策（为什么这样设计）
- → 代码片段（具体实现）
- → 20_核心系统/（底层机制）

## 文档层次关系

UnityKnowledge 有 **4个层次** 的文档，从理论到实践：

```
层次1：理论基础 (10_架构设计/)
├─ 侧重点：为什么这样设计
├─ 文档类型：【设计原理】、【架构决策】
└─ 示例：设计原理-对象池本质.md

层次2：系统深入 (20_核心系统/)
├─ 侧重点：Unity系统如何工作
├─ 文档类型：【设计原理】、【源码解析】、【教程】
└─ 示例：【设计原理】场景加载底层机制.md

层次3：性能优化 (30_性能优化/)
├─ 侧重点：如何优化性能
├─ 文档类型：【性能数据】、【最佳实践】、【踩坑记录】
└─ 示例：【性能数据】对象池vs实例化.md

层次4：项目实战 (100_项目实战/)
├─ 侧重点：端到端实现
├─ 文档类型：需求分析、核心实现、集成
└─ 示例：01_休闲游戏云存档系统/
```

## 跨目录交叉引用规则

### 规则1：建立主题索引

当某个主题有多个相关文档时，创建"主题索引"：

```markdown
## 对象池专题索引

### 理论基础
- [[设计原理-对象池本质]] ← 为什么用对象池

### 使用指南
- [[教程-对象池模式]] ← 如何使用对象池
- [[代码片段-对象池通用实现]] ← 复制即用的代码

### 性能数据
- [[../30_性能优化/32_内存管理/【性能数据】对象池vs实例化]] ← 性能测试

### 实战案例
- [[../100_项目实战/XX项目]] ← 项目中的使用
```

### 规则2：文档开头定位

在每个文档开头添加"文档定位"章节：

```markdown
## 文档定位

本文档从**架构设计角度**讲解场景管理系统。

**底层机制详解**：参见 [[../20_核心系统/27_场景系统/【设计原理】场景加载底层机制]]

## 文档内容
- 场景管理器完整实现代码
- 加载界面与过渡动画
- 场景数据传递
```

### 规则3：相关链接标准格式

```markdown
## 相关链接

### 同一主题的系列文档
- **理论层**: [[设计原理-XXX]] ← 底层原理
- **实现层**: [[代码片段-XXX]] ← 具体实现
- **性能层**: [[../30_性能优化/XXX/【性能数据】XXX]] ← 性能数据
- **实战层**: [[../100_项目实战/XXX]] ← 项目案例

### 深入阅读
- [[../20_核心系统/XX系统/【设计原理】XXX]] ← 系统深度解析
- [[../10_架构设计/架构决策-XXX]] ← 方案对比
```

## 新建文档检查清单

创建新文档时，确保：

- [ ] 选择合适的文档类型
- [ ] 添加 YAML frontmatter
- [ ] 在文档开头添加"文档定位"章节
- [ ] 在文档末尾添加"相关链接"章节
- [ ] 如果是系列文档之一，添加"专题索引"
- [ ] 至少添加 2 个 tags
- [ ] 填写 description（50字以内）
- [ ] 说明 Unity 版本（如适用）
- [ ] 更新父目录的 README.md（如需要）
