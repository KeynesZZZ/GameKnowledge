> 本文档记录整个 UnityKnowledge 知识库需要进行的规范化工作

## 一、规范化规则汇总

### 1.1 目录命名规则

| 规则 | 格式 | 示例 |
|------|------|------|
| 顶级目录 | `NN_中文名称/` | `10_架构设计/` |
| 二级目录 | `NN_英文名称/` | `21_动画系统/` |
| 编号范围 | 按功能分类 | 00元数据, 10-30技术, 40-60工具, 90-100实战 |

**禁止**：
- ❌ 中文目录名（如 `动画系统/`）
- ❌ 空格（如 `渲染 与 Shader/`）
- ❌ 无编号（如 `DOTS技术栈/`）

### 1.2 文件命名规则

| 文档类型 | 格式 | 示例 |
|---------|------|------|
| 代码片段 | `代码片段-主题.md` | `代码片段-事件总线.md` |
| 设计原理 | `设计原理-主题本质.md` | `设计原理-状态机本质.md` |
| 最佳实践 | `最佳实践-主题.md` | `最佳实践-GC优化清单.md` |
| 架构决策 | `架构决策-选择问题.md` | `架构决策-单例vs依赖注入.md` |
| 系统架构 | `系统架构-系统名.md` | `系统架构-场景管理.md` |
| 实战案例 | `实战案例-项目名.md` | `实战案例-三消游戏架构.md` |
| 反模式 | `反模式-问题名.md` | `反模式-常见架构陷阱.md` |
| 教程 | `教程-主题.md` | `教程-状态机模式.md` |
| 架构演进 | `架构演进-主题.md` | `架构演进-从Demo到量产.md` |
| 性能数据 | `性能数据-测试主题.md` | `性能数据-输入系统基准测试.md` |
| 踩坑记录 | `踩坑记录-问题描述.md` | `踩坑记录-协程泄漏.md` |

**禁止**：
- ❌ 空格（如 `UI 架构.md`）
- ❌ 无前缀（如 `状态机.md`）
- ❌ 前缀顺序错误（如 `单例模式最佳实践.md`）

### 1.3 YAML Frontmatter 规范

```yaml
---
title: 【类型】主题名称
tags: [C#, Unity, 分类标签1, 分类标签2, 文档类型]
category: 一级分类/二级分类
created: YYYY-MM-DD HH:MM
updated: YYYY-MM-DD HH:MM
description: 一句话描述文档内容（50字以内）
unity_version: 2021.3+
dependencies: [可选，依赖的包或库]
---
```

**必需字段**：title, tags, category, created, updated, description

### 1.4 内部链接规范

```markdown
# 同目录链接（推荐）
[[代码片段-策略模式]]

# 跨目录链接（相对路径）
[教程](../10_架构设计/教程-状态机模式.md)

# 外部链接
[官方文档](https://docs.unity3d.com/...)
```

### 4.2 YAML Frontmatter 添加脚本

```bash
#!/bin/bash
# 为文件添加 YAML frontmatter

add_yaml() {
    local file=$1
    local title=$2
    local tags=$3
    local category=$4
    local desc=$5

    local yaml="---
title: $title
tags: $tags
category: $category
created: $(date '+%Y-%m-%d %H:%M')
updated: $(date '+%Y-%m-%d %H:%M')
description: $desc
unity_version: 2021.3+
---
"

    echo "$yaml" | cat - "$file" > /tmp/tmpfile && mv /tmp/tmpfile "$file"
}

# 使用示例
# add_yaml "文件名.md" "【类型】标题" "[C#, Unity, 标签]" "分类" "描述"
```
