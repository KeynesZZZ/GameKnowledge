# 【笔记标题】如：Unity策略模式-代码实现与应用

## 一、核心概述
### 1.1 定义/作用
- 一句话讲清核心概念，例：策略模式是一种行为型设计模式，将算法/行为封装为独立策略，实现行为与调用方解耦
- 核心价值：例：Unity中可快速切换AI行为、技能效果，降低代码耦合，便于扩展新策略

### 1.2 适用场景
- 场景1：例：AI角色的多行为切换（巡逻/追击/攻击/逃跑）
- 场景2：例：技能系统的多效果组合（伤害/治疗/减速/护盾）
- 避坑点：例：行为逻辑简单且无需扩展时，无需使用（避免过度设计）

### 2.1 代码结构（按逻辑分层）

### 2.2 关键要点
- 核心亮点：例：使用泛型策略管理器实现策略的注册、获取与复用，支持延迟创建
- 注意事项：例：策略类需保证单一职责，仅封装对应行为逻辑，不包含业务判断
- 语法/API细节：例：Unity中Time.deltaTime需在Update/FixedUpdate中使用，保证移动速度统一

### 2.3 扩展/变种实现
- 例：带优先级的策略选择（逃跑>攻击>追击>巡逻）
- 例：结合对象池实现策略对象的复用，减少GC分配

## 三、实际应用案例
### 3.1 案例场景
例：Unity 3D俯视角游戏中，敌人AI的行为切换逻辑

### 3.2 落地代码（精简版）
```csharp
// 粘贴可直接运行的精简案例代码，保留核心逻辑，剔除冗余
public class AIController : MonoBehaviour
{
    private Dictionary<string, IBehaviorStrategy> strategies;
    private void Awake()
    {
        // 注册策略
        strategies = new Dictionary<string, IBehaviorStrategy>()
        {
            { "Patrol", new PatrolStrategy(GetPatrolPoints(), 2f) },
            { "Chase", new ChaseStrategy(3f, 1f) }
        };
    }
    private void Update()
    {
        // 策略选择逻辑
        if (strategies["Flee"].CanExecute(enemy)) SetStrategy("Flee");
        else if (strategies["Attack"].CanExecute(enemy)) SetStrategy("Attack");
    }
}
```

### 3.3 效果/性能
- 实现效果：例：敌人可根据血量、与玩家距离自动切换行为，无卡顿
- 性能优化：例：策略对象仅初始化一次，避免每帧创建，减少GC 0B分配

## 相关链接
### 4.1 与相似模式对比
| 设计模式 | 核心区别 | 适用场景 |
|----------|----------|----------|
| 策略模式 | 封装不同**行为/算法**，可动态切换 | AI行为、技能效果、排序算法 |
| 状态模式 | 封装不同**状态**，状态间有固定转换关系 | 角色状态（Idle/Run/Jump/Dead） |
| 命令模式 | 封装**请求**，支持撤销/重做 | 技能释放、操作记录、输入绑定 |

## 五、常见问题/排错技巧
1. 问题：策略切换无效果 → 排查CanExecute方法的条件是否写反，例：距离判断用>=而非<=
2. 问题：策略对象重复创建 → 排查是否在Update中注册策略，应移至Awake/Start
3. 问题：Unity中物体移动卡顿 → 排查是否将移动逻辑写在Update，且未乘Time.deltaTime

## 六、总结/个人心得
- 核心收获：例：策略模式的核心是"封装变化、面向接口编程"，在Unity中可大幅提升代码的扩展性和可维护性
- 个人技巧：例：将常用策略封装为通用类，通过构造函数传参实现定制化，避免重复写相似策略
- 后续拓展：例：结合事件总线实现策略的动态注册与注销，支持热更新策略

## 使用说明

### 1. 在 Obsidian 中使用

1. 打开 Obsidian → 设置 → 插件 → 启用 **Templates**
2. 将此模板保存为 MD 文件，放在 Obsidian 库的 `_templates/` 文件夹中
3. 新建笔记时，按 `Ctrl+N` → 右键 → 插入模板 → 选择此模板

### 2. 自动填充时间/标题

安装插件 **Frontmatter Helper** + **Templater**，可实现：
- 新建笔记时自动填充 `created`/`updated` 的当前时间
- 自动将文件名同步为 `title` 字段

### 3. 标签规范

标签按「**语言/引擎 + 核心知识点 + 使用场景**」设置，例：

```yaml
tags: [C#, Unity, 设计模式, 性能优化]
```

配合 **Tag Wrangler** 插件可批量管理标签，检索更高效。
