## 必填字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `title` | string | 文档标题，带类型前缀 | `【代码片段】对象池通用实现` |
| `tags` | array | 标签数组，至少2个 | `[C#, Unity, 性能优化]` |
| `category` | string | 分类路径 | `架构设计/代码片段` |
| `created` | datetime | 创建时间 | `2024-01-15 10:30` |
| `updated` | datetime | 更新时间 | `2024-03-04 15:20` |
| `description` | string | 一句话描述 | `C#泛型对象池的基础实现，减少GC分配` |

## 可选字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `unity_version` | string | 适用的Unity版本 | `2022.3+` |
| `cover` | string | 封面图路径 | `./attachments/cover.png` |
| `complexity` | string | 复杂度 | `简单` / `中等` / `复杂` |
| `use_frequency` | string | 使用频率 | `高频` / `中频` / `低频` |
| `dependencies` | array | 依赖项 | `[DOTween, UniTask]` |
| `status` | string | 文档状态（见下方生命周期） | `提议` / `讨论中` / `已采纳` / `已实施` / `已验证` / `已过时` |
| `validation` | string | 验证程度 | `未经测试` / `Demo验证` / `项目实战` / `多项目验证` |

## 关系型元数据（v2.0 新增）

用于表达文档之间的**结构化关系**，支撑知识依赖图、自动学习路径生成和重构影响分析。

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `prerequisite` | array | 前置知识，阅读本文前应先阅读的文档 | `["【设计原理】状态机本质"]` |
| `depends_on` | array | 依赖系统，本文内容依赖的系统/模块文档 | `["【系统架构】场景管理"]` |
| `is_example_for` | array | 本文是某篇文档的实战案例 | `["【教程】状态机模式"]` |
| `refutes` | array | 本文反驳/纠正了某个观点 | `["【反模式】常见架构陷阱#Update泛滥"]` |
| `supersedes` | array | 本文取代了哪些旧文档 | `["【架构决策】旧版单例方案"]` |
| `related` | array | 相关文档（无方向性的弱关联） | `["【性能数据】foreach vs for"]` |

### 关系型元数据示例

  - "【设计原理】策略模式本质"

## 状态与生命周期（v2.0 新增）

### status 状态流转

文档从创建到归档的完整生命周期：

```
提议 → 讨论中 → 已采纳 → 已实施 → 已验证 → 已过时
                    ↓                       ↓
                  已拒绝                   已归档
```

| 状态 | 含义 | 适用场景 |
|------|------|----------|
| `提议` | 初始想法，尚未讨论 | 新建的架构决策草案 |
| `讨论中` | 团队正在评审 | ADR正在Code Review |
| `已采纳` | 方案已通过，待实施 | 团队达成共识的技术方案 |
| `已实施` | 代码已落地，待验证 | 已在项目中编码实现 |
| `已验证` | 经过实战验证，可信赖 | 上线运行且效果达标 |
| `已过时` | 因技术演进不再适用 | Unity大版本升级导致API变更 |
| `已拒绝` | 评审后未通过 | 被否决的架构方案 |
| `已归档` | 保留但不再维护 | 历史参考文档 |

### validation 验证程度

| 验证级别 | 含义 | 可信度 |
|----------|------|--------|
| `未经测试` | 纯理论分析或经验总结 | ⭐ |
| `Demo验证` | 在独立Demo中测试通过 | ⭐⭐ |
| `项目实战` | 在至少一个正式项目中使用 | ⭐⭐⭐⭐ |
| `多项目验证` | 在多个项目中反复验证 | ⭐⭐⭐⭐⭐ |

## 适用场景元数据（v2.0 新增）

用于实现**智能推荐**：新项目启动时，可根据项目属性自动筛选适用的架构模式与最佳实践。

```yaml
applicability:
  project_scale: ["小型原型", "中型团队", "大型量产"]
  project_type: ["3D动作", "2D策略", "联网多人", "休闲", "三消"]
  performance_critical: true
```

| 字段 | 类型 | 说明 | 可选值 |
|------|------|------|--------|
| `applicability.project_scale` | array | 适用项目规模 | `小型原型` / `中型团队` / `大型量产` |
| `applicability.project_type` | array | 适用游戏类型 | `3D动作` / `2D策略` / `联网多人` / `休闲` / `三消` / `Roguelike` / `卡牌` / `SLG` |
| `applicability.performance_critical` | bool | 是否性能敏感 | `true` / `false` |

## 版本兼容性元数据（v2.0 新增）

避免"知识过期"，Unity大版本升级或包更新时可快速扫描受影响文档。

```yaml
unity_version: "2022.3+"
package_versions:
  - "com.unity.entities": "1.0.0"
  - "com.unity.burst": "1.8.0"
  - "VContainer": "1.1.0"
```

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `unity_version` | string | 适用的Unity最低版本 | `2022.3+` |
| `package_versions` | array | 依赖的包及版本 | `["com.unity.dots: 1.0.0"]` |

## Obsidian 兼容性

### 自动填充

使用 Templater 插件自动填充：

```yaml
---
title: {{title}}
created: {{date:YYYY-MM-DD HH:mm}}
updated: {{date:YYYY-MM-DD HH:mm}}
---
```

### 搜索语法

在 Obsidian 中搜索：

```
# 按标签搜索
tag:性能优化

# 按分类搜索（使用 Dataview 插件）
list
from "UnityKnowledge/10_架构设计"
where contains(category, "架构设计")

# 按时间范围搜索
created:>=2024-03-01
```

## Dify 同步兼容性

### 向量化处理

YAML frontmatter 会被 Dify 自动识别：
- `title` → 文档标题
- `tags` → 元数据标签
- `description` → 文档摘要

### 分段建议

Dify 索引参数配置：
- 分段方式: 自动分段
- 分段标识符: `\n\n\n`
- 最大分段长度: 500 tokens
- 重叠长度: 50 tokens

重构文档时，确保：

- [ ] 添加完整的 YAML frontmatter

- [ ] tags 数组至少包含 2 个标签
- [ ] category 使用标准分类路径

- [ ] description 简洁准确（建议 50 字以内）
