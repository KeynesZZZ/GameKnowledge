## 自动化脚本

位于 `../scripts/` 目录：

| 脚本 | 用途 | 使用场景 |
|------|------|----------|
| `validate_metadata.py` | 校验元数据完整性 | CI流水线、提交前检查 |
| `generate_index.py` | 生成文档索引和知识图谱 | 定期更新、构建站点 |
| `check_links.py` | 检查内部链接有效性 | 发现断链、维护链接健康 |

### 快速使用

```bash
# 校验所有文档
python scripts/validate_metadata.py

# 严格模式（警告也视为错误）
python scripts/validate_metadata.py --strict

# 检查链接
python scripts/check_links.py

# 生成索引
python scripts/generate_index.py
```

## 快速开始

### 新建文档三步走

1. **选择类型** → 查看 [[文档定位指南]]，确定文档类型
2. **复制模板** → 从上方模板列表选择对应模板
3. **填写元数据** → 参考 [[元数据规范]] 填写完整YAML Frontmatter

### 元数据最小示例

```yaml
---
title: 【代码片段】对象池通用实现
tags: [C#, Unity, 架构, 性能优化, 代码片段]
category: 架构设计/代码片段
created: 2026-03-06 10:00
updated: 2026-03-06 10:00
description: C#泛型对象池的基础实现，减少GC分配
status: 已验证
validation: 多项目验证
prerequisite:
  - "【设计原理】对象池本质"
depends_on:
  - "【系统架构】对象管理"
related:
  - "【性能数据】对象池vs实例化"
---
```
