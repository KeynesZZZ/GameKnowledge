## 三、数据流设计

### 关键数据流

描述系统中关键数据的**生产者、消费者、存储位置和传输方式**。

| 数据 | 生产者 | 传输方式 | 消费者 | 存储位置 |
|------|--------|----------|--------|----------|
| `InputData` | InputSystem | EventBus | PlayerController | 无（实时消费） |
| `DamageEvent` | CombatSystem | 事件队列 | UIManager, VFXManager | 无（实时消费） |
| `SaveData` | GameState | 序列化 | SaveManager | 本地文件/云端 |

### 数据流时序图

```
（描述关键流程的时序）

用户输入 → InputSystem → EventBus → PlayerController
                                   → AnimationController
                                   → CameraController

PlayerController → CombatSystem → DamageCalculator
                                → VFXManager
                                → UIManager (血条更新)
```

## 四、并发与线程安全

### 线程模型

| 操作 | 执行线程 | 说明 |
|------|----------|------|
| 游戏逻辑 | 主线程 | MonoBehaviour生命周期 |
| 物理计算 | 物理线程 | FixedUpdate |
| 资源加载 | 异步线程 | Addressables异步加载 |
| AI寻路 | Job System | Burst编译优化 |

### 线程安全策略

```csharp
// 示例：如何保证线程安全
// ❌ 不安全的做法
public class UnsafeManager
{
    private List<Enemy> enemies = new(); // 主线程和Job同时访问
}

// ✅ 安全的做法
public class SafeManager
{
    private NativeArray<EnemyData> enemies; // 使用NativeContainer
    // 或使用ConcurrentQueue进行线程间通信
}
```

### 同步机制

- 描述模块间如何同步状态
- 使用的锁/无锁策略
- 事件队列的消费模式（同步/异步）

## 六、错误处理

| 错误场景 | 处理策略 | 降级方案 |
|----------|----------|----------|
| 资源加载失败 | 重试3次后使用默认资源 | 显示占位符 |
| 网络断开 | 切换到本地模式 | 缓存最后状态 |
| 内存不足 | 触发资源卸载 | 降低画质 |
