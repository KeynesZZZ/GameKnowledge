# 【模板】架构决策记录（ADR）

> 使用说明：复制此模板创建新的架构决策文档，填写相关信息。ADR的核心价值在于记录"为什么选A而不选B"的完整推理过程。

## 一、决策概述

| 属性 | 值 |
|------|-----|
| **决策编号** | ADR-XXX |
| **决策日期** | YYYY-MM-DD |
| **决策者** | @姓名 |
| **参与讨论** | @姓名1, @姓名2 |
| **状态** | 提议 / 讨论中 / 已采纳 / 已实施 / 已验证 / 已过时 / 已拒绝 |
| **验证程度** | 未经测试 / Demo验证 / 项目实战 / 多项目验证 |
| **影响范围** | 如：全局架构 / 战斗系统 / UI模块 |
| **取代的决策** | 如：ADR-001（如有） |

### 背景与问题

用1-3段话描述：
1. 当前面临什么问题？
2. 为什么需要做出决策？
3. 有哪些约束条件？

### 决策结论

> **我们决定采用 [方案X]**，因为 [一句话核心理由]。

| 评估维度 | 权重 | 方案A | 方案B | 方案C |
