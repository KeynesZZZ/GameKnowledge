## 一、资源卸载概述

### 1.1 为什么需要资源卸载

```
┌─────────────────────────────────────────────────────────────┐
│              不卸载资源的后果                                  │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  问题1：内存持续增长                                         │
│  ├─ 加载的资源不释放                                        │
│  ├─ 内存占用越来越高                                        │
│  └─ 最终导致内存溢出（OOM）                                  │
│                                                             │
│  问题2：性能下降                                             │
│  ├─ 内存压力导致频繁GC                                      │
│  ├─ GC触发导致卡顿                                          │
│  └─ 帧率下降                                                │
│                                                             │
│  问题3：应用被终止                                           │
│  ├─ 移动平台内存限制严格                                     │
│  ├─ 内存超限被系统杀掉                                       │
│  └─ 用户体验极差                                            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

### 1.2 资源生命周期

```
资源生命周期：
1. 加载（Load）
   ├─ Resources.Load()
   ├─ AssetBundle.LoadAsset()
   └─ Instantiate()

2. 使用（In Use）
   ├─ 场景中引用
   ├─ 代码中引用
   └─ 系统中引用

3. 卸载（Unload）
   ├─ 释放显存（GPU）
   ├─ 释放内存（CPU）
   └─ 释放文件句柄
```

## 二、Resources资源卸载

### 2.1 Resources.UnloadUnusedAssets

#### 基本用法

```csharp
// 同步卸载未使用的资源
Resources.UnloadUnusedAssets();

// 异步卸载（推荐）
AsyncOperation operation = Resources.UnloadUnusedAssets();
while (!operation.isDone)
{
    Debug.Log($"卸载进度: {operation.progress * 100}%");
    yield return null;
}
```

#### 使用场景

```csharp
// 场景1：切换场景时卸载
IEnumerator LoadSceneAsync(string sceneName)
{
    // 卸载当前场景未使用的资源
    var unloadOp = Resources.UnloadUnusedAssets();
    yield return unloadOp;

    // 加载新场景
    var loadOp = SceneManager.LoadSceneAsync(sceneName);
    yield return loadOp;
}

// 场景2：内存警告时卸载
void OnApplicationPause(bool pause)
{
    if (pause)
    {
        // 应用进入后台时卸载
        Resources.UnloadUnusedAssets();
    }
}

// 场景3：定时卸载
IEnumerator PeriodicUnload()
{
    while (true)
    {
        yield return new WaitForSeconds(60f);
        Resources.UnloadUnusedAssets();
    }
}
```

---

### 2.2 Resources.UnloadAsset

#### 基本用法

```csharp
// 卸载单个资源
Texture2D texture = Resources.Load<Texture2D>("Textures/Background");

// 使用完毕后卸载
if (texture != null)
{
    Resources.UnloadAsset(texture);  // 卸载纹理
    texture = null;  // 清空引用
}
```

#### 注意事项

```
⚠️ 卸载前必须确保：
1. 没有对象在使用该资源
2. 清空所有对该资源的引用
3. 资源不是场景的一部分

❌ 不能卸载的情况：
- GameObject在场景中
- 组件在对象上
- 材质被Renderer使用
- Shader被材质使用
```

## 三、AssetBundle卸载

### 3.1 AssetBundle.Unload

#### 基本用法

```csharp
// 加载AssetBundle
AssetBundle bundle = AssetBundle.LoadFromFile("path/to/bundle");

// 加载资源
GameObject prefab = bundle.LoadAsset<GameObject>("Prefab");

// 使用完毕后卸载
bundle.Unload(true);  // true = 卸载所有资源，包括从bundle加载的对象
// 或
bundle.Unload(false);  // false = 只卸载bundle本身，保留已加载的资源
```

#### Unload(true) vs Unload(false)

```
┌─────────────────────────────────────────────────────────────┐
│         Unload(true) vs Unload(false)                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  Unload(true)                                              │
│  ├─ 卸载bundle本身                                         │
│  ├─ 卸载所有从该bundle加载的资源                            │
│  ├─ 销毁所有从该资源实例化的对象                           │
│  └─ 彻底清理                                               │
│                                                             │
│  Unload(false)                                             │
│  ├─ 卸载bundle本身                                         │
│  ├─ 保留已加载的资源                                       │
│  ├─ 不销毁已实例化的对象                                   │
│  └─ 需要手动卸载资源                                       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

### 3.2 AssetBundle管理器

```csharp
/// <summary>
/// AssetBundle管理器
/// </summary>
public class AssetBundleManager : MonoBehaviour
{
    private Dictionary<string, AssetBundle> loadedBundles = new Dictionary<string, AssetBundle>();
    private Dictionary<string, int> bundleRefCounts = new Dictionary<string, int>();

    /// <summary>
    /// 加载AssetBundle
    /// </summary>
    public AssetBundle LoadBundle(string bundleName)
    {
        if (loadedBundles.TryGetValue(bundleName, out AssetBundle bundle))
        {
            bundleRefCounts[bundleName]++;
            return bundle;
        }

        AssetBundle newBundle = AssetBundle.LoadFromFile(GetBundlePath(bundleName));
        loadedBundles[bundleName] = newBundle;
        bundleRefCounts[bundleName] = 1;

        return newBundle;
    }

    /// <summary>
    /// 卸载AssetBundle
    /// </summary>
    public void UnloadBundle(string bundleName)
    {
        if (!loadedBundles.ContainsKey(bundleName))
            return;

        bundleRefCounts[bundleName]--;

        if (bundleRefCounts[bundleName] <= 0)
        {
            loadedBundles[bundleName].Unload(true);
            loadedBundles.Remove(bundleName);
            bundleRefCounts.Remove(bundleName);
        }
    }

    /// <summary>
    /// 卸载所有Bundle
    /// </summary>
    public void UnloadAllBundles()
    {
        foreach (var bundle in loadedBundles.Values)
        {
            bundle.Unload(true);
        }

        loadedBundles.Clear();
        bundleRefCounts.Clear();
    }
}
```

## 四、纹理资源卸载

### 4.1 纹理卸载策略

```csharp
// ❌ 错误：只清空引用
Texture2D texture = Resources.Load<Texture2D>("Textures/Background");
spriteRenderer.sprite = Sprite.Create(texture, new Rect(0, 0, texture.width, texture.height), Vector2.zero);
texture = null;  // 错误：Sprite仍然引用纹理

// ✅ 正确：完全卸载
Texture2D texture = Resources.Load<Texture2D>("Textures/Background");
Sprite sprite = Sprite.Create(texture, new Rect(0, 0, texture.width, texture.height), Vector2.zero);
spriteRenderer.sprite = sprite;

// 卸载时
spriteRenderer.sprite = null;
Destroy(sprite);  // 销毁Sprite
Resources.UnloadAsset(texture);  // 卸载纹理
texture = null;
```

### 4.2 纹理压缩与内存

```
纹理格式内存占用：
├─ RGBA32：4MB/像素
├─ RGBA16：2MB/像素
├─ RGB24：3MB/像素
├─ ASTC 4x4：0.5MB/像素（压缩）
├─ ETC2：0.5MB/像素（压缩）
└─ PVRTC：0.5MB/像素（压缩）

建议：
✅ 使用压缩格式（ASTC/ETC2/PVRTC）
✅ 根据平台选择合适的压缩格式
✅ 大纹理使用mipmap
✅ 不需要时立即卸载
```

## 五、音频资源卸载

### 5.1 AudioClip卸载

```csharp
// 加载音频
AudioClip clip = Resources.Load<AudioClip>("Audio/BGM");

// 播放
AudioSource source = gameObject.AddComponent<AudioSource>();
source.clip = clip;
source.Play();

// 播放完毕后卸载
IEnumerator PlayAndUnload(AudioClip clip)
{
    AudioSource source = gameObject.AddComponent<AudioSource>();
    source.clip = clip;
    source.Play();

    // 等待播放完成
    yield return new WaitWhile(() => source.isPlaying);

    // 卸载
    source.clip = null;
    Destroy(source);
    Resources.UnloadAsset(clip);
}
```

---

### 5.2 音频压缩与内存

```
音频格式内存占用：
├─ WAV（未压缩）：10MB/分钟
├─ MP3（压缩）：1MB/分钟
├─ Vorbis（压缩）：1-2MB/分钟
└─ ADPCM（压缩）：4MB/分钟

建议：
✅ 使用压缩格式（Vorbis/MP3）
✅ 长音频使用流式加载（Load Type = Streaming）
✅ 短音频可以解压（Load Type = Decompress On Load）
✅ 不需要时立即卸载
```

## 六、GameObject和组件卸载

### 6.1 GameObject销毁

```csharp
// 创建对象
GameObject obj = Instantiate(prefab);

// 销毁对象
Destroy(obj);

// 立即销毁（危险，可能导致错误）
DestroyImmediate(obj);

// 延迟销毁
Destroy(obj, 2f);  // 2秒后销毁
```

---

### 6.2 卸载前的清理

```csharp
// ✅ 正确的销毁流程
void DestroyGameObject(GameObject obj)
{
    // 1. 停止协程
    StopAllCoroutines();

    // 2. 取消事件订阅
    var eventListeners = obj.GetComponentsInChildren<IEventSubscriber>();
    foreach (var listener in eventListeners)
    {
        listener.UnsubscribeAll();
    }

    // 3. 清理资源
    var cleanables = obj.GetComponentsInChildren<ICleanable>();
    foreach (var cleanable in cleanables)
    {
        cleanable.Cleanup();
    }

    // 4. 销毁对象
    Destroy(obj);
}
```

## 七、场景切换时的资源管理

### 7.1 场景卸载

```csharp
IEnumerator LoadSceneAsync(string sceneName)
{
    // 1. 卸载当前场景未使用的资源
    AsyncOperation unloadOp = Resources.UnloadUnusedAssets();
    yield return unloadOp;

    // 2. 强制GC（可选）
    System.GC.Collect();
    yield return new WaitForEndOfFrame();
    System.GC.WaitForPendingFinalizers();

    // 3. 卸载当前场景
    AsyncOperation unloadSceneOp = SceneManager.UnloadSceneAsync(SceneManager.GetActiveScene());
    yield return unloadSceneOp;

    // 4. 加载新场景
    AsyncOperation loadSceneOp = SceneManager.LoadSceneAsync(sceneName, LoadSceneMode.Single);
    yield return loadSceneOp;
}
```

---

### 7.2 多场景管理

```csharp
public class SceneManager : MonoBehaviour
{
    private List<Scene> loadedScenes = new List<Scene>();

    IEnumerator LoadSceneAdditive(string sceneName)
    {
        AsyncOperation loadOp = UnityEngine.SceneManagement.SceneManager.LoadSceneAsync(sceneName, LoadSceneMode.Additive);
        yield return loadOp;

        loadedScenes.Add(UnityEngine.SceneManagement.SceneManager.GetSceneByName(sceneName));
    }

    IEnumerator UnloadScene(string sceneName)
    {
        Scene scene = UnityEngine.SceneManagement.SceneManager.GetSceneByName(sceneName);

        // 卸载场景中的资源
        AsyncOperation unloadOp = UnityEngine.SceneManagement.SceneManager.UnloadSceneAsync(scene);
        yield return unloadOp;

        // 卸载未使用的资源
        yield return Resources.UnloadUnusedAssets();

        loadedScenes.Remove(scene);
    }
}
```

## 八、内存泄漏预防

### 8.1 常见泄漏原因

```
┌─────────────────────────────────────────────────────────────┐
│                    常见内存泄漏原因                            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│  1. 事件未取消订阅                                           │
│     ├─ 订阅了事件但忘记取消                                 │
│     ├─ 导致对象无法被GC回收                                  │
│     └─ 对象永久驻留内存                                     │
│                                                             │
│  2. 协程未停止                                               │
│     ├─ 销毁对象时协程仍在运行                               │
│     ├─ 对象引用被协程持有                                   │
│     └─ 对象无法被GC回收                                     │
│                                                             │
│  3. 静态集合持有引用                                         │
│     ├─ static List/Dictionary持有对象                        │
│     ├─ 对象永远不会被GC回收                                  │
│     └─ 内存持续增长                                         │
│                                                             │
│  4. 单例持有引用                                             │
│     ├─ 单例持有对象的引用                                    │
│     ├─ 对象无法被释放                                       │
│     └─ 内存泄漏                                             │
│                                                             │
│  5. DOTween未Kill                                            │
│     ├─ Tween对象持有目标引用                                 │
│     ├─ 目标对象无法被GC回收                                  │
│     └─ 内存泄漏                                             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

---

### 8.2 内存检查清单

```
□ 事件订阅配对取消
□ 协程在OnDestroy中停止
□ 静态集合有清理机制
□ 单例不持有对象引用
□ DOTween动画Kill
□ 资源使用完毕后卸载
□ 定期调用Resources.UnloadUnusedAssets
□ 场景切换时清理资源
□ 使用Memory Profiler检查
□ 测试长时间运行内存
```

## 九、资源管理最佳实践

### ✅ DO

1. **及时卸载不需要的资源**
   ```csharp
   Resources.UnloadAsset(texture);
   ```

2. **使用对象池复用**
   ```csharp
   var obj = objectPool.Get();
   // 使用完毕后
   objectPool.Return(obj);
   ```

3. **场景切换时清理**
   ```csharp
   Resources.UnloadUnusedAssets();
   ```

4. **异步卸载大资源**
   ```csharp
   yield return Resources.UnloadUnusedAssets();
   ```

5. **使用AssetBundle管理资源**
   ```csharp
   bundle.Unload(true);
   ```

### ❌ DON'T

1. **不要忘记卸载资源**
   ```csharp
   // ❌ 错误
   var texture = Resources.Load<Texture2D>("BigTexture");
   // 使用完毕后忘记卸载
   ```

2. **不要在Update中加载资源**
   ```csharp
   // ❌ 错误
   void Update()
   {
       Resources.Load<GameObject>("Prefab");  // 每帧加载
   }
   ```

3. **不要持有已销毁对象的引用**
   ```csharp
   // ❌ 错误
   GameObject obj = Instantiate(prefab);
   Destroy(obj);
   var pos = obj.transform.position;  // 错误：对象已销毁
   ```

4. **不要滥用静态集合**
   ```csharp
   // ❌ 错误
   public static List<GameObject> allObjects = new List<GameObject>();
   // 永不清理，导致内存泄漏
   ```
