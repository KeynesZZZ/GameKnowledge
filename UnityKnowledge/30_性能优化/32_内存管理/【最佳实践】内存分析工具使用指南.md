# 【最佳实践】内存分析工具使用指南

## 文档定位

Unity内存分析工具的完整使用指南，涵盖Memory Profiler安装与快照捕获对比、Profiler窗口内存模块指标解读（Total Allocated/GC Alloc/Used Heap）以及开发阶段和发布前的内存检查最佳实践。是进行内存泄漏排查前必须掌握的基础工具教程。

---

## 核心工具

```
Unity内存分析工具：
1. Memory Profiler
2. Profiler窗口
3. System.GC.Collect()
```

---

## 一、Memory Profiler

### 1.1 基本使用

```
步骤：
1. Window → Analysis → Memory Profiler
2. 打开Memory Profiler窗口
3. 点击"Capture"按钮捕获快照
4. 分析内存使用情况

功能：
├─ 查看总内存占用
├─ 查看各类型对象数量
├─ 查看引用关系
└─ 查找内存泄漏
```

---

### 1.2 快照对比

```
快照对比流程：
1. 捕获初始快照（Before）
2. 执行操作
3. 捕获对比快照（After）
4. 对比两个快照的差异

用途：
├─ 查找内存泄漏
├─ 验证资源释放
└─ 分析内存增长
```

---

## 二、常见内存问题诊断

### 2.1 内存泄漏排查

```
排查流程：
1. 打开Memory Profiler
2. 捕获快照A
3. 进入场景再退出
4. 捕获快照B
5. 对比快照A和B
6. 查找未释放的对象

常见泄漏：
├─ 事件未取消订阅
├─ 协程未停止
├─ 静态集合持有引用
└─ 单例持有对象引用
```

---

## 三、Profiler窗口

### 3.1 内存分析

```
Profiler → Memory：
├─ Total Allocated：总分配内存
├─ GC Alloc：GC分配
├─ Used Heap：堆内存使用
└─ GC.Collect：GC触发

监控指标：
├─ GC Allocations > 100KB/秒 → 检查代码
├─ GC触发频率 > 1次/秒 → 检查代码
└─ Total Allocated持续增长 → 可能泄漏
```
