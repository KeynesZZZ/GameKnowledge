# 踩坑记录 - 内存泄漏模式

> Unity常见内存泄漏模式及解决方案 `#内存管理` `#踩坑记录` `#性能优化`

## 概述

内存泄漏指不再使用的内存无法被回收。Unity中常见泄漏类型：

| 类型 | 来源 | 检测难度 |
|------|------|----------|
| 托管内存泄漏 | C#对象引用未释放 | 中等 |
| 原生内存泄漏 | Unity对象未销毁 | 较高 |
| GPU内存泄漏 | 纹理/网格未释放 | 高 |

---

## 模式一：事件订阅未取消

### 问题描述

订阅事件后未取消订阅，导致发布者持有订阅者引用，无法被GC回收。

### 泄漏代码

```csharp
public class Player : MonoBehaviour
{
    private void OnEnable()
    {
        EventBus.Subscribe<ScoreChangedEvent>(OnScoreChanged);
        // 忘记在OnDisable中取消订阅！
    }

    private void OnScoreChanged(ScoreChangedEvent e)
    {
        // 处理事件
    }
}

// 即使销毁Player对象，EventBus仍持有引用
Destroy(playerGameObject);  // 内存泄漏！
```

### 修复方案

```csharp
public class Player : MonoBehaviour
{
    private void OnEnable()
    {
        EventBus.Subscribe<ScoreChangedEvent>(OnScoreChanged);
    }

    private void OnDisable()
    {
        EventBus.Unsubscribe<ScoreChangedEvent>(OnScoreChanged);  // ✅
    }

    private void OnScoreChanged(ScoreChangedEvent e) { }
}
```

### 预防措施

```csharp
// 方案1: 使用IDisposable模式
public class SafeEventListener : MonoBehaviour, IDisposable
{
    private List<IDisposable> subscriptions = new();

    protected void Subscribe<T>(Action<T> handler) where T : struct
    {
        subscriptions.Add(DisposableEventBus.Subscribe(handler));
    }

    public void Dispose()
    {
        foreach (var sub in subscriptions)
            sub.Dispose();
        subscriptions.Clear();
    }

    private void OnDestroy() => Dispose();
}

// 方案2: 使用弱引用事件系统
// 方案3: 使用可视化检测工具
```

## 模式二：静态集合持续增长

### 问题描述

静态集合不断添加元素但从不清理，导致内存持续增长。

### 泄漏代码

```csharp
public static class ObjectManager
{
    private static List<GameObject> allObjects = new();

    public static void Register(GameObject go)
    {
        allObjects.Add(go);  // 持续添加
    }

    // 没有 Remove 方法！
}

// 每次创建对象都会泄漏
var go = Instantiate(prefab);
ObjectManager.Register(go);
Destroy(go);  // 列表中仍持有null引用
```

### 修复方案

```csharp
public static class ObjectManager
{
    private static List<GameObject> allObjects = new();

    public static void Register(GameObject go)
    {
        allObjects.Add(go);
    }

    public static void Unregister(GameObject go)
    {
        allObjects.Remove(go);  // ✅
    }

    // 定期清理null引用
    public static void CleanNulls()
    {
        allObjects.RemoveAll(x => x == null);
    }
}

// 使用示例
public class ManagedObject : MonoBehaviour
{
    private void OnEnable() => ObjectManager.Register(gameObject);
    private void OnDisable() => ObjectManager.Unregister(gameObject);
}
```

## 模式三：协程未正确停止

### 问题描述

协程持有MonoBehaviour引用，即使对象销毁，协程可能仍在运行。

### 泄漏代码

```csharp
public class Spawner : MonoBehaviour
{
    private void Start()
    {
        StartCoroutine(SpawnLoop());  // 没有保存引用
    }

    private IEnumerator SpawnLoop()
    {
        while (true)  // 无限循环
        {
            yield return new WaitForSeconds(1f);
            // 持有this引用
        }
    }
}

// 销毁对象时协程可能继续
Destroy(spawner);  // 协程停止，但如果协程中引用了其他对象？
```

### 修复方案

```csharp
public class Spawner : MonoBehaviour
{
    private Coroutine spawnCoroutine;

    private void OnEnable()
    {
        spawnCoroutine = StartCoroutine(SpawnLoop());
    }

    private void OnDisable()
    {
        if (spawnCoroutine != null)
        {
            StopCoroutine(spawnCoroutine);  // ✅
            spawnCoroutine = null;
        }
    }

    private IEnumerator SpawnLoop()
    {
        while (true)
        {
            yield return new WaitForSeconds(1f);
            // 使用null检查防止引用泄漏对象
            if (this == null) yield break;
        }
    }
}
```

## 模式四：闭包捕获this

### 问题描述

Lambda表达式捕获this引用，导致对象无法释放。

### 泄漏代码

```csharp
public class DownloadManager : MonoBehaviour
{
    private void Start()
    {
        StartCoroutine(DownloadFile("url", (data) =>
        {
            // 闭包捕获了this
            ProcessData(data);  // 持有this引用
        }));
    }

    private void ProcessData(byte[] data) { }
}

// 即使销毁DownloadManager，回调仍持有引用
```

### 修复方案

```csharp
public class DownloadManager : MonoBehaviour
{
    private bool isDestroyed;

    private void Start()
    {
        StartCoroutine(DownloadFile("url", OnDownloadComplete));
    }

    private void OnDestroy()
    {
        isDestroyed = true;  // ✅ 标记销毁
    }

    private void OnDownloadComplete(byte[] data)
    {
        if (isDestroyed) return;  // ✅ 检查销毁状态
        ProcessData(data);
    }

    private void ProcessData(byte[] data) { }
}

// 或者使用弱引用
public class WeakAction<T>
{
    private readonly WeakReference targetRef;
    private readonly Action<T> action;

    public WeakAction(object target, Action<T> action)
    {
        targetRef = new WeakReference(target);
        this.action = action;
    }

    public void Invoke(T param)
    {
        if (targetRef.IsAlive && targetRef.Target != null)
        {
            action(param);
        }
    }
}
```

## 模式五：资源未释放

### 问题描述

动态加载的资源（纹理、网格、AssetBundle）未正确释放。

### 泄漏代码

```csharp
public class TextureLoader : MonoBehaviour
{
    public void LoadTexture(string path)
    {
        var texture = Resources.Load<Texture>(path);
        // 使用texture
        // 忘记 Resources.UnloadAsset 或 UnloadUnusedAssets
    }
}

// AssetBundle泄漏
public class AssetBundleLoader : MonoBehaviour
{
    private void LoadAsset()
    {
        var bundle = AssetBundle.LoadFromFile("path");
        var asset = bundle.LoadAsset<GameObject>("Asset");
        // 忘记 bundle.Unload
    }
}
```

### 修复方案

```csharp
public class TextureLoader : MonoBehaviour
{
    private List<Texture> loadedTextures = new();

    public void LoadTexture(string path)
    {
        var texture = Resources.Load<Texture>(path);
        loadedTextures.Add(texture);
    }

    private void OnDestroy()
    {
        foreach (var tex in loadedTextures)
        {
            Resources.UnloadAsset(tex);  // ✅
        }
        loadedTextures.Clear();
    }
}

// AssetBundle正确释放
public class AssetBundleLoader : MonoBehaviour
{
    private AssetBundle currentBundle;

    public void LoadAsset()
    {
        currentBundle = AssetBundle.LoadFromFile("path");
        var asset = currentBundle.LoadAsset<GameObject>("Asset");
        Instantiate(asset);
    }

    private void OnDestroy()
    {
        if (currentBundle != null)
        {
            currentBundle.Unload(true);  // ✅ true = 同时卸载实例
            currentBundle = null;
        }
    }
}
```

## 模式六：定时器泄漏

### 问题描述

使用Invoke或InvokeRepeating但未取消。

### 泄漏代码

```csharp
public class Enemy : MonoBehaviour
{
    private void Start()
    {
        InvokeRepeating("UpdateAI", 0f, 0.5f);  // 没有保存引用
    }

    private void UpdateAI() { }

    // 忘记CancelInvoke
}
```

### 修复方案

```csharp
public class Enemy : MonoBehaviour
{
    private void OnEnable()
    {
        InvokeRepeating(nameof(UpdateAI), 0f, 0.5f);
    }

    private void OnDisable()
    {
        CancelInvoke();  // ✅ 取消所有Invoke
        // 或 CancelInvoke(nameof(UpdateAI));
    }

    private void UpdateAI() { }
}
```

## 模式七：DOTween动画泄漏

### 问题描述

DOTween动画未正确Kill导致回调持有引用。

### 泄漏代码

```csharp
public class UIButton : MonoBehaviour
{
    private void Start()
    {
        transform.DOScale(1.2f, 0.3f)
            .SetLoops(-1, LoopType.Yoyo)
            .OnComplete(() =>
            {
                // 闭包捕获this
                Debug.Log("Complete: " + name);
            });
    }

    // 销毁时动画仍在运行
}
```

### 修复方案

```csharp
public class UIButton : MonoBehaviour
{
    private Tween scaleTween;

    private void OnEnable()
    {
        scaleTween = transform.DOScale(1.2f, 0.3f)
            .SetLoops(-1, LoopType.Yoyo)
            .SetAutoKill(false);  // 手动控制
    }

    private void OnDisable()
    {
        if (scaleTween != null)
        {
            scaleTween.Kill();  // ✅
            scaleTween = null;
        }
    }
}

// 或使用DOTween的安全模式
DOTween.Init(true, true);  // safeMode = true
```

## 检测方法

### Memory Profiler

```
1. Package Manager > Memory Profiler
2. Window > Analysis > Memory Profiler
3. Capture Snapshot
4. 对比多个快照找出增长的对象
```

### 代码检测

```csharp
// 自定义泄漏检测器
public class LeakDetector : MonoBehaviour
{
    private static Dictionary<string, int> objectCounts = new();

    public static void Track(string typeName)
    {
        if (!objectCounts.ContainsKey(typeName))
            objectCounts[typeName] = 0;
        objectCounts[typeName]++;
    }

    public static void Untrack(string typeName)
    {
        if (objectCounts.ContainsKey(typeName))
            objectCounts[typeName]--;
    }

    [RuntimeInitializeOnLoadMethod]
    static void Initialize()
    {
        Application.logMessageReceived += (condition, stackTrace, type) =>
        {
            if (condition.Contains("Leak"))
            {
                Debug.LogWarning($"Potential leak detected: {condition}");
            }
        };
    }

    public static void PrintStats()
    {
        foreach (var kvp in objectCounts)
        {
            if (kvp.Value > 0)
                Debug.Log($"{kvp.Key}: {kvp.Value}");
        }
    }
}
```

## 泄漏模式速查表

| 模式 | 症状 | 检测方法 | 修复 |
|------|------|----------|------|
| 事件未取消订阅 | 对象销毁后内存不降 | Memory Profiler | OnDisable取消订阅 |
| 静态集合增长 | 内存持续增长 | 检查静态集合大小 | 添加Remove方法 |
| 协程未停止 | 销毁后仍有日志 | StopAllCoroutines | OnDisable停止协程 |
| 闭包捕获 | 对象不释放 | 检查Lambda | 使用弱引用或检查销毁 |
| 资源未释放 | 纹理/网格内存高 | Frame Debugger | UnloadAsset/Unload |
| 定时器泄漏 | 对象不释放 | CancelInvoke | OnDisable取消 |
| DOTween泄漏 | Tween继续运行 | DOTween.KillAll | OnDisable Kill |
