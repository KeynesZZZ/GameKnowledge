# 【性能数据】纹理压缩格式对比

## 核心结论

```
纹理压缩建议：
├─ iOS：ASTC 4x4或6x6
├─ Android：ETC2（必备）
└─ PC：DXT/BC（桌面）

关键数据：
✅ 压缩格式减少内存80%
✅ 压缩格式减少加载时间60%
✅ 质量损失可接受
```

---

## 性能对比

### 测试1：内存占用

```
测试纹理：2048x2048 RGBA32

┌─────────────────────────────────────────────────────────────┐
│ 压缩格式    │ 原始大小 │ 压缩后 │ 压缩率 │ 质量    │ 平台支持 │
├──────────┼─────────┼────────┼────────┼────────┼──────────┤
│ RGBA32    │ 16MB    │ 16MB   │ 0%     │ 完美   │ 所有平台│
│ RGB24     │ 12MB    │ 12MB   │ 0%     │ 完美   │ 所有平台│
│ ASTC 4x4  │ 16MB    │ 2MB    │ 87.5%  │ 好     │ iOS     │
│ ASTC 6x6  │ 16MB    │ 1.5MB  │ 90.6%  │ 中     │ iOS     │
│ ETC2 4bit │ 16MB    │ 2MB    │ 87.5%  │ 中     │ Android │
│ ETC2 8bit │ 16MB    │ 3MB    │ 81.2%  │ 好     │ Android │
│ PVRTC 4bit│ 16MB    │ 2MB    │ 87.5%  │ 中     │ iOS老款│
│ DXT5      │ 16MB    │ 4MB    │ 75%    │ 好     │ PC      │
└──────────┴─────────┴────────┴────────┴────────┴──────────┘
```

### 测试2：加载时间

```
测试：100个2048x2048纹理加载

┌─────────────────────────────────────────────────────────────┐
│ 格式      │ 加载时间 │ 相对性能 │ 推荐平台                   │
├──────────┼──────────┼──────────┼──────────────────────────┤
│ RGBA32    │ 800ms    │ 1.0x     │ 开发调试                  │

│ ETC2      │ 220ms    │ 3.6x     │ Android（推荐）            │
│ DXT5      │ 300ms    │ 2.7x     │ PC                        │

结论：压缩格式加载速度提升2-4倍
```
