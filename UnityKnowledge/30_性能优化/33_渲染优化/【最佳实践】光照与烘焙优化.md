# 【最佳实践】光照与烘焙优化

## 核心策略

```
光照优化原则：
1. 优先使用烘焙光照（Lightmap）
2. 减少实时光源数量
3. 移动平台禁用实时光照
4. 使用Light Probe
5. 合理使用Reflection Probe
```

## 二、烘焙光照优化

### 2.1 Lightmap优化

```
优化技巧：
1. 降低Lightmap分辨率
   ├─ 移动平台：1024-2048
   └─ PC平台：2048-4096

2. 使用纹理压缩
   ├─ ASTC/ETC2（移动）
   └─ DXT/BC（PC）

3. 合理设置Lightmap参数
   ├─ Atlas大小：适中
   ├─ 压缩：启用
   └─ Mipmaps：禁用

性能提升：
├─ 内存占用：减少60%
├─ 加载时间：减少40%
```

---

## 三、实时光源优化

```
优化策略：
1. 限制实时光源数量
   ├─ 移动平台：0-1个
   └─ PC平台：2-3个

2. 使用简单光照
   ├─ Vertex Lit（移动）
   └─ Unlit（无光照）

3. 优化阴影
   ├─ 减少阴影距离
   ├─ 降低阴影分辨率
   └─ 使用硬阴影

4. 烘焙静态物体
   ├─ 标记为Static
   ├─ 烘焙到Lightmap
   └─ 不参与实时光照
```

---

## 四、移动平台策略

```
移动平台光照配置：
✅ 完全烘焙光照
✅ 禁用实时光照
✅ 使用Light Probe
✅ 禁用实时阴影
❌ 避免实时GI
❌ 避免多个实时光源

性能提升：
├─ 移除实时光源：提升30-50%
├─ 禁用阴影：提升20-30%
└─ 使用Light Probe：无明显损失
```
