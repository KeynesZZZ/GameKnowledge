# 性能优化与发布 - 学习路径

> 模块四 | Unity高级开发者系统性学习计划

## 文档定位

本文档从**使用角度**讲解性能优化学习路径。

**相关文档**：、、[[【教程】性能优化学习路径]]

---

## 模块概述

本模块将系统讲解Unity性能优化的方法论和实操技巧，从分析工具使用到各层面的优化策略，最终实现游戏的流畅运行和成功发布。

---

## 学习目标

完成本模块后，你将能够：

1. ✅ 熟练使用性能分析工具定位瓶颈
2. ✅ 掌握CPU端优化技巧（GC、Job System）
3. ✅ 实现内存优化和泄漏排查
4. ✅ 优化渲染性能（DrawCall、Shader）
5. ✅ 完成游戏打包与热更新部署

## 性能优化方法论

```
┌─────────────────────────────────────────────────────────────┐
│                    性能优化工作流程                           │
│                                                             │
│   ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐ │
│   │  测量   │ -> │  分析   │ -> │  优化   │ -> │  验证   │ │
│   │Measure  │    │ Analyze │    │ Optimize│    │ Verify  │ │
│   └─────────┘    └─────────┘    └─────────┘    └─────────┘ │
│       ↑                                              │      │
│       └──────────────────────────────────────────────┘      │
│                      持续迭代                                │
└─────────────────────────────────────────────────────────────┘

关键原则：
1. 先测量，后优化（不要过早优化）
2. 优化最大瓶颈（80/20法则）
3. 设置性能预算
4. 持续监控回归
```

## 性能目标参考

### 三消+Roguelike项目目标

| 指标 | 目标值 | 底线值 | 说明 |
|------|--------|--------|------|
| 帧率 | 60 FPS | 30 FPS | 主流设备稳定 |
| 内存 | < 300 MB | < 500 MB | 峰值内存 |
| 包体 | < 100 MB | < 200 MB | 首包大小 |
| 启动时间 | < 3 秒 | < 5 秒 | 冷启动 |
| DrawCall | < 100 | < 200 | 游戏场景 |
| SetPassCall | < 20 | < 50 | 材质切换次数 |

## 优化检查清单

### 发布前检查

```markdown
## 性能检查清单

### CPU
- [ ] Profiler确认无GC Spike
- [ ] Update函数无空实现
- [ ] 协程使用yield优化
- [ ] 物理碰撞层级正确设置

### 内存
- [ ] 无内存泄漏（Memory Profiler验证）
- [ ] 纹理压缩格式正确
- [ ] 音频压缩设置合理
- [ ] 资源正确卸载

### 渲染
- [ ] DrawCall在预算内
- [ ] UI合批优化完成
- [ ] 粒子数量合理
- [ ] Shader变体数量控制

### 打包
- [ ] 未使用资源已清理
- [ ] 包体大小在预算内
- [ ] 首场景资源最小化
- [ ] 热更新流程测试通过
```
