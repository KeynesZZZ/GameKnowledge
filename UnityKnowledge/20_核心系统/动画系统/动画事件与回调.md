# 动画事件与回调

> Unity 动画事件和回调函数完整指南 `#动画系统` `#动画事件` `#回调函数`

## 概述

Unity 提供了多种动画事件和回调机制，允许在动画的关键帧或状态变化时执行代码。这对于同步视觉效果和逻辑至关重要。

## 动画事件（Animation Events）

### 1. 在 Inspector 中添加事件

```
1. 打开 Animation Window
2. 选择动画剪辑
3. 在时间线上选择关键帧
4. 点击 "Add Event" 按钮
5. 设置函数名称和参数
```

### 2. 代码中处理事件

```csharp
using UnityEngine;

public class AnimationEventHandler : MonoBehaviour
{
    // 无参数事件
    public void OnFootStep()
    {
        Debug.Log("Foot step");
        // 播放脚步声
    }

    // 有参数事件（float）
    public void OnDamage(float damage)
    {
        Debug.Log($"Damage: {damage}");
    }

    // 有参数事件（int）
    public void OnAttackCount(int count)
    {
        Debug.Log($"Attack count: {count}");
    }

    // 多参数事件
    public void OnCombo(int hitCount, float damage)
    {
        Debug.Log($"Combo: {hitCount} hits, {damage} damage");
    }
}
```

## Animator 回调

### 1. 状态机回调

```csharp
[RequireComponent(typeof(Animator))]
public class AnimatorStateCallbacks : MonoBehaviour
{
    public Animator animator;
    private string currentStateName;

    void Start()
    {
        animator = GetComponent<Animator>();
    }

    // 进入状态时调用
    void OnStateEnter(AnimatorStateInfo stateInfo, int layerIndex)
    {
        currentStateName = stateInfo.shortNameHash.ToString();
        Debug.Log($"Entered state: {currentStateName}");
        
        // 状态进入时的逻辑
        switch (currentStateName)
        {
            case "Idle":
                // 待机逻辑
                break;
            case "Run":
                // 跑步逻辑
                break;
        }
    }

    // 状态更新时调用（每帧）
    void OnStateUpdate(AnimatorStateInfo stateInfo, int layerIndex)
    {
        if (currentStateName == "Run")
        {
            // 跑步时的每帧逻辑
        }
    }

    // 退出状态时调用
    void OnStateExit(AnimatorStateInfo stateInfo, int layerIndex)
    {
        Debug.Log($"Exited state: {currentStateName}");
        
        // 状态退出时的清理
    }
}
```

## 实战示例

### 1. 脚步声音同步

```csharp
using UnityEngine;

public class FootstepAudio : MonoBehaviour
{
    public AudioClip[] footstepSounds;
    public float volume = 0.5f;

    private AudioSource audioSource;

    void Start()
    {
        audioSource = gameObject.AddComponent<AudioSource>();
        audioSource.volume = volume;
    }

    // 动画事件调用
    public void OnFootstep()
    {
        if (footstepSounds != null && footstepSounds.Length > 0)
        {
            AudioClip randomClip = footstepSounds[Random.Range(0, footstepSounds.Length)];
            audioSource.PlayOneShot(randomClip);
        }
    }
}
```

### 2. 攻击伤害事件

```csharp
using UnityEngine;

public class AttackDamage : MonoBehaviour
{
    public float baseDamage = 10f;

    // 动画事件调用
    public void OnAttackHit()
    {
        Collider[] hits = Physics.OverlapSphere(transform.position, 1.5f);
        
        foreach (Collider hit in hits)
        {
            IDamageable damageable = hit.GetComponent<IDamageable>();
            if (damageable != null)
            {
                damageable.TakeDamage(baseDamage);
            }
        }
    }
}

// 伤害接口
public interface IDamageable
{
    void TakeDamage(float damage);
}
```

### 3. 状态切换逻辑

```csharp
[RequireComponent(typeof(Animator))]
public class StateLogic : MonoBehaviour
{
    public Animator animator;

    private string previousState;

    void Start()
    {
        animator = GetComponent<Animator>();
        previousState = "";
    }

    void OnStateEnter(AnimatorStateInfo stateInfo, int layerIndex)
    {
        string newState = stateInfo.shortNameHash.ToString();
        
        // 状态切换时的逻辑
        if (previousState != newState)
        {
            OnStateChanged(previousState, newState);
            previousState = newState;
        }
    }

    void OnStateChanged(string oldState, string newState)
    {
        Debug.Log($"State changed: {oldState} -> {newState}");
        
        switch (newState)
        {
            case "Idle":
                OnIdleEnter();
                break;
            case "Run":
                OnRunEnter();
                break;
            case "Attack":
                OnAttackEnter();
                break;
        }
    }

    void OnIdleEnter()
    {
        // 恢复体力
    }

    void OnRunEnter()
    {
        // 消耗体力
    }

    void OnAttackEnter()
    {
        // 激活攻击判定
    }
}
```

### 4. 根运动处理

public class RootMotion : MonoBehaviour

## 性能优化

### 1. 减少动画事件

```csharp
// ❌ 错误：每帧调用
void Update()
{
    CheckFootsteps();
}

// ✅ 正确：使用动画事件
public void OnFootstep()
{
    PlayFootstepSound();
}
```

### 2. 使用状态机回调替代 Update 检查

```csharp
// ❌ 错误：每帧检查状态
void Update()
{
    AnimatorStateInfo state = animator.GetCurrentAnimatorStateInfo(0);
    if (state.shortNameHash == "Idle")
    {
        // 逻辑
    }
}

// ✅ 正确：使用状态机回调
void OnStateEnter(AnimatorStateInfo stateInfo, int layerIndex)
{
    if (state.shortNameHash.ToString() == "Idle")
    {
        // 逻辑
    }
}
```

## 最佳实践

### DO ✅

- 使用动画事件同步声音和效果
- 使用状态机回调管理状态逻辑
- 为动画事件使用有意义的函数名
- 在 OnStateExit 中清理状态相关的资源
- 使用根运动处理角色移动
- 减少 Update 中的状态检查

### DON'T ❌

- 不要在 Update 中频繁查询状态（使用回调）
- 不要使用过多的动画事件（性能问题）
- 不要忘记在状态退出时清理
- 不要混淆动画事件和状态机回调的使用场景
- 不要在动画事件中执行耗时操作

## 常见问题

### Q: 动画事件不触发？
**A**: 
1. 检查函数名是否匹配
2. 检查动画事件是否正确添加
3. 检查 GameObject 是否有对应的组件
4. 检查动画是否正在播放

### Q: 状态机回调不调用？
**A**: 
1. 检查 AnimatorController 是否正确配置
2. 检查状态名称是否匹配
3. 检查层级索引是否正确
4. 检查是否在正确的 GameObject 上
