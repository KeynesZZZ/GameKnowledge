## 相关链接

```csharp
// BinaryFormatter (System.Runtime.Serialization)
using System.Runtime.Serialization;
using System.IO;

[Serializable]
public class PlayerData
{
    public int level;
    public float health;
    public Vector3 position;
}

public class BinaryFormatterSave
{
    public void Save(string path, PlayerData data)
    {
        // 创建格式化器
        var formatter = new BinaryFormatter();
        using (var stream = new FileStream(path, FileMode.Create))
        {
            // 序列化到二进制
            formatter.Serialize(stream, data);
        }
    }

    public PlayerData Load(string path)
    {
        var formatter = new BinaryFormatter();
        using (var stream = new FileStream(path, FileMode.Open))
        {
            // 从二进制反序列化
            return (PlayerData)formatter.Deserialize(stream);
        }
    }
}

// protobuf (Google Protocol Buffers)
using ProtoBuf;

[ProtoContract]
public class PlayerData
{
    [ProtoMember(1)]
    public int level { get; set; }

    [ProtoMember(2)]
    public float health { get; set; }

    [ProtoMember(3)]
    public Vector3 position { get; set; }
}

public class ProtobufSave
{
    public void Save(string path, PlayerData data)
    {
        // 序列化到字节数组
        byte[] bytes = Serializer.Serialize(data);

        // 写入文件
        File.WriteAllBytes(path, bytes);
    }

    public PlayerData Load(string path)
    {
        // 从文件读取字节数组
        byte[] bytes = File.ReadAllBytes(path);

        // 反序列化
        return Serializer.Deserialize<PlayerData>(bytes);
    }
}

// Json.NET (Newtonsoft.Json)
using Newtonsoft.Json;

public class JsonSave
{
    public void Save(string path, PlayerData data)
    {
        // 序列化到JSON
        string json = JsonConvert.SerializeObject(data, Formatting.Indented);

        // 写入文件
        File.WriteAllText(path, json);
    }

    public PlayerData Load(string path)
    {
        // 从文件读取JSON
        string json = File.ReadAllText(path);

        // 反序列化
        return JsonConvert.DeserializeObject<PlayerData>(json);
    }
}
```

## 适用版本

- **Unity版本**: 2019.4 LTS+, 2020.3 LTS+, 2021.3 LTS+, 2022.3 LTS+, 2023.2 LTS+
- **序列化方案**:
  - **BinaryFormatter**: System.Runtime.Serialization (内置)
  - **protobuf-net**: Package (v3.x推荐)
  - **Json.NET**: Package (Newtonsoft.Json 13.0.0+)
  - **System.Text.Json**: .NET Core 3.0+ (Unity 2021.2+)
- **平台**: Windows, macOS, iOS, Android, WebGL
- **兼容性说明**:
  - BinaryFormatter: Unity 2018.3+ 完全支持
  - protobuf-net: Unity 2019.1+ 支持，需IL2CPP
  - Json.NET: Unity 2018.1+ 支持
  - System.Text.Json: Unity 2021.2+ 支持（推荐）
- **注意**: 本文档对比基于Unity 2022.3 LTS测试验证

## 序列化机制对比

### 1. BinaryFormatter

```
BinaryFormatter机制:

┌─────────────────────────────────────────────┐
│            BinaryFormatter                 │
├─────────────────────────────────────────────┤
│                                                      │
│  数据类型                                        │
│  ├─> 基本类型 (int, float, string)              │
│  ├─> 结构体 (struct)                            │
│  ├─> 类 (class, 需[Serializable])              │
│  ├─> 集合 (List, Dictionary)                     │
│  └─> Unity类型 (Vector3, Quaternion)             │
│                                                      │
│  格式                                              │
│  ├─> 二进制 (Binary)                              │
│  ├─> 元数据 (类型+数据)                          │
│  ├─> 引用处理 (自动)                             │
│  └─> 类型信息嵌入                                │
│                                                      │
│  优势                                              │
│  ├─> 性能好 (直接内存映射）                        │
│  ├─> Unity内置支持                                  │
│  ├─> 支持循环引用                                 │
│  └─> 类型安全                                       │
│                                                      │
│  劣势                                              │
│  ├─> 不跨平台 (不同版本兼容性差）               │
│  ├─> 二进制不可读                                   │
│  ├─> 反射开销 (运行时反射）                         │
│  └─> 文件体积大                                     │
│                                                      │
└─────────────────────────────────────────────┘

文件格式:
┌────────┬─────────┬─────────┬──────────┬─────────┐
│ TypeID │ Length  │ Type    │ Data     │  Padding  │
├────────┼─────────┼─────────┼──────────┼─────────┤
│ 0x01   │ 4 bytes │ 0x0C... │ 0x00...  │ 0x00...  │
└────────┴─────────┴─────────┴──────────┴─────────┘
```

### 2. protobuf-net

```
protobuf-net机制:

┌─────────────────────────────────────────────┐
│            protobuf-net                      │
├─────────────────────────────────────────────┤
│                                                      │
│  数据类型 (通过.proto定义)                      │
│  ├─> 基本类型 (int32, float, string)              │
│  ├─> 枚举 (enum)                                    │
│  ├─> 消息 (message)                                 │
│  ├─> 重复字段 (repeated)                            │
│  └─> 嵌套消息 (nested)                               │
│                                                      │
│  格式                                              │
│  ├─> 二进制 (Binary)                              │
│  ├─> Tag-Length-Value (TLV)                        │
│  ├─> Varint (变长整数)                              │
│  ├─> ZigZag (变长编码)                             │
│  └─> 压缩 (可选)                                    │
│                                                      │
│  优势                                              │
│  ├─> 性能最优 (零反射)                             │
│  ├─> 文件体积小                                      │
│  ├─> 跨平台兼容                                      │
│  ├─> 支持版本演进 (proto文件)                     │
│  └─> 支持向后兼容                                    │
│                                                      │
│  劣势                                              │
│  ├─> 需要.proto文件定义                              │
│  ├─> 不支持循环引用                                   │
│  ├─> 学习曲线陡峭                                    │
│  └─> 需要代码生成                                      │
│                                                      │
└─────────────────────────────────────────────┘

文件格式:
┌────────┬─────────┬─────────┬──────────┬─────────┐
│ Field  │ WireType│ Tag      │ Value     │  Padding  │
├────────┼─────────┼─────────┼──────────┼─────────┤
│ field1 │ 0x02    │ 0x08    │ 0x000008 │          │
│ field2 │ 0x00    │ 0x03    │ 0x00...  │ 0x00...  │
│ field3 │ 0x00    │ 0x05    │ 0x...    │          │
└────────┴─────────┴─────────┴──────────┴─────────┘
```

## 详细对比分析

### 1. BinaryFormatter

#### 优势
```csharp

public class SaveData
{
    // 无需额外配置，直接序列化

// ✅ 优势2: 支持复杂对象
[Serializable]
public class ComplexData
{
    public List<GameObject> objects;
    public Dictionary<string, int> scores;

// ✅ 优势3: 支持循环引用
[Serializable]
public class CircularReference
{
    public CircularReference next;
}

// ✅ 优势4: 支持版本兼容
[Serializable]
public class VersionedData
{
    public int version = 1;
    public VersionedData nextVersion;
}
```

#### 劣势与解决方案
```csharp
// ❌ 劣势1: 二进制不可读
// 解决: 调试时使用JSON格式

// ❌ 势势2: 文件体积大
// 解决: 使用GZip压缩
using System.IO.Compression;

public static byte[] Compress(byte[] data)
{
    using (var output = new MemoryStream())
    using (var gzip = new GZipStream(output, CompressionMode.Compress))
    {
        gzip.Write(data, 0, data.Length);
        gzip.Close();
        return output.ToArray();
    }
}

// ❌ 势势3: 版本兼容性差
// 解决: 实现版本化序列化
public interface IVersioned
{
    int Version { get; }
    bool CanDeserialize(int version);
}
```

### 3. Json.NET

#### 优势
```csharp
// ✅ 优势1: 可读性好
public class SaveData
{
    public int level;
    public float health;
    public Vector3 position;
}

string json = JsonConvert.SerializeObject(data, Formatting.Indented);
// 结果: 可读的JSON格式

// ✅ 优势2: 跨平台兼容
// JSON是标准格式，所有平台都支持

// ✅ 优势3: 支持类型推断
var data = new { level = 5, health = 100 };
var json = JsonConvert.SerializeObject(data);

// ✅ 优势4: 支持忽略null属性
public class SaveData
{
    public int level;
    public string name = null;
}

var settings = new JsonSerializerSettings
{
    NullValueHandling = NullValueHandling.Ignore
};
string json = JsonConvert.SerializeObject(data, settings);
// name字段将被忽略

// ✅ 优势5: 灵活支持
public class SaveData
{
    [JsonProperty("player_level")]
    public int level;
}

string json = JsonConvert.SerializeObject(data);
// JSON字段名为 "player_level"
```

#### 劣势与解决方案
```csharp
// ❌ 势势1: 性能较差
// 解决: 使用压缩格式
public static string ToJsonCompressed(object obj)
{
    string json = JsonConvert.SerializeObject(obj);
    byte[] bytes = Encoding.UTF8.GetBytes(json);
    bytes = Compress(bytes);
    return Convert.ToBase64String(bytes);
}

// ❌ 势势2: 文件体积大
// 解决: 移除空格和缩进
var json = JsonConvert.SerializeObject(data, Formatting.None);

// ❌ 势势3: 内存占用高
// 解决: 使用流式解析
using (var stream = new FileStream(path, FileMode.Open))
using (var reader = new StreamReader(stream))
{
    var data = JsonConvert.DeserializeObject<SaveData>(reader.ReadToEnd());
}

// ❌ 势势4: 不支持循环引用
// 解决: 保留引用ID或手动展平
public class SaveData
{
    public int id;
    public int parentId;  // 引用ID
}
```

// 解决: 封装常用方法
public static class JsonSerializer
{
    public static string Serialize<T>(T obj)
    {
        return System.Text.Json.JsonSerializer.Serialize(obj);
    }

    public static T Deserialize<T>(string json)
    {
        return System.Text.Json.JsonSerializer.Deserialize<T>(json);

    WriteIndented = true

│  ├─> 是 → protobuf-net
│  │
│  └─> 否 → 继续
│

### 推荐方案

| 场景 | 推荐方案 | 备选方案 |
|------|----------|----------|
| **开发阶段** | Json.NET | System.Text.Json |
| **移动端** | protobuf-net + GZip | BinaryFormatter + GZip |
| **大容量存档** | protobuf-net | BinaryFormatter |
| **跨平台** | protobuf-net | Json.NET |
| **简单存档** | Json.NET | BinaryFormatter |
| **复杂对象** | BinaryFormatter | Json.NET |
| **需要编辑** | Json.NET | System.Text.Json (Readable) |
| **性能优先** | protobuf-net | System.Text.Json |

## 最佳实践

### DO ✅

- 使用protobuf-net处理大容量存档
- 使用Json.NET处理需要可读性的存档
- 使用GZip压缩存档文件
- 预分配缓冲区减少GC
- 使用流式序列化减少内存
- 支持版本兼容和演进
- 使用加密保护敏感数据
- 使用校验和验证存档完整性

### DON'T ❌

- 不要在Update中序列化
- 不要忽略存档错误处理
- 不要忘记版本兼容性
- 不要在移动端使用大JSON文件
- 不要忽略文件体积
- 不要使用不支持循环引用的方案
- 不要忽略加密需求
- 不要忽略平台差异
