# 事件系统设计

> 零GC事件系统，基于接口回调实现

## 一、设计目标

- **零GC分配**：订阅/触发事件无堆分配
- **类型安全**：泛型约束，编译期检查
- **高性能**：直接数组遍历，无虚表开销
- **易用性**：接口实现简单直观

## 二、GC分配分析

| 操作 | Handler是class | Handler是struct |
|------|---------------|-----------------|
| Register(handler) | 零GC | 装箱，有GC |
| Unregister(handler) | 零GC | 装箱，有GC |
| Publish(eventData) | 零GC | 零GC |
| 事件数据传递 | 零GC（struct） | 零GC（struct） |
| 数组扩容 | 有GC（一次性） | 有GC（一次性） |

**结论：Handler必须用class（如MonoBehaviour）才能实现真正的零GC**

## 三、核心代码

### 3.1 事件接口定义

```csharp
/// <summary>
/// 事件标记接口，所有事件必须实现此接口
/// 事件必须是struct以实现零GC
/// </summary>
public interface IEvent { }

/// <summary>
/// 可取消事件接口
/// </summary>
public interface ICancelableEvent : IEvent
{
    bool IsCanceled { get; set; }
}

/// <summary>
/// 事件处理器接口（只能由class实现）
/// </summary>
public interface IEventHandler<T> where T : struct, IEvent
{
    void OnEvent(T eventData);
}
```

### 3.2 EventBus核心实现

```csharp
/// <summary>
/// 类型安全的事件总线
/// 零GC前提：Handler是class，数组已预分配
/// </summary>
public static class EventBus<T> where T : struct, IEvent
{
    // 预分配数组
    private const int INITIAL_CAPACITY = 64;
    private static IEventHandler<T>[] _handlers = new IEventHandler<T>[INITIAL_CAPACITY];
    private static int _count;
    private static int _capacity = INITIAL_CAPACITY;

    /// <summary>
    /// 初始化（启动时调用，预分配足够容量）
    /// </summary>
    public static void Initialize(int capacity = 64)
    {
        _capacity = capacity;
        _handlers = new IEventHandler<T>[capacity];
        _count = 0;
    }

    /// <summary>
    /// 订阅事件（零GC）
    /// </summary>
    public static void Register(IEventHandler<T> handler)
    {
#if UNITY_EDITOR
        // 编辑器下检测struct装箱
        if (handler.GetType().IsValueType)
        {
            Debug.LogWarning($"[EventBus] Struct handler will cause boxing: {typeof(T)}");
        }
#endif

        if (_count >= _capacity)
        {
            // 扩容（应通过预分配避免）
            _capacity *= 2;
            Array.Resize(ref _handlers, _capacity);
        }
        _handlers[_count++] = handler;
    }

    /// <summary>
    /// 取消订阅（零GC）
    /// </summary>
    public static void Unregister(IEventHandler<T> handler)
    {
        for (int i = 0; i < _count; i++)
        {
            if (ReferenceEquals(_handlers[i], handler))
            {
                _handlers[i] = _handlers[--_count];
                _handlers[_count] = default;
                return;
            }
        }
    }

    /// <summary>
    /// 触发事件（零GC）
    /// </summary>
    [MethodImpl(MethodImplOptions.AggressiveInlining)]
    public static void Publish(T eventData)
    {
        for (int i = 0; i < _count; i++)
        {
            _handlers[i].OnEvent(eventData);
        }
    }

    /// <summary>
    /// 清空所有订阅
    /// </summary>
    public static void Clear()
    {
        Array.Clear(_handlers, 0, _count);
        _count = 0;
    }

    /// <summary>
    /// 当前订阅者数量
    /// </summary>
    public static int Count => _count;
}
```

### 3.3 统一入口（可选）

```csharp
/// <summary>
/// 事件总线统一入口
/// </summary>
public static class EventBus
{
    public static void Register<T>(IEventHandler<T> handler) where T : struct, IEvent
    {
        EventBus<T>.Register(handler);
    }

    public static void Unregister<T>(IEventHandler<T> handler) where T : struct, IEvent
    {
        EventBus<T>.Unregister(handler);
    }

    public static void Publish<T>(T eventData) where T : struct, IEvent
    {
        EventBus<T>.Publish(eventData);
    }

    public static void Initialize<T>(int capacity = 64) where T : struct, IEvent
    {
        EventBus<T>.Initialize(capacity);
    }
}
```

## 四、事件定义示例

```csharp
/// <summary>
/// 玩家血量变化事件
/// </summary>
public struct PlayerHpChangedEvent : IEvent
{
    public int PlayerId;
    public int OldHp;
    public int NewHp;
    public int MaxHp;

    public float HpPercent => (float)NewHp / MaxHp;

    public static PlayerHpChangedEvent Create(int playerId, int oldHp, int newHp, int maxHp)
        => new() { PlayerId = playerId, OldHp = oldHp, NewHp = newHp, MaxHp = maxHp };
}

/// <summary>
/// 金币变化事件
/// </summary>
public struct CoinChangedEvent : IEvent
{
    public long OldAmount;
    public long NewAmount;
    public long Delta => NewAmount - OldAmount;

    public static CoinChangedEvent Create(long oldAmount, long newAmount)
        => new() { OldAmount = oldAmount, NewAmount = newAmount };
}

/// <summary>
/// 关卡完成事件（可取消）
/// </summary>
public struct LevelCompleteEvent : ICancelableEvent
{
    public int LevelId;
    public int Stars;
    public int Score;

    public bool IsCanceled { get; set; }

    public static LevelCompleteEvent Create(int levelId, int stars, int score)
        => new() { LevelId = levelId, Stars = stars, Score = score };
}

/// <summary>
/// UI面板事件
/// </summary>
public struct UIOpenedEvent : IEvent
{
    public string PanelName;
    public object UserData;

    public static UIOpenedEvent Create(string panelName, object data = null)
        => new() { PanelName = panelName, UserData = data };
}

/// <summary>
/// 游戏状态事件
/// </summary>
public struct GameStateChangedEvent : IEvent
{
    public GameState OldState;
    public GameState NewState;

    public static GameStateChangedEvent Create(GameState oldState, GameState newState)
        => new() { OldState = oldState, NewState = newState };
}
```

## 五、使用示例

### 5.1 基本使用

```csharp
// 订阅者：实现接口
public class PlayerUI : MonoBehaviour,
    IEventHandler<PlayerHpChangedEvent>,
    IEventHandler<CoinChangedEvent>
{
    [SerializeField] private Slider _hpBar;
    [SerializeField] private TMP_Text _hpText;
    [SerializeField] private TMP_Text _coinText;

    private void OnEnable()
    {
        // 订阅（传递this，零GC）
        EventBus<PlayerHpChangedEvent>.Register(this);
        EventBus<CoinChangedEvent>.Register(this);
    }

    private void OnDisable()
    {
        // 取消订阅（零GC）
        EventBus<PlayerHpChangedEvent>.Unregister(this);
        EventBus<CoinChangedEvent>.Unregister(this);
    }

    // 实现接口方法
    public void OnEvent(PlayerHpChangedEvent e)
    {
        _hpBar.value = e.HpPercent;
        _hpText.text = $"{e.NewHp}/{e.MaxHp}";
    }

    public void OnEvent(CoinChangedEvent e)
    {
        _coinText.text = e.NewAmount.ToString("N0");
    }
}
```

### 5.2 触发事件

```csharp
public class Player : MonoBehaviour
{
    private int _hp;
    private int _maxHp = 100;
    private long _coin;

    public void TakeDamage(int damage)
    {
        int oldHp = _hp;
        _hp = Mathf.Max(0, _hp - damage);

        // 触发事件（零GC）
        EventBus<PlayerHpChangedEvent>.Publish(
            PlayerHpChangedEvent.Create(1, oldHp, _hp, _maxHp)
        );
    }

    public void AddCoin(long amount)
    {
        long oldAmount = _coin;
        _coin += amount;

        EventBus<CoinChangedEvent>.Publish(
            CoinChangedEvent.Create(oldAmount, _coin)
        );
    }
}
```

### 5.3 可取消事件

```csharp
// 触发者
public class LevelManager : MonoBehaviour
{
    public void CompleteLevel(int stars, int score)
    {
        var e = LevelCompleteEvent.Create(1, stars, score);

        // 发布事件，让其他系统有机会取消
        EventBus<LevelCompleteEvent>.Publish(e);

        if (!e.IsCanceled)
        {
            SaveProgress();
            ShowReward();
        }
    }
}

// 订阅者：验证逻辑
public class LevelValidator : MonoBehaviour, IEventHandler<LevelCompleteEvent>
{
    private void OnEnable() => EventBus<LevelCompleteEvent>.Register(this);
    private void OnDisable() => EventBus<LevelCompleteEvent>.Unregister(this);

    public void OnEvent(LevelCompleteEvent e)
    {
        // 取消无效的关卡完成
        if (e.Score < 0 || e.Stars > 3)
        {
            e.IsCanceled = true;
        }
    }
}
```

## 七、设计要点总结

| 要点 | 说明 |
|------|------|
| 事件数据 | 必须是struct，值类型无堆分配 |
| 事件处理器 | 必须是class，避免接口装箱 |
| 订阅/取消 | 在OnEnable/OnDisable中成对调用 |
| 数组预分配 | 启动时初始化足够容量 |
| 静态工厂 | 事件Create方法简化创建 |

## 八、与Action方案对比

| 特性 | 接口方案 | Action方案 |
|------|---------|-----------|
| 订阅GC | 零（class） | 有（委托创建） |
| 触发GC | 零 | 零 |
| 代码量 | 多（需实现接口） | 少 |
| 灵活性 | 低（固定方法名） | 高 |
| 热更友好 | 一般（接口限制） | 好 |
| 适用场景 | 高频事件 | 低频/热更代码 |

## 九、注意事项

1. **Handler必须是class**：struct实现接口会产生装箱
2. **成对订阅/取消**：避免内存泄漏
3. **预分配容量**：避免运行时数组扩容
4. **不要在事件处理中订阅/取消同类事件**：可能导致遍历问题
5. **编辑器检测**：开发时开启装箱警告
