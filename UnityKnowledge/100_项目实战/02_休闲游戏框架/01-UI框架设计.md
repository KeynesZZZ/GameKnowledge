# UI框架设计

> 适用于Unity 2022.3 LTS，支持iOS/Android/小游戏多平台

## 一、设计目标

- **热更友好**：View和Presenter均为纯C#，可热更
- **Prefab纯净**：不挂载任何脚本，节点变更无需重新打包
- **代码生成**：自动生成View绑定代码，Transform.Find动态绑定
- **复杂动画**：支持序列/并行/嵌套的组合动画
- **可视化预览**：编辑器工具实时预览动画效果

## 四、核心代码

### 4.1 UIViewBase - View基类

```csharp
/// <summary>
/// UI视图基类（纯C#，可热更）
/// 通过Transform.Find动态绑定组件
/// </summary>
public abstract class UIViewBase
{
    public string ViewName { get; protected set; }
    public GameObject GameObject { get; protected set; }
    public RectTransform Transform { get; protected set; }

    // 动画配置（可在Presenter中覆盖）
    public UIAnimConfig ShowAnimConfig { get; set; }
    public UIAnimConfig HideAnimConfig { get; set; }

    // 动画相关组件
    protected CanvasGroup _canvasGroup;
    public CanvasGroup CanvasGroup
    {
        get
        {
            if (_canvasGroup == null)
                _canvasGroup = GameObject.GetOrAddComponent<CanvasGroup>();
            return _canvasGroup;
        }
    }

    // 记录原始位置（用于滑动动画恢复）
    private Vector2 _originalPos;

    /// <summary>
    /// 初始化入口
    /// </summary>
    public void Initialize(GameObject go)
    {
        GameObject = go;
        Transform = go.transform as RectTransform;
        ViewName = go.name;
        _originalPos = Transform.anchoredPosition;
        BindComponents();
        OnInitialized();
    }

    /// <summary>
    /// 子类实现：绑定组件
    /// </summary>
    protected abstract void BindComponents();

    /// <summary>
    /// 可选的初始化回调
    /// </summary>
    protected virtual void OnInitialized() { }

    /// <summary>
    /// 核心查找方法
    /// </summary>
    protected T Find<T>(string path) where T : Component
    {
        var trans = Transform.Find(path);
        if (trans == null)
        {
            Debug.LogWarning($"[UIView] 路径未找到: {path}");
            return null;
        }
        return trans.GetComponent<T>();
    }

    /// <summary>
    /// 便捷方法
    /// </summary>
    protected T Find<T>() where T : Component => Transform.GetComponent<T>();

    public void SetVisible(bool visible) => GameObject.SetActive(visible);

    public virtual void Dispose()
    {
        GameObject = null;
        Transform = null;
    }
}
```

### 4.2 UIPresenter - Presenter基类

```csharp
/// <summary>
/// UI表现层基类（纯C#，可热更）
/// MVP模式中的P，负责业务逻辑
/// </summary>
public abstract class UIPresenter<TView> where TView : UIViewBase, new()
{
    protected TView View { get; private set; }
    protected GameObject GameObject => View?.GameObject;
    protected RectTransform Transform => View?.Transform;

    public bool IsShowing { get; protected set; }

    // 是否播放动画
    protected bool PlayAnimation { get; set; } = true;

    /// <summary>
    /// 绑定View
    /// </summary>
    public void BindView(GameObject go)
    {
        View = new TView();
        View.Initialize(go);
        OnBindView();
    }

    protected virtual void OnBindView() { }

    // 生命周期
    public virtual UniTask OnCreate() => UniTask.CompletedTask;
    public virtual UniTask OnShowAsync(object data = null) => UniTask.CompletedTask;
    public virtual UniTask OnHideAsync() => UniTask.CompletedTask;

    public virtual void OnDestroy()
    {
        View?.Dispose();
        View = null;
    }

    // 覆盖动画配置
    protected void SetShowAnim(UIAnimConfig config) => View.ShowAnimConfig = config;
    protected void SetHideAnim(UIAnimConfig config) => View.HideAnimConfig = config;

    // 内部调用
    public async UniTask InternalShowAsync(object data)
    {
        IsShowing = true;

        if (PlayAnimation)
        {
            await UIAnimator.PlayShowAsync(View.Transform, View.ViewName, View.ShowAnimConfig);
        }
        else
        {
            View.SetVisible(true);
        }

        await OnShowAsync(data);
    }

    public async UniTask InternalHideAsync()
    {
        await OnHideAsync();
        IsShowing = false;

        if (PlayAnimation)
        {
            await UIAnimator.PlayHideAsync(View.Transform, View.ViewName, View.HideAnimConfig);
        }
        else
        {
            View.SetVisible(false);
        }
    }
}
```

### 5.1 动画节点类型

### 5.2 动画类型

| 分类 | 动画类型 | 说明 |
|------|---------|------|
| **透明度** | Fade, Alpha | CanvasGroup/Image透明度 |
| **缩放** | Scale, ScaleX/Y/Z, PunchScale, ShakeScale | 大小变化 |
| **移动** | Move, MoveX/Y, MoveFrom, PunchPosition, ShakePosition | 位置变化 |
| **旋转** | Rotate, RotateX/Y/Z, PunchRotation, ShakeRotation | 角度变化 |
| **颜色** | Color, Alpha, Gradient | 颜色渐变 |
| **文字** | TextColor, TextAlpha, TextFontSize, TextMaxVisible, TextCount | 文字特效 |
| **进度** | FillAmount, SliderValue | 进度条/滑动条 |
| **路径** | Path, PathConstantSpeed | 沿路径移动 |
| **材质** | MaterialColor, MaterialFloat | 材质属性 |
| **控制** | Callback, Delay | 回调/延迟 |

## 九、设计决策总结

| 模块 | 方案 | 理由 |
|------|------|------|
| View | 不继承Mono，代码生成 | 可热更，Prefab纯净 |
| Presenter | 纯C# | 业务逻辑可热更 |
| Prefab | 纯净，无脚本 | 节点变更无需重打包 |
| 动画 | 混合方案（预设 + 代码） | 策略可调，特殊效果可代码 |
| 缓存 | 只缓存常用面板 | 内存与性能平衡 |
| 导航 | Tab + Stack混合 | 适应休闲游戏场景 |
| 预览 | 编辑器工具 | 提高开发效率 |

## 十、热更场景处理

| 场景 | 处理方式 |
|------|---------|
| 节点路径变化 | 重新扫描Prefab → 重新生成View代码 → 热更DLL |
| 新增节点 | 按命名约定命名 → 重新生成 → 热更DLL |
| 组件类型变化 | 重新生成 → 热更DLL |
| 动画参数调整 | 预设资源热更 或 代码热更 |
| 业务逻辑修改 | 热更DLL |
