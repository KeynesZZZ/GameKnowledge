## 文档规范

### 项目结构

每个项目子目录应包含：

```
01-项目名称/
├── README.md              ← 项目概述与快速开始
├── 01-需求分析与架构.md    ← 需求、技术选型、架构图
├── 02-核心实现A.md        ← 核心模块实现
├── 03-核心实现B.md        ← 核心模块实现
├── ...
└── N-集成与部署.md        ← 完整集成示例
```

### 代码规范

- 所有代码可直接运行
- 包含完整的命名空间
- 使用#region组织大段代码
- 关键逻辑有中文注释
- 提供使用示例
