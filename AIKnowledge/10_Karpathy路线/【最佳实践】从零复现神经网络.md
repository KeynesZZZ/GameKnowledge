# 从零复现神经网络

> 复现的重点不是把原项目抄一遍，而是把“为什么这样写”拆成可验证的小实验。

## 操作原则

| 原则 | 做法 |
|------|------|
| 最小闭环 | 先让 20-50 行代码产生一个可观察结果 |
| 单变量修改 | 每次只改一个超参或模块 |
| 形状优先 | 所有核心张量先写 shape，再写语义 |
| 对照验证 | 用 PyTorch 或原项目结果校验自己的实现 |
| 记录失败 | loss 不降、梯度为 0、生成异常都要沉淀 |

## 复现流程

```mermaid
flowchart LR
    A["阅读目标代码"] --> B["摘出最小概念"]
    B --> C["写伪代码"]
    C --> D["实现最小版本"]
    D --> E["跑通样例"]
    E --> F["对照原实现"]
    F --> G["记录差异和问题"]
```

## micrograd 复现检查点

- [ ] `Value` 能保存 `data`、`grad`、`_prev`、`_backward`。
- [ ] 加法、乘法、幂、`relu` 都能建立计算图。
- [ ] `backward()` 能按拓扑序反向传播。
- [ ] 简单表达式的梯度能和手算结果一致。
- [ ] MLP 参数能被收集并用 SGD 更新。

## 语言模型复现检查点

- [ ] 字符到 id、id 到字符的映射稳定。
- [ ] 输入 `x` 和目标 `y` 的时间偏移正确。
- [ ] logits 最后一维等于词表大小。
- [ ] 交叉熵 loss 的 batch/time 维度处理正确。
- [ ] 训练 loss 下降后，采样文本比随机输出更有结构。

## Transformer 复现检查点

- [ ] Attention mask 不允许当前位置看到未来 token。
- [ ] Q/K/V 的 shape 和 head 拆分逻辑清楚。
- [ ] 残差连接前后的 shape 一致。
- [ ] LayerNorm 的归一化维度是 channel 维。
- [ ] 训练和生成共用同一个模型 forward 逻辑。

## 失败排查

| 现象 | 可能原因 | 排查动作 |
|------|----------|----------|
| loss 完全不降 | 标签错位、学习率不合适、参数没更新 | 打印 `x/y` 样例、梯度范数、参数前后差异 |
| loss 变成 NaN | 学习率过大、softmax 数值不稳定 | 降低学习率，检查 logits 范围 |
| 生成全是重复字符 | 训练不足、采样温度太低、数据太小 | 增加训练步数，调整 temperature/top-k |
| 训练很慢 | batch/context/model 过大 | 先用极小配置验证逻辑 |
