# AI知识库元数据规范

> 用最小元数据保证 Obsidian 检索、RAG 同步和长期维护可用，避免为了填字段而不写笔记。

## 必填字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `title` | string | 带文档类型前缀的标题 | `【最佳实践】Prompt与上下文管理` |
| `tags` | array | 至少 2 个标签 | `[AI编码, Prompt, 上下文管理]` |
| `created` | date | 创建日期 | `2026-06-17` |
| `description` | string | 一句话摘要 | `AI辅助开发中的上下文组织与Prompt管理方法` |

## 可选字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `category` | string | 分类路径 | `工作流/Prompt` |
| `updated` | date | 重要更新日期 | `2026-06-17` |
| `tools` | array | 涉及工具 | `[Codex, Claude Code, Cursor]` |
| `model_scope` | string | 适用模型范围 | `通用大语言模型` |
| `complexity` | string | 复杂度 | `简单` / `中等` / `复杂` |
| `use_frequency` | string | 使用频率 | `高频` / `中频` / `低频` |
| `status` | string | 文档状态 | `草稿` / `待验证` / `已归档` |
| `author` | string | 作者来源 | `human` / `llm` |
| `sources` | array | 结论来源（综述必填） | `["【最佳实践】Prompt与上下文管理"]` |

## 分类路径

| 一级分类 | 二级分类示例 |
|----------|--------------|
| `元数据与模板` | `规范`, `模板`, `标签体系` |
| `Karpathy路线` | `学习路径`, `复现`, `源码` |
| `LLM基础` | `Tokenizer`, `Attention`, `Transformer`, `Agent`, `RAG` |
| `工作流` | `Prompt`, `上下文管理`, `自动化`, `知识库运营` |
| `实验复现` | `训练记录`, `参数对比`, `失败排查` |
| `复盘案例` | `学习复盘`, `任务复盘`, `项目案例` |

## 标签建议

| 标签类型 | 示例 |
|----------|------|
| 主题标签 | `AI编码`, `Prompt`, `上下文管理`, `自动化`, `RAG`, `知识库` |
| 工具标签 | `Codex`, `ClaudeCode`, `Cursor`, `Obsidian`, `MCP` |
| 类型标签 | `笔记`, `踩坑`, `复盘`, `片段` |
| 状态标签 | `草稿`, `待验证`, `已归档` |

## 综述页（LLM 维护）

`【综述】` 类型由 LLM 跨文档提炼生成，`author: llm`，`sources` 必填：

```yaml
---
title: 【综述】AI编码工作流全景
tags: [AI编码, 工作流, 综述]
created: 2026-06-18
description: 跨多篇笔记提炼的 AI 辅助开发工作流全貌
author: llm
sources: ["【最佳实践】Prompt与上下文管理", "【教程】AI辅助开发工作流"]
status: 待验证
---
```
