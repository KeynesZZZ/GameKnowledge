## 知识沉淀原则

### 先沉淀流程，再沉淀工具

AI 工具更新很快，但稳定的工作流更值得长期保存。记录工具时，优先写清楚：

- 它解决什么问题
- 它处在流程的哪个环节
- 输入和输出是什么
- 失败时如何兜底

### 每篇文档都要可复用

文档不是聊天记录归档，而是下一次可以直接拿来用的工作资产。每篇文档至少提供一种可复用内容：

- Prompt 模板
- 检查清单
- 命令流程
- 决策标准
- 失败案例和修复方式
