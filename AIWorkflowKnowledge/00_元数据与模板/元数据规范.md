# AI工作流元数据规范

> 用统一元数据保证 Obsidian 检索、RAG 同步和长期维护都能稳定工作。

## 必填字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `title` | string | 带文档类型前缀的标题 | `【最佳实践】Prompt与上下文管理` |
| `tags` | array | 至少 2 个标签 | `[AI编码, Prompt, 上下文管理]` |
| `category` | string | 分类路径 | `AI编码方法论/最佳实践` |
| `created` | datetime | 创建时间 | `2026-05-29 00:00` |
| `updated` | datetime | 更新时间 | `2026-05-29 00:00` |
| `description` | string | 一句话摘要 | `AI辅助开发中的上下文组织与Prompt管理方法` |

## 可选字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `tools` | array | 涉及工具 | `[Codex, Claude Code, Cursor]` |
| `model_scope` | string | 适用模型范围 | `通用大语言模型` |
| `complexity` | string | 复杂度 | `简单` / `中等` / `复杂` |
| `use_frequency` | string | 使用频率 | `高频` / `中频` / `低频` |
| `status` | string | 文档状态 | `草稿` / `待验证` / `已归档` |
